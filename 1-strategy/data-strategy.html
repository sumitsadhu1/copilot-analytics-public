1. Why Upload Org Data? (Entra ID vs. Org Data Upload)

A common question: “We already have Department, Manager, and Organisation in Entra ID — why do we need to upload a CSV?”

Entra ID provides a baseline, but org data uploads unlock significantly more analytics capability:

CapabilityEntra ID OnlyWith Org Data Upload
Department filter on dashboardYes (Entra ID “Department” attribute)Yes (from CSV — can use different/cleaner values)
Job Function / Job Discipline filterNoYes (via FunctionType / Microsoft_JobDiscipline)
Location filterNoYes (via Location attribute)
Level / Seniority filterNoYes (via LevelDesignation)
Custom attributes (Cost Centre, Agency, BU, Region)NoYes (up to 105 custom attributes)
Multi-agency / multi-BU partitioningYes — if each agency has a distinct email domain, use the system-generated Domain attribute (no CSV needed)Yes — use a custom Agency attribute for maximum flexibility (works even when agencies share one domain)
Before/after cohort segmentation in Power BINo segmentationYes (segment by any uploaded attribute)
Manager hierarchy accuracyDepends on Entra ID completenessCan supplement/override with CSV ManagerId
Data freshness controlUpdated by directory team (often delayed)Updated by Insights Admin on a defined cadence
TIP: When Entra ID Alone Is Sufficient

If you only need the Copilot Dashboard with tenant-level views and Department filtering — and the Entra ID Department attribute is accurate and consistently populated — you can skip the org data upload. However, if you need any of the following, org data upload is required: Location or Job Function filters, custom attributes (Cost Centre, Agency), or before/after cohort analysis in Power BI.

2. What Breaks with an Incomplete Entra ID Hierarchy?

If the Entra ID Manager attribute is missing or inconsistent for many users:

FeatureImpact
Auto CxO IdentificationFewer leaders are auto-identified. Users without a clear reporting line are excluded from the “top 3 levels” calculation.
Manager Settings (team-level data)Managers without a populated Manager attribute or without enough direct reports won't see team-level data.
Filter by manager hierarchy (Publish Reports)Published reports with “Filter by manager hierarchy” will show incomplete data.
Advanced Analytics cohort segmentationManager-based grouping will be inaccurate.

Recommendation: If more than 20% of users lack a Manager attribute, prioritise fixing directory data or supplement with an org data CSV upload that includes correct ManagerId values.

3. The Data Source Change Warning

Once org data is uploaded, it replaces Entra ID as the data source for the uploaded attributes in dashboard filters:

  • Department values come from the CSV, not from Entra ID — even if they differ
  • If you stop uploading, the data becomes stale (not reverted to Entra ID)
  • Reversal is now self-service: You can use parallel data ingestion to keep Entra as the source for ManagerId and Organization while adding custom attributes via CSV. To stop using uploaded data entirely, clear the Entra connection selections or use the “Replace All data” option with a single dummy row. See Microsoft Learn.
  • Before uploading: Confirm readiness to own the ongoing refresh cycle (monthly re-uploads recommended)

4. Choose Your Upload Method

Start with "What HRIS do you use?" — not "how do I upload a CSV."

MethodWho ManagesStatusBest For
SAP SuccessFactors ConnectorGlobal Admin + SAP SF AdminGAOrganisations using SAP SuccessFactors. No CSV needed. Auto-sync weekly/monthly.
Workday ConnectorGlobal Admin + Workday AdminGAOrganisations using Workday (≤100K users). For >100K, use Workday RaaS → Azure Blob.
Azure Blob StorageGlobal Admin + Data EngineerGAAutomated CSV drop. Good for data engineering teams or Workday RaaS export.
API-Based ImportGlobal Admin + DeveloperPreviewProgrammatic push via REST API. Supports incremental refresh.
CSV Upload — M365 Admin CenterOrg Data Source AdministratorGAUniversal fallback. Lowest technical barrier. Works with any HRIS (export to CSV).
CSV Upload — Viva InsightsInsights AdministratorDeprecated (mid-2026)Legacy path. Use only if your tenant has not been migrated.
NOTE: Decision Logic
  • Use SAP SuccessFactors or Workday if your org runs either HRIS — eliminates manual CSV entirely
  • Use Azure Blob Storage if you want automated CSV drops from a data warehouse
  • Use API-based import if you need incremental refresh or have a custom HRIS
  • Use CSV upload for everything else — universal compatibility
TIP: Bypass On-Premises AD

If your HRIS data flows through SAP SuccessFactors → middleware → on-premises AD → Entra ID, be aware that Viva Insights only picks up 3 attributes from Entra (PersonId, ManagerId, Organization). Custom fields in Entra extensionAttributes are not synced.

Solution: Use the HRIS connector to send data directly to M365 — bypassing AD entirely for analytics purposes.

Complete this before any upload. Share answers with your IT team when you hand off to the Data Onboarding Guide.

6. Organizational Data Upload Migration

IMPORTANT: April 2026 Migration

Microsoft is migrating all tenants from the Viva Insights web app to the M365 Admin Center as the single upload point. By mid-2026, all tenants will be migrated.

  • New platform: Organizational Data in Microsoft 365 at admin.microsoft.com → Setup → Migration and imports
  • New role: Organizational Data Source Administrator manages uploads
  • Supports HRIS connectors, Azure Blob, API-based import — not just CSV
  • Data serves multiple M365 apps (Copilot Dashboard, Viva Insights, M365 User Profile)

Which path am I on?

  1. Go to admin.microsoft.com → Setup → Migration and imports
  2. If you see "Organizational Data in Microsoft 365" → you're migrated → follow the M365 Admin Center path in the Data Onboarding Guide
  3. If you don't see it → you're pre-migration → follow the Viva Insights legacy path
  4. If unsure → check Message Center for migration notifications
