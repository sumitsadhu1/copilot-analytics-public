1. Why Upload Org Data? (Entra ID vs. Org Data Upload)

A common question: “We already have Department, Manager, and Organisation in Entra ID — why do we need to upload a CSV?”

Entra ID provides a baseline, but org data uploads unlock significantly more analytics capability:

CapabilityEntra ID OnlyWith Org Data Upload
Department filter on dashboardYes (Entra ID “Department” attribute)Yes (from CSV — can use different/cleaner values)
Job Function / Job Discipline filterNoYes (via FunctionType / Microsoft_JobDiscipline)
Location filterNoYes (via Location attribute)
Level / Seniority filterNoYes (via LevelDesignation)
Custom attributes (Cost Centre, Agency, BU, Region)NoYes (up to 105 custom attributes)
Multi-agency / multi-BU partitioningYes — if each agency has a distinct email domain, use the system-generated Domain attribute (no CSV needed)Yes — use a custom Agency attribute for maximum flexibility (works even when agencies share one domain)
Before/after cohort segmentation in Power BINo segmentationYes (segment by any uploaded attribute)
Manager hierarchy accuracyDepends on Entra ID completenessCan supplement/override with CSV ManagerId
Data freshness controlUpdated by directory team (often delayed)Updated by Insights Admin on a defined cadence
TIP: When Entra ID Alone Is Sufficient

If you only need the Copilot Dashboard with tenant-level views and Department filtering — and the Entra ID Department attribute is accurate and consistently populated — you can skip the org data upload. However, if you need any of the following, org data upload is required: Location or Job Function filters, custom attributes (Cost Centre, Agency), or before/after cohort analysis in Power BI.

2. What Breaks with an Incomplete Entra ID Hierarchy?

If the Entra ID Manager attribute is missing or inconsistent for many users:

FeatureImpact
Auto CxO IdentificationFewer leaders are auto-identified. Users without a clear reporting line are excluded from the “top 3 levels” calculation.
Manager Settings (team-level data)Managers without a populated Manager attribute or without enough direct reports won't see team-level data.
Filter by manager hierarchy (Publish Reports)Published reports with “Filter by manager hierarchy” will show incomplete data.
Advanced Analytics cohort segmentationManager-based grouping will be inaccurate.

Recommendation: If more than 20% of users lack a Manager attribute, prioritise fixing directory data or supplement with an org data CSV upload that includes correct ManagerId values.

3. Source Precedence Is Surface- and Attribute-Specific

An upload changes the source for the attributes supplied to the apps you select; it does not universally or permanently replace Microsoft Entra ID across Microsoft 365.

Surface / attributeControlling behaviourOperational rule
Microsoft 365 User ProfileMicrosoft Entra is the default source. Organizational Data in Microsoft 365 can be prioritised; when Entra is missing or incomplete, uploaded data can fill gaps.Keep Profile precedence at Entra unless there is an approved reason to override it.
Copilot Dashboard and Viva InsightsUploaded data shared with these apps supplies the mapped dashboard and analysis attributes. If both Viva and Microsoft 365 uploads exist, Microsoft documents a merge with the more recent upload taking priority.Document the owner and source for every supplied attribute; verify values in Dashboard filters and Viva data quality after each change.
Viva ManagerId and OrganizationParallel ingestion can make Entra the default for either attribute, even when the upload also contains it. Other uploaded fields continue to come from CSV or connectors.Use Configure Entra connection when the directory owns these two attributes; allow about 24 hours for the change.
Uploaded optional and custom attributesThey remain upload/connector sourced unless an app has its own documented precedence rule.Do not assume an Entra fallback. Keep the source current and test each consuming app.

Sources: Organizational Data in Microsoft 365 — Data precedence (Microsoft Learn, updated 2 December 2025); Manage settings for the Microsoft Copilot Dashboard, Agent Dashboard, and Viva Insights web app (20 March 2026); and Use Microsoft Entra plus .csv files for parallel data uploads (visible date 30 June 2025; page content refreshed 7 July 2026).

Supported correction and rollback paths

  • Update: Upload only affected users and all attributes you intend to change. A blank value preserves the existing value because new uploads merge with previous data.
  • Delete a value: Use the documented type-specific marker: two single quotes ('') in CSV for strings (three in Excel), -1 for integers, a nonexistent in-tenant address for email values, or 01-01-0001 for dates.
  • Correct history: Use Historical import with a CSV and a File valid as of date. Microsoft documents historical updates for Viva Insights data only.
  • Return ManagerId/Organization to Entra: Enable those attributes under Configure Entra connection, then delete uploaded optional attributes you no longer need.
  • Migration failure: Stop further imports and open a Microsoft support request. Microsoft documents temporary rollback only for critical migration issues in special cases.
Do Not Use a Dummy-Row Rollback

Do not use a one-row “Replace All” file as a general removal or rollback technique. It can leave an artificial hierarchy and break filters, partitions, or reports. Use the supported update/delete markers, parallel Entra restoration, or Microsoft Support instead.

Source: Import and manage your organizational data with a .csv file — Update or make other changes (Microsoft Learn, updated 2 April 2026), including retroactive updates; migration rollback is documented in Upload and maintain data through the Microsoft 365 admin center (5 December 2025).

4. Choose Your Upload Method

Start with "What HRIS do you use?" — not "how do I upload a CSV."

MethodWho ManagesStatusBest For
SAP SuccessFactors ConnectorGlobal Admin + SAP SF AdminGAOrganisations using SAP SuccessFactors. No CSV needed. Auto-sync weekly/monthly.
Workday ConnectorGlobal Admin + Workday AdminSupported, up to 100,000 usersWorkday SOAP integration. For larger datasets or where SOAP is unsuitable, use Workday RaaS through Azure Blob or API import.
Azure Blob StorageGlobal Admin + Data EngineerPublic Preview; Worldwide onlyAutomated CSV drop. Unavailable in GCC; confirm tenant rollout before production use.
API-Based ImportGlobal Admin + DeveloperPublic Preview; Worldwide onlyProgrammatic import using CSV payloads. Unavailable in GCC; confirm tenant rollout before production use.
CSV Upload — M365 Admin CenterOrg Data Source AdministratorGAUniversal fallback. Lowest technical barrier. Works with any HRIS (export to CSV).
CSV Upload — Viva Insights web appInsights AdministratorOfficial sources conflict; tenant-specificDo not infer availability from a date. Follow Message Center and the actual tenant UI; use only when both explicitly show add/edit upload is supported.

Sources: Import organizational data from Workday (Microsoft Learn, updated 6 April 2026); Import organizational data using API-based import (preview) (6 April 2026); and Supported Microsoft 365 environments for Viva Insights features (30 June 2026).

NOTE: Decision Logic
  • Use SAP SuccessFactors or Workday if your org runs either HRIS — eliminates manual CSV entirely
  • Consider Azure Blob Storage for automated CSV drops from a data warehouse, subject to public-preview, cloud, and tenant availability
  • Consider API-based import for programmatic integration or a custom HRIS, subject to public-preview, cloud, and tenant availability
  • Use CSV upload for everything else — universal compatibility
TIP: Bypass On-Premises AD

If your HRIS data flows through SAP SuccessFactors → middleware → on-premises AD → Entra ID, be aware that Viva Insights only picks up 3 attributes from Entra (PersonId, ManagerId, Organization). Custom fields in Entra extensionAttributes are not synced.

Solution: Use the HRIS connector to send data directly to M365 — bypassing AD entirely for analytics purposes.

Complete this before any upload. Share answers with your IT team when you hand off to the Data Onboarding Guide.

6. Determine the Current Tenant State

Official Microsoft Sources Conflict

Manage settings for the Microsoft Copilot Dashboard, Agent Dashboard, and Viva Insights web app (20 March 2026) still calls Viva web-app upload the recommended path for Viva Insights customers. Use Microsoft Entra plus .csv files for parallel data uploads (visible date 30 June 2025; refreshed 7 July 2026) says adding and editing uploads is no longer supported there. Upload and maintain data through the Microsoft 365 admin center (5 December 2025) says migration occurs in tenant-specific waves announced in Message Center.

Safest rule: Message Center and the actual tenant UI control the procedure. A calendar date or missing tile alone does not prove which path is supported.

Current-state gate

  1. Read the latest tenant-specific Message Center post for organizational-data migration.
  2. Open admin.microsoft.com → Setup → Migration and imports and record whether Organizational Data in Microsoft 365 and its add/edit controls are available.
  3. Open the Viva Insights web app and record whether organizational-data add/edit, delete, and Replace All controls are present. Do not assume all four move together.
  4. Review existing connections so a second source is not introduced unintentionally.
  5. If Message Center and the live UI agree, follow that path in the Data Onboarding Guide.
  6. If they conflict, or neither surface offers a supported add/edit path, stop. Capture the Message Center post ID/date, screenshots, roles, tenant cloud, and existing connection status; then escalate through Microsoft Support or the account team before uploading.
