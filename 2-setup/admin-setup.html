NOTE: Copilot Licence = Full Viva Insights Access

Every M365 Copilot licence includes the Viva Insights service plan at no additional cost. This covers all Viva Insights capabilities — including behavioural collaboration metrics, Copilot Dashboard, Copilot Analytics Reports, Agent Dashboard, and Advanced Analytics (Power BI). There is no separate “premium” tier to purchase if you already have Copilot licences.

NOTE: 50-Licence Threshold

The 50-licence threshold required for full dashboard features can be met by a combination of Copilot and Viva Insights licences (e.g., 30 Copilot + 20 Viva Insights = 50). Below 50, you get only the Readiness page and tenant-level adoption metrics.

WARNING: AI Administrator Self-Access

The AI Administrator role does NOT grant automatic access to view the Copilot Dashboard. AI Admins must explicitly add themselves via Admin Centre → Settings → Viva → Viva Insights → Manage user access for Copilot Dashboard. This is the most common point of confusion during implementations.

WARNING: Multi-Agency Tenants

If your tenant hosts multiple independent agencies that must not see each other’s Copilot data, do not follow the standard enablement steps below. Auto CxO Identification and broad manager access grant dashboard access showing “Your company” data across all agencies. Instead, follow the Multi-Agency Setup Guide which prescribes a lockdown-first posture.

3. Tier 1: Admin Centre Reports

Estimated Time: 15–30 minutes (plus 72 hours for data population)

The M365 Admin Centre Copilot reports are built-in and require no additional configuration. They provide user-level licence utilisation and per-application usage data.

Step 1: Verify Admin Role (5 min)
  1. Navigate to Microsoft 365 Admin Centre: https://admin.microsoft.com
  2. Go to Users → Active users → Select your account
  3. Click Manage roles and verify you have Reports Reader or higher
  4. If not, request role assignment from Global Administrator
RoleAccess LevelCan Export CSV?
Global AdministratorFull access to all reports and settingsYes
Reports ReaderFull access to usage reportsYes
Usage Summary Reports ReaderAggregate data only; no user-level detailsNo
Step 2: Access Reports → Usage → Microsoft 365 Copilot (5 min)
  1. In Admin Centre, navigate to Reports → Usage
  2. Scroll to find Microsoft 365 Copilot or use the search function
  3. Click to open the Copilot usage report
Step 3: Configure Report Parameters (5 min)
  1. Select timeframe: 7, 30, 90, or 180 days
  2. Review summary cards: Total assigned licences, Active users, Adoption rate
  3. Scroll to per-application breakdown (Word, Excel, PowerPoint, Outlook, Teams)
Step 4: Export User-Level CSV Data (5 min)
  1. Click Export in the top right corner
  2. Select CSV format
  3. Open downloaded file to verify user-level activity data

Success Criteria

  • Can access Admin Centre → Reports → Usage → Microsoft 365 Copilot
  • Summary cards display: Assigned licences count, Active users count, Adoption percentage
  • Per-application breakdown shows usage across Word, Excel, PowerPoint, Outlook, Teams
  • Can export CSV file with user-level activity data
  • Data is no older than 72 hours (check report timestamp)
NOTE: Data Availability
Reports populate within approximately 72 hours of user activity. If this is a new Copilot deployment, allow 3 business days before expecting data.

4. Tier 2: Copilot Dashboard Enablement

Estimated Time: 2–4 hours active configuration + up to 7 days for data processing

The Copilot Dashboard is part of the Viva Insights web app. It provides ROI metrics (Copilot Assisted Hours & Value), adoption trends, and sentiment tracking at group/department level.

Step 1: Verify Licence Threshold (10 min)
  1. Navigate to https://admin.microsoft.com
  2. Go to Billing → Licenses
  3. Locate Microsoft 365 Copilot and note the assigned count (not purchased)
  4. Confirm 50 or more licences are assigned
  5. If less than 50, check whether Viva Insights licences can make up the difference (e.g., 30 Copilot + 20 Viva Insights = 50)
Step 2: Verify Entra ID Hierarchy (15 min)
  1. Navigate to https://entra.microsoft.com
  2. Go to Users → select several sample users across different levels
  3. For each user, verify the Manager attribute is populated
  4. If more than 20% of users lack a Manager attribute, work with your directory team to populate the hierarchy before proceeding
  5. Note: Automatic CxO access criteria are analysed weekly by Microsoft
Step 3: Verify VFAM Policies Are Enabled (10 min)

Requires AI Administrator or Global Administrator role.

  1. Navigate to https://admin.microsoft.com
  2. Go to Settings → Microsoft Viva → Viva Insights
  3. Select the Set up and management tab
  4. Under Access management, select Turn Advanced Insights on or off
  5. Verify the following policies are set to On:
PolicyRequired StateWhat Happens If Off
Viva Insights web app (replaces former “Copilot Dashboard” toggle)OnThe entire Viva Insights web app is disabled — no Copilot Dashboard, no Analytics Reports, no admin or analyst tools
Copilot Dashboard DelegationOnUsers cannot share dashboard access with delegates
Behavioural ExperiencesOnCollaboration data (meeting time, email, chat) does not flow — breaks Impact page comparisons
Auto CxO Identification (Dashboard AutoEnablement)On (recommended)Senior leaders do not get automatic access — must be added manually
Copilot Advanced AnalyticsOn (if planning Tier 5)Analysts cannot access the Advanced Analysis workbench

If any policy shows Off or is scoped to specific users instead of Everyone, verify whether this was intentional. If not, select the policy → edit → set to On for Everyone (or the appropriate scope).

IMPORTANT: This Is the #1 Cause of “Dashboard Not Working”

If a previous admin created a custom VFAM policy that restricts the Copilot Dashboard to specific users, new users added via the access management settings will not be able to view the dashboard unless they are also included in the VFAM policy. Always check this screen first when troubleshooting access issues.

Accidental VFAM disable — blast radius & recovery:

  • If the policy is turned Off, all users (auto-identified leaders, manually added users, delegates) immediately lose dashboard access
  • Delegates are not removed — their delegation records survive, but they can’t access the dashboard until the policy is re-enabled
  • Recovery: Turn the policy back On for Everyone (or the required scope). Access takes up to 24 hours to propagate. No data is lost.
  • Prevention: Restrict VFAM policy editing to Global Administrators only. Document which policies are enabled and why.
Step 4: Configure Dashboard Access — Who Should See It (20 min)

Requires AI Administrator or Global Administrator role.

There are three ways users get access to the Copilot Dashboard. Decide which apply:

Access MethodWho Gets AccessHow to Configure
Automatic (Entra ID) Senior leaders in top 3 org levels with significant reporting lines (tenants with 2,500+ users) No action needed — happens automatically if Auto CxO Identification is ON. Refreshed weekly.
Manual (Admin-assigned) Specific users or groups you choose — AI Admins, BU leaders, HR, analysts, etc. Admin Centre → Settings → Viva → Viva Insights → Set up and management → Under Copilot Dashboard, select Manage user access → Add users/groups
Delegated People that existing dashboard users choose to share with (max 75 per person) Configured by users themselves via Settings → Delegation in the Viva Insights web app. No admin action needed.

Recommended approach: Enable automatic access for senior leaders + manually add the AI Admin, Insights Admin, Insights Analyst, and any additional stakeholders (BU leaders, HR, change management leads).

Step 5: Enable Manager Access to Team-Level Data (15 min)

Requires Insights Administrator or AI Administrator role.

  1. Navigate to Viva Insights web app: https://insights.cloud.microsoft
  2. Go to Manager Settings (under Platform Setup on the left)
  3. Toggle on “Grant managers scoped access to Copilot insights”
  4. Set the Minimum team size (minimum 5; includes manager in count)
  5. Choose which managers to enable:
Audience OptionDescription
All managersAll measured managers meeting team size threshold
Select Active Directory groupChoose a specific Entra ID group of managers
Upload .csv fileList specific manager email addresses
  1. Click Save changes (takes effect within 1 hour)

Alternative path via Admin Centre: Settings → Viva → Viva Insights → Set up and management → Under Copilot Dashboard, select Turn Insights for Managers on or off and Assign group managers to Insights for Managers.

Step 6: Upload Organisational Data (30–45 min)

Organisational data (department, function, location, agency) enables group-level filtering and advanced segmentation on the Copilot Dashboard and in Power BI reports.

See the Data Onboarding Guide for full details on upload methods, attribute mapping, and troubleshooting.

Step 7: Configure Minimum Group Size (10 min)

Default is 10; minimum allowed is 5. This controls the privacy threshold — groups smaller than this value will show “N/A” instead of data.

For tenants WITHOUT Viva Insights licence:

  1. https://admin.microsoft.com → Settings → Microsoft Viva → Viva Insights
  2. Under Copilot Dashboard, select Manage minimum group size
  3. Enter preferred minimum (≥5) → Save (takes effect in 24 hours)

For tenants WITH Viva Insights licence:

  1. https://insights.cloud.microsoft → Admin → Privacy settings
  2. Configure minimum group size → Save (takes effect in 24 hours)
Step 8: Create Exclusion List (Optional, 20 min)

Requires AI Administrator role. Excludes specific users from all Copilot Dashboard and Viva Insights data.

  1. https://admin.microsoft.com → Settings → Microsoft Viva → Viva Insights
  2. Select Unified exclusion list
  3. Choose method: CSV upload (PersonId column), Entra ID groups, or Replace all
  4. Wait up to 5 days for exclusion to take effect
NOTE: Exclusion List Scope

The exclusion list removes users from both the Copilot Dashboard AND Viva Insights collaboration reports in Advanced Analysis. It also excludes M365 Copilot Chat (web) usage data.

Step 9: Configure Hourly Rate for ROI (10 min)
  1. Navigate to the Impact page on the Copilot Dashboard
  2. Select Hours and Value calculator, then select Value calculator
  3. Default hourly rate is $72/hour (USD, based on U.S. Bureau of Labor Statistics)
  4. Adjust to your loaded labour rate and select the appropriate currency
  5. Document the rate used for future reference

How to calculate a loaded labour rate: (Average base salary + benefits + overheads) ÷ annual productive hours (typically ~1,880).

WARNING: Hourly Rate Is Session-Only

As of March 2026, changes to the hourly rate and currency are not saved persistently. You must re-enter your preferred rate every time you reopen the Copilot Dashboard. This is a current platform limitation — the settings reset on page reload. Agree on the rate with stakeholders and document it so it can be consistently re-applied.

Step 10: Sentiment & Survey Configuration (30–60 min)

Usage metrics tell you how much Copilot is used. Sentiment tells you how employees feel about it. The Copilot Dashboard’s Impact page includes two distinct sentiment signals:

  • Satisfaction Rate (automatic) — Thumbs-up/down reactions to Copilot responses across M365 apps. No setup needed. Shows as a 28-day rolling metric.
  • Sentiment section (requires survey data) — Employee responses to Microsoft’s four recommended questions, compared against Microsoft’s benchmark.

Choose Your Sentiment Data Source

OptionBest ForLicence RequiredFilters & Heat Maps?Manager Team View?Setup Effort
A: Viva Glint Organisations with Viva Glint already deployed Viva Glint (part of Viva Suite or standalone) Yes — full group filters + heat maps Yes Medium
B: Viva Pulse Quick, lightweight surveys from the Dashboard Viva Pulse (included with M365 Copilot licence) Yes — with Viva Insights subscription Yes Low
C: CSV Upload Third-party survey tools (Qualtrics, SurveyMonkey, etc.) Viva Insights subscription Yes — with org data uploaded Yes Medium
D: Aggregated Upload Organisations without Viva Insights wanting a simple summary None (Global Admin only) No — tenant-level summary only No Low

Microsoft’s Four Recommended Copilot Survey Questions

All survey options use the same four questions. Include these verbatim to enable comparison with Microsoft’s global benchmarks. Respondents answer on a 5-point Likert scale (Strongly Disagree → Strongly Agree).

Question #Statement (use verbatim)
1Using Copilot helps improve the quality of my work or output
2Using Copilot helps me spend less mental effort on mundane or repetitive tasks
3Using Copilot allows me to complete tasks faster
4When using Copilot I am more productive

Option A: Viva Glint

  1. In the Viva Glint admin portal, create a new survey program using the Microsoft Copilot Impact Survey template.
  2. The template includes the four recommended questions pre-configured.
  3. Set the survey audience (all Copilot-licenced users, or specific groups).
  4. Set the survey cadence — Microsoft recommends recurring deployment to track trends.
  5. Launch the survey. Results flow automatically to the Dashboard’s Sentiment section.

Option B: Viva Pulse

  1. In the Copilot Dashboard, navigate to the Impact page.
  2. Scroll to the Sentiment section.
  3. Select Start new Pulse survey.
  4. The Copilot Impact template opens with the four questions pre-loaded.
  5. Customise audience and add optional extra questions → Send.
  6. Results automatically appear in the Dashboard Sentiment section once responses are collected.

Option C: CSV Upload

  1. Run your Copilot survey using any tool. Include the four recommended questions using the exact wording above.
  2. Export results and convert to a 100-point scale: Strongly Disagree=0, Disagree=25, Neutral=50, Agree=75, Strongly Agree=100.
  3. Format the CSV with columns: User AAD ID/Email address, Question 1, Question 2, Question 3, Question 4. Leave cells blank for unanswered questions.
  4. Navigate to insights.cloud.microsoft → Survey data → Data connections → Manage data sources.
  5. Under M365 Copilot Impact Survey data upload, select Start.
  6. Enter optional survey name and date range → browse to CSV → Upload.
  7. Results appear in the Dashboard Sentiment section within 2 days.

Option D: Aggregated Results (Admin Centre)

  1. Run your Copilot survey and compute the % of users who Agreed or Strongly Agreed with each question.
  2. Navigate to admin.microsoft.com → Reports → Adoption Score.
  3. Select AI adoption score → View details.
  4. Under Assess Copilot sentiment for your org, select Record survey results.
  5. Enter the aggregated percentages for each question → Save.
Priority Rule: If You Upload Both

If you upload survey data via both CSV (.csv file) and aggregated results (Admin Centre), only the CSV results are shown on the Dashboard. The aggregated results only appear if you subsequently delete the CSV data.

Step 11: Wait for Data Processing
  • Initial data processing: up to 7 days after licences assigned
  • Ongoing refresh: daily, 28-day rolling window, ~6-day delay from current date
  • Newly licenced employees: additional up to 7-day delay before inclusion

5. Tier 3: Copilot Analytics Reports

Copilot Analytics Reports are out-of-the-box reports that appear automatically in the Viva Insights web app once the Copilot Dashboard is operational. No additional setup is needed.

What you get: Targeted reports providing deeper analysis than the Dashboard — including usage patterns by application, adoption trends by group, and focused impact reports. These are available to all users who have Dashboard access.

What to verify:

  • Users with Dashboard access can see the Analytics Reports section in the Viva Insights web app left navigation
  • Reports reflect the same population as the Dashboard (same org data, same exclusion list)
NOTE: Same Prerequisites as Dashboard

Copilot Analytics Reports share all prerequisites with the Copilot Dashboard (Tier 2). If the Dashboard is working, Analytics Reports are automatically available. No separate setup is required.

6. Tier 4: Agent Dashboard

The Agent Dashboard is on by default as part of the Viva Insights web app. It shows adoption and usage data for Copilot agents (Copilot Studio agents, custom agents, etc.).

What to verify:

  1. Navigate to https://insights.cloud.microsoft
  2. Confirm the Agent Dashboard section is visible in the left navigation
  3. If not visible, verify that the Viva Insights web app VFAM policy is enabled (Tier 2 Step 3)

Delegated access note: Users who receive delegated access need “Your company” scope to view Agent Dashboard data. “Your group” scope does not include Agent data.

IMPORTANT: Team-Level Agent Data Not Available

The Agent Dashboard currently provides company-level and group-level data only. Team-level (manager) views are not available for agent usage data. Managers enabled via Manager Settings will see Copilot data for their team but not agent-specific breakdowns.

7. Tier 5: Advanced Analytics Setup

Estimated Time: 4–8 hours active + query runtime + ongoing maintenance

Advanced Analytics enables custom Power BI reports using Viva Insights person queries. This is for analysts who need before/after adoption comparison, custom cohort segmentation, and historical data beyond the 28-day rolling window.

Step 1: Assign Insights Analyst Role (15 min + 1–2 hours propagation)
  1. Identify the analyst(s) who will manage reports
  2. In M365 Admin Centre: Settings → Viva → Viva Insights → Add-on Plan → Insights Analyst
  3. Add users → Save
  4. Wait 1–2 hours for permissions to propagate
  5. Verify: analyst should see “Create analysis” button in Viva Insights web app left navigation
Step 2: Configure Partition Access (15 min)

A partition defines which employees an analyst can query. Most tenants have a default Global partition covering all employees.

To assign an analyst to an existing partition:

  1. Navigate to https://insights.cloud.microsoft
  2. Go to Admin → Privacy settings → Partitions
  3. Select the partition (usually Global) → click Edit
  4. Under the Analysts tab, select Add analysts
  5. Search for the Insights Analyst by name or email → select → Save
  6. The analyst can access this partition’s data within 1–2 hours
NOTE: Global Partition = Dashboard Access

An Insights Analyst assigned to the Global partition automatically receives Copilot Dashboard access. This is a built-in shortcut — no separate dashboard access configuration is needed for global partition analysts.

Step 3: Upload Organisational Data

If not already completed in Tier 2 Step 6, upload org data now. Advanced Analytics segmentation (by department, function, location, agency) requires org data. See the Data Onboarding Guide for full instructions.

Step 4: Install Power BI Desktop
  1. Download Power BI Desktop (version June 2022 or later)
  2. Install on the analyst’s workstation
  3. Sign in with the same Microsoft 365 account that has the Insights Analyst role
  4. Verify connectivity: in Viva Insights → Create analysis → select a template → choose “Connect to Power BI”
IMPORTANT: Insights Admin ≠ Insights Analyst

These are two separate roles with different capabilities:

RoleCan Configure Settings?Can Create/Run Queries?Can Upload Org Data?
Insights AdminYesNoYes
Insights AnalystNoYesNo
Both roles combinedYesYesYes

If a user opens the Viva Insights web app and sees the Admin panel but no “Create analysis” button, they have the Insights Admin role only. The “Create analysis” button only appears for users with the Insights Analyst role. If one person needs to do both, assign both roles.
