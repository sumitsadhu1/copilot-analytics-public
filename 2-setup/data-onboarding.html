1. Determine the Current Tenant State

Do not infer the supported upload path from a calendar date or from one missing tile. The migration is tenant-specific, and current Microsoft Learn pages conflict about whether Viva Insights web-app add/edit uploads remain supported.

  1. Read the latest tenant-specific Message Center post for organizational-data migration and record its post ID and date.
  2. Open admin.microsoft.com → Setup → Migration and imports. Record whether Organizational Data in Microsoft 365 and its Get started or New import controls are available.
  3. Open the Viva Insights web app. Record whether organizational-data add/edit, delete, and Replace All controls are present; these capabilities can differ.
  4. Review existing connections before creating another source.
  5. If Message Center and the tenant UI agree, use that path. If they conflict, or neither surface offers add/edit, stop and escalate with screenshots, roles, tenant cloud, Message Center evidence, and connection status.
Official Microsoft Sources Conflict

Manage settings for the Microsoft Copilot Dashboard, Agent Dashboard, and Viva Insights web app (Microsoft Learn, updated 20 March 2026) still recommends Viva web-app upload for Viva Insights customers. Use Microsoft Entra plus .csv files for parallel data uploads (visible date 30 June 2025; content refreshed 7 July 2026) says add/edit uploads are no longer supported there. Upload and maintain data through the Microsoft 365 admin center (5 December 2025) says tenant timing is communicated in Message Center. Follow the actual tenant state; do not silently choose one article over another.

TIP: Start From Microsoft’s Official CSV Template

For CSV upload (Options A and B), start from Microsoft’s official organizational data .csv template on Microsoft Learn — it documents the required headers, structure, and formatting rules, which helps you avoid the most common validation errors. On the M365 Admin Center path, you download the template in-product at Prepare organizational data → Download CSV template.

NOTE: Decision Logic
  • Running SAP SuccessFactors? → Use Option C (connector). Zero CSV.
  • Running Workday (≤100K users)? → Use Option D (connector). Zero CSV.
  • Running Workday (>100K) or have a data warehouse? → Consider Workday RaaS through Option E (Azure Blob) or Option F (API), subject to preview and cloud availability.
  • Need programmatic control or custom HRIS? → Consider Option F (API), subject to preview and cloud availability.
  • Everything else? → Use Option A (CSV to M365 Admin Center). Universal fallback.
  • Viva add/edit controls still explicitly enabled? → Use Option B only when Message Center and tenant UI agree.

TIP: Bypass On-Premises AD

If your HRIS data flows through SAP SuccessFactors → middleware → on-premises AD → Entra ID, be aware that Viva Insights only picks up 3 attributes from Entra (PersonId, ManagerId, Organization). Custom fields in Entra extensionAttributes are not synced.

Solution: Use the HRIS connector to send data directly to M365 — bypassing AD entirely for analytics purposes.

3. Upload Steps — M365 Admin Center (Primary Path)

Use this path when Message Center and the live Microsoft 365 admin center show that add/edit organizational-data imports are supported for the tenant.

Step 1: Ensure Role Assignment

The person performing the upload must have the Organizational Data Source Administrator role assigned in Entra ID. This role manages Organizational Data in Microsoft 365 and is separate from the Insights Administrator role.

  • Assign via: admin.microsoft.com → Users → Active Users → select user → Manage roles → search for “Organizational Data Source Administrator”

Step 2: Prepare Your CSV File

  • Encoding: UTF-8 (with or without BOM)
  • File size: Upload directly from local files up to 25 MB. For larger CSV files, upload through SharePoint; the documented SharePoint file limit is 2 GB.
  • Attribute naming: Use Microsoft_* naming convention (e.g., Microsoft_PersonEmail, Microsoft_ManagerEmail, Microsoft_Organization)
  • No EffectiveDate column: The M365 Admin Center uses a date picker in the UI, not a CSV column
  • No spaces in header names: Use underscores (e.g., Cost_Center not Cost Center)
  • One row per employee: Include every employee you want reflected in analytics

Source: Import and manage your organizational data with a .csv file — Step 3 (Microsoft Learn, updated 2 April 2026).

Step 3: Navigate to Data Connections

admin.microsoft.com → Setup → Migration and imports → Organizational Data in Microsoft 365 → Data Connections

Step 4: Start Import

  • First upload: Select Get started
  • Subsequent uploads: Select New import

Step 5: Select Target Apps

Choose which M365 applications will receive the uploaded data:

  • Viva Insights — required for Copilot Dashboard and Advanced Analytics
  • Copilot Dashboard — enables org-level filters on the dashboard
  • Microsoft 365 Profile — populates org data in user profiles

You can select all that apply. Each app may have its own required attributes.

Step 6: Upload File and Map Reserved Attributes

  • Upload your CSV file
  • Map the reserved attributes: Microsoft_PersonEmail, Microsoft_ManagerEmail, Microsoft_Organization
  • The system will auto-detect columns that match Microsoft_* naming

Step 7: Review Custom Attributes and Configure Access

For each custom attribute, set the access level:

Access LevelWho Can SeeUse Case
PublicAll selected apps and their usersDepartment, Location, Job Function
ConfidentialOnly authorised analysts (Viva Insights only)HourlyRate, Salary Band, Performance Rating
CustomSpecific apps onlyAttributes shared with Profile but not Insights

Step 8: Validate and Submit

  • Review the summary and submit
  • Processing time: A few hours for small files, up to 3 days for large files (>100K rows)
  • Monitor status in the Data Connections page: Validating → Processing → Complete

4. Upload Steps — Viva Insights (Only When Tenant-Enabled)

Do Not Infer This Path from a Missing Admin-Center Tile

Follow these steps only when the tenant’s Message Center notice and live Viva Insights UI explicitly show add/edit CSV upload is supported. If the signals conflict, stop and escalate. Microsoft Learn currently contains conflicting statements about this path; see Section 1.

Step 1: Prepare Your CSV File

  • Encoding: UTF-8 (with or without BOM)
  • File size: Follow the limit shown in the live Viva upload UI; the 25 MB local / 2 GB SharePoint limits above apply to Organizational Data in Microsoft 365, not automatically to this tenant-controlled path.
  • Include EffectiveDate column: The date must be earlier than the upload date (e.g., if uploading today, use yesterday’s date or earlier)
  • Use Viva Insights attribute names: PersonId, ManagerId, Organization

Step 2: Navigate to Data Hub

analysis.insights.cloud.microsoft → Admin → Data hub → .csv upload → Start

Step 3: Upload and Map Fields

  • Upload your CSV file
  • Map required fields: PersonId, ManagerId, Organization, EffectiveDate
  • Map optional fields as needed

Step 4: Validate and Submit

  • Review validation results
  • Processing time: Up to 7 days for large files
  • Monitor status in the Data hub page

6. Post-Upload Verification Checklist

After every upload, complete this checklist to confirm data quality before relying on dashboards.

Step 1: Check Import Status

StatusMeaningAction
Success All rows processed. No errors. Proceed to Step 2.
Partial Success Most rows processed. Some rows had warnings (e.g., unmatched manager emails). Download import log (Step 2). Fix and re-upload problem rows.
Failed File rejected. No data was imported. Check error details. Common causes: missing required columns, encoding issues, attribute name conflicts.
Suspended Processing was halted (usually by system or admin). Check Data Connections for details. Re-submit if needed.

Step 2: Download Import Log

Look for rows with unmatched Microsoft_ManagerEmail values. These indicate managers whose email addresses don’t exist in the uploaded data or in Entra ID.

Step 3: Download Manager Log

This log shows employees whose ManagerEmail doesn’t resolve to an active user in Entra ID. These employees will appear as “orphaned” in the manager hierarchy.

Step 4: Check Attribute Coverage

Navigate to the Data Quality tab. Use an organization-approved acceptance threshold; this guide uses >95% coverage for critical segmentation attributes as an operational target, not as a Microsoft product requirement. Investigate any material drop from the pre-change baseline, because missing values can exclude people from filtered views.

Step 5: Check Manager Tree Count

The number of root nodes in the manager hierarchy should be small (typically 1–5, representing CEO/top executives). If you see hundreds or thousands, many ManagerEmail values are missing or incorrect.

Step 6: Spot-Check Dashboard

  • Open the Copilot Dashboard and verify the Organisation filter shows your uploaded values (not just Entra defaults)
  • Confirm new employees appear in the population count
  • Check that Job Function, Location, and other optional filters are populated (if uploaded)

Step 7: Approve Connector Data (If Applicable)

If using an HRIS connector (SAP SuccessFactors or Workday) with “Manually approve” enabled: you have a 48-hour review window to inspect the data before it goes live. If not approved within 48 hours, the data is automatically approved.

7. Re-Upload & Maintenance

Refresh Behaviour by Method

MethodRefresh BehaviourKey Differences
M365 Admin Center CSV Incremental merge Upload only affected users. Rows and values omitted from the new file remain unchanged; blank values also preserve the previously uploaded value. Use documented type-specific deletion markers when a value must be removed.
Tenant-enabled Viva Insights CSV Full replacement Must re-upload all employees every time. Missing employees will lose their org data.
HRIS connectors Automated (weekly/monthly) No manual uploads required. Connector syncs on schedule. Monitor connector health in Data Connections.
API-based import Incremental or full (configurable) Controlled via automation. Supports programmatic incremental refresh.

Common Maintenance Scenarios (M365 Admin Center CSV)

ScenarioWhat to Upload
New hire 1 row with the new employee’s data (incremental merge adds them)
Department change 1 row with updated Microsoft_Organization value (merge updates existing record)
Employee left Update the authoritative employment-status/source record and the app-specific attributes required by your retention and DSR process. Do not use Layer=-1 as a universal user-deletion instruction.
Historical correction Use the Historical import option in Data Connections to submit data for a past date

Supported Update, Delete, and Historical-Correction Operations

ChangeSupported operation
Update current valuesUpload only affected users and include each attribute to update. Blank or omitted values preserve existing data.
Delete a string valueUse two single quotes ('') in CSV, or three single quotes in Excel.
Delete an integer valueUse -1 for that integer attribute.
Delete an email valueUse a nonexistent email address in the tenant, such as noname@contoso.com.
Delete a date valueUse 01-01-0001.
Correct historical Viva Insights dataStart a CSV Historical import, select the File valid as of date, and upload only the impacted employees with corrected values. Historical update is documented for Viva Insights only.

Source: Import and manage your organizational data with a .csv file — Update or make other changes and Make retroactive updates (Microsoft Learn, updated 2 April 2026).

Recommended Cadence

  • CSV upload: Monthly (minimum). Set a recurring calendar reminder.
  • HRIS connectors: Weekly or monthly (configured in connector settings)
  • API-based import: As frequently as your automation runs (daily is possible)
Do Not Use a Dummy-Row Rollback

A one-row “Replace All” upload can create an artificial hierarchy and disrupt filters, partitions, or reports. Use a corrective upload, documented deletion markers, parallel Entra restoration, or Microsoft Support.

PhaseRequired action and evidence
1. Baseline and backupPreserve the exact source CSV, mapping, selected apps, access classifications, file-valid date, connection configuration, import ID, and pre-change population/coverage counts. Export current organizational data where the authorized app exposes export, and capture representative Profile and Dashboard values.
2. PreflightConfirm Message Center and tenant UI path, tenant cloud, active connections, required attributes, 25 MB local / 2 GB SharePoint route, valid UPNs, manager-chain integrity, no unexpected row-count change, no prohibited personal fields, and no attributes on which partitions or auto-refresh queries depend being removed.
3. Validate and approveRun import validation, review every error/warning and the attribute mapping tally, and use manual approval when the connection offers it. Do not approve unexplained drops in people, hierarchy roots, or critical-attribute coverage.

EscalateOpen Microsoft Support when Message Center and UI conflict, processing exceeds the documented window, a migration causes critical issues, a required delete/restore operation is unavailable, or tenant behaviour differs from Learn. 9. Key Rules

  • Upload for every employee — include every person you want reflected in analytics, not just Copilot-licensed users
  • No TimeZone column — TimeZone is system-generated from Outlook settings. Including it in your CSV causes a validation error.
  • M365 Admin Center path: no EffectiveDate column — the date is selected via the UI date picker
  • M365 Admin Center path: no spaces in header names — use underscores (Cost_Center, not Cost Center)
  • Tenant-enabled Viva Insights add/edit path: include EffectiveDate — follow the date rule shown in the live workflow and current Microsoft guidance
  • Incremental merge is the default in the M365 Admin Center — you only need to upload changed or new rows. Unmentioned employees retain their existing data.
  • Data precedence is surface- and attribute-specific — Entra is the default for Microsoft 365 User Profile, while selected uploads supply mapped Copilot Dashboard/Viva attributes; parallel ingestion can return ManagerId and Organization to Entra.
  • One import at a time per tenant — if an import is in progress, you must wait for it to complete before starting another

10. Common Upload Errors

ErrorCauseFix
Blank Microsoft_PersonEmail rows Rows with empty email addresses Remove blank rows or populate with valid UPNs. Validation fails if any required field is blank.
Missing required attributes for selected apps Target app (e.g., Viva Insights) requires specific attributes that are not in the CSV Check which attributes each selected app requires and add them to the CSV.
Empty data rows Rows with all-blank values (often trailing rows from Excel export) Remove empty rows from the CSV before upload. These block validation.
Sending data to unlicensed apps Selected a target app that your tenant doesn’t have licences for Deselect the unlicensed app in Step 5 or acquire the required licences.
Custom attributes sent to unsupported apps Not all apps support custom attributes Review app compatibility for custom attributes. Viva Insights supports custom attributes; others may not.
Attribute name conflicts with Microsoft_ reserved names Custom attribute uses a name that starts with Microsoft_ and conflicts with reserved attributes Rename the custom attribute to avoid the Microsoft_ prefix, or map it to the correct reserved attribute.

For a comprehensive list of validation error codes and resolutions, see Validation errors and warnings on Microsoft Learn.
