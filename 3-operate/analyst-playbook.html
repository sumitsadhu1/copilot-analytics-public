1. Prerequisites

Before you can create Copilot analytics queries and connect Power BI templates, the following must be in place:

PrerequisiteDetails
Viva Insights licenceMinimum 50 licences in the tenant for Advanced Analytics. Included with M365 Copilot licences — no separate purchase required. (The Copilot Dashboard itself is available with 1+ Copilot licences.)
Insights Analyst roleMust be assigned to your account by an Insights Admin or Global Admin via the Microsoft 365 admin centre.
Partition accessAn Insights Admin must grant you access to the relevant partition(s). Without partition access, you cannot run queries.
Org data uploadedOrganisational data (department, agency, location, etc.) must be uploaded for segmentation and group-by analysis.
Power BI DesktopJune 2022 or later. Required to open .pbit template files downloaded from Viva Insights.
Insights Admin ≠ Insights Analyst

These are different roles with different permissions. The Insights Admin configures the system (uploads org data, creates partitions, manages access). The Insights Analyst runs queries and builds reports. You need the Analyst role to follow this playbook. If you have the Admin role but not the Analyst role, ask your Global Admin to assign both.

Data Requirements

Timing matters. Plan your data collection window before Copilot rolls out if you need before/after analysis:

Data RequirementMinimumNotes
Org data uploadRequiredEnables segmentation by department, agency, location, job function, and other attributes.
Pre-adoption baseline4 weeksNeeded for before/after comparison in the Copilot Impact template. Must be collected before Copilot licence assignment.
Post-adoption data4 weeksMinimum period after Copilot rollout for meaningful usage patterns to emerge.
Plan Ahead for Before/After Analysis

If Copilot has already been rolled out across your tenant, before/after comparison may not be possible. The Impact template requires pre-adoption collaboration data as a baseline. Plan your analytics strategy before licence assignment to ensure you capture baseline data.

2. Available Copilot PBI Templates

Microsoft provides several Power BI templates purpose-built for Copilot analytics. Each template creates a pre-built query with predefined metrics and a corresponding .pbit file for Power BI Desktop.

TemplatePurposeKey Metrics
M365 Copilot Impact Primary ROI and productivity analysis Assisted hours, assisted value, before/after behavioural comparison, Copilot vs non-Copilot user comparison
M365 Copilot Adoption Usage trends and feature adoption depth Active users over time, per-app usage, feature-level adoption, usage intensity trends
Copilot Headlines (preview) Auto-scanned executive insights Power users, adoption benchmarks, auto-generated insight summaries
Copilot Business Impact Correlate Copilot usage with business outcomes Business KPI correlation (requires business outcome data upload via org data)
Copilot Studio Agents Agent usage and impact analysis Agent session outcomes, credits consumed, user engagement with custom agents
Copilot Employee Experience (Glint) Correlate Copilot usage with employee sentiment Usage patterns mapped to Glint survey responses (requires Viva Glint licence and integration)
Where to Find These Templates

The official Copilot templates are not distributed as standalone download links. You access them in-product: in the Viva Insights analyst experience (analysis.insights.viva.office.com) select Create analysis, then pick a template from the Copilot section and select Set up analysis. The .pbit file is generated from the query results page after the query runs (see Section 4).

For the full catalogue and per-report setup guides, see the Microsoft Learn template catalogue. Not all templates may be visible in your tenant — availability depends on your licence entitlements, feature rollout status, and whether prerequisites (e.g., Glint licence, business outcome data upload) are met.

3. Step-by-Step: Create a Report

This walkthrough uses the M365 Copilot Impact template as the example, but the process is similar for all templates.

Step 1 — Sign In to the Analyst Experience

Navigate to analysis.insights.viva.office.com and sign in with your organisational account that has the Insights Analyst role.

Step 2 — Start a New Analysis

Select Create analysis → Templates → Copilot → choose the desired template (e.g., M365 Copilot Impact) → Set up analysis.

Step 3 — Configure Query Settings

Set the following:

  • Query name — use a descriptive name (e.g., “Copilot Impact Q1 2026 — All Agencies”)
  • Time period — select the analysis window (up to 13 months of historical data)
  • Auto-refresh — enable if you want the query to re-run automatically on a schedule
  • Description — optional but recommended for team documentation

Step 4 — Review Predefined Metrics

The template pre-selects the required metrics (shown as grey tags — these cannot be removed). You can optionally add extra metrics to include in the query output.

Step 5 — Set Employee Filters

Apply filters to scope the query population:

  • “Is Active” — always recommended to exclude departed employees
  • Org attributes — filter by department, agency, location, or any uploaded org data attribute as needed

Step 6 — Add Organisational Attributes

Select up to 20 organisational attributes to include in the query output. These attributes become available as group-by and filter dimensions in Power BI. Include any attribute you may want to slice by later — you cannot add them after the query runs.

Step 7 — Run the Query

Select Run. Depending on population size and time window, the query may take minutes to hours to complete. You will receive a notification when results are ready.

Data Quality Warnings

During query setup, the system may flag data quality issues:

  • Yellow warning — low quality or partial coverage for an attribute. The query will still run.
  • Red warning — missing attribute entirely. The query will run but that attribute will not be usable for segmentation.

Address data quality issues by working with your Insights Admin to update the org data upload.

4. Connect Power BI

Once your query completes, connect it to the Power BI template:

Step 1 — Download the Template

From the query results page in Viva Insights, download the .pbit template file.

Step 2 — Open in Power BI Desktop

Open the downloaded .pbit file in Power BI Desktop. When prompted, paste:

  • Partition ID — found in the Viva Insights analyst experience
  • Query ID — found on the query results page

Step 3 — Set Minimum Group Size

Set the minimum group size for privacy thresholds (e.g., 10). Groups smaller than this threshold will be suppressed in the report. Click Load.

Step 4 — Sign In

When prompted for credentials, select Organisational account and sign in with the same account used to run the query.

Step 5 — Configure Report Settings

After data loads, configure the report settings on the Settings page:

  • Time period — define the reporting window within the query’s date range
  • Group-by attribute — select the primary organisational attribute for segmentation (e.g., Department, Agency)
  • Active user definition — set the threshold for what constitutes an “active” Copilot user
  • Language — set report language

Explore the Report Pages

The Copilot Impact template includes several pre-built pages:

  • Summary — high-level overview of Copilot impact across the organisation
  • Before vs After — compares collaboration patterns before and after Copilot adoption
  • Copilot vs Non-Copilot — compares metrics between users with and without Copilot licences
  • Two-Group Comparison — compare two organisational segments side by side
  • Research Insights — deeper statistical analysis and trend exploration

5. Programmatic Reporting Options

Choose the reporting path based on the question, population, privacy model, and automation requirement. These options are not interchangeable: a dashboard export is not an Advanced query, Microsoft Graph does not cover unlicensed Copilot Chat usage, and audit events are not equivalent to adoption metrics.

OptionBest forPopulation and scopeKey limits
Copilot Dashboard export (preview)Offline adoption analysis, custom communications, and short- or medium-term trend reportingDe-identified user-level rows for licensed Microsoft 365 Copilot and unlicensed Copilot Chat usage; available only to users with global Dashboard accessAt least 50 Copilot or Viva Insights licences assigned; fixed, non-customisable dataset; no Copilot assisted hours; secure handling required
Advanced Reporting queryCustom metrics, filters, org attributes, cohort analysis, and Power BI templatesMeasured employees and rows available to the Insights Analyst's assigned partitionInsights Analyst role and partition access required; query design, privacy thresholds, data quality, and refresh ownership remain operational responsibilities
Microsoft Graph Copilot report API (v1.0)Scheduled licensed-user usage reporting and integration into governed data pipelinesOnly users with a Microsoft 365 Copilot licenceReports.Read.All; no unlicensed Copilot Chat usage; no per-user prompt counts; Global service only
Admin report and Purview audit sourcesUnlicensed Copilot Chat reporting, compliance investigation, and event-level automationMicrosoft 365 admin center Copilot Chat Usage report; Copilot/AI audit events subject to assigned audit permissionsAudit event semantics, retention, latency, privacy, and aggregation differ from adoption reports

Dashboard export: fixed user-level dataset

The Copilot Dashboard's Export data feature is public preview. Week-level export provides the last six months, refreshes weekly, and can be up to six days old. Day-level export provides the last 28 days, refreshes daily, and can be up to three days old. Both remove personal identifiers and replace them with anonymized IDs; neither can be customized. The feature currently excludes Copilot assisted hours.

Global access carries a wider data-handling responsibility

Eligible exporters include senior leaders with global access, users manually assigned global access, delegates of globally scoped users, Insights Analysts with global access, and Microsoft 365 Global Administrators. Managers and their delegates with team-level views cannot export. Treat every download as a governed user-level dataset even though direct identifiers are removed.

Advanced queries: customizable and partition-scoped

Use Advanced Reporting when you need selected metrics, employee filters, organizational attributes, a defined privacy threshold, or a reusable Power BI model. Microsoft documents more than 100 Copilot metrics. Unlike the fixed Dashboard export, the available query population is constrained by the analyst's assigned partition. For Copilot metrics, design within the documented 13-month history and account for data freshness.

Microsoft Graph v1.0: licensed users only

The current Copilot report root exposes both a user-count summary and user detail in v1.0:

Unlicensed Copilot Chat: admin report or audit

Unlicensed Copilot Chat usage is not returned by the Graph Copilot reports API. Use the Microsoft 365 admin center Copilot Chat Usage report for the current operational report. For programmatic event extraction, Microsoft documents Microsoft Purview Audit, Search-UnifiedAuditLog in PowerShell, and the Office 365 Management Activity API.

Audit is an alternative source, not a metric-compatible substitute

Audit data supports compliance and event analysis. Do not present event counts as if they were the admin report's active-user or adoption measures. Define event types, deduplication, retention, permissions, latency, identity handling, and aggregation before publishing a trend.

Architecture decision

  1. Use the Dashboard export for a governed, fixed adoption dataset that includes current unlicensed Chat and usage-intensity measures.
  2. Use Advanced queries when partition scope, custom metrics, filters, organizational attributes, or Power BI templates are required.
  3. Use Graph v1.0 for scheduled licensed-user usage integrations with Reports.Read.All.
  4. Use the admin report for current unlicensed Chat reporting; use Purview/Management Activity only when event-level programmatic extraction is the actual requirement.
  5. Persist only the minimum required data, apply access and retention controls, record source/period/refresh time, and validate totals against the source experience.

6. Business Impact and Value Interpretation

6.1 Copilot Business Impact setup

The Copilot Business Impact report relates Copilot usage patterns to business outcomes supplied by your organization. It does not create the outcome data and does not establish that Copilot caused an observed difference.

PrerequisiteCurrent requirement
Roles and softwareInsights Analyst role; Power BI Desktop June 2022 or newer
Measured populationEmployees in scope have both Viva Insights and Microsoft 365 Copilot licences
Comparison sizeMicrosoft recommends at least 65 Copilot and 65 non-Copilot users
Outcome dataUpload through a manual CSV or Azure blob import; include one row per user for every month in the query period
Time alignmentOutcome months must align with the query; Copilot metrics have up to 13 months of history and can be up to six days behind
Query designSelect up to five business outcome metrics; fixed monthly grouping; retain the predefined Is Active filter; add up to 20 organizational attributes
  1. Agree the outcome owner, unit, direction of favorable performance, population, baseline period, and expected lag before loading data.
  2. Upload business outcome data through the Insights Administrator's approved CSV or Azure path and verify one user-month row for each queried month.
  3. In Create analysis, select Copilot Business Impact report and Set up analysis.
  4. Choose a time period that overlaps complete outcome data, then select up to five outcome metrics.
  5. Keep Is Active, add only defensible filters and attributes, review data-quality warnings, and run the query.
  6. In report settings, document the favorable-outcome threshold and any changes to low, medium, and high Copilot usage categories.
  7. Review results with the outcome owner and a statistical or evaluation specialist before publishing a business claim.
Relationship is not causation

Microsoft states that group differences in this report have not been evaluated for statistical significance and that Copilot is unlikely to be the only variable involved. Role mix, prior performance, self-selection, rollout targeting, seasonality, business conditions, and the time needed for an outcome to change can all affect the result. Describe findings as associations or observed differences, publish sample sizes and definitions, and do not claim ROI or causal impact without an appropriate evaluation design.

6.2 Agent Assisted Hours and Value

Copilot assisted hours and Agent Assisted Hours are modeled estimates. They use different activity inputs and assumptions; neither is a timesheet, realized saving, budget reduction, or proof of business value.

MeasureModel and assumptionsHow to use it
Microsoft 365 Copilot assisted hours/valueActions are multiplied by research-derived time-savings factors. Value multiplies estimated hours by an average hourly rate, set to US$72 by default and adjustable in the calculator. Microsoft states the methodology will evolve.Scenario estimate for adoption planning; disclose the rate, multipliers, period, diagnostics coverage, and sensitivity range.
Maker-led Agent Assisted ImpactInputs are supplied from Copilot Studio and displayed in Viva Insights; maker-led values are not editable in Viva Insights.Validate the maker's baseline, calculation, unit, and evidence outside the report before using the figure.
Inferred conversational-agent AAH(knowledge-source references + weighted sessions without knowledge sources) × time-savings multiplier. Resolved sessions receive full weight; escalated and abandoned sessions are weighted at 0.7. The default information-retrieval multiplier is six minutes and can be changed per agent.Compare scenarios for a specific agent; report the session outcomes and selected multiplier with the result.
Inferred autonomous-agent AAHCombines time estimated for knowledge retrieval, actions, and successful sessions without independent actions. Action-specific and generic multipliers are used where applicable.Review action-level assumptions and avoid comparing unlike agent types without normalization.
Agent Assisted ValueAgent Assisted Hours multiplied by an average hourly rate, US$72 by default. The calculator can change the rate for a specific agent; it cannot modify maker-led inputs.Present a low/base/high sensitivity range and keep cost, avoided cost, capacity released, and realized cash savings separate.

Before publishing value, pair the estimate with observed outcomes: volume, resolution, escalation, abandonment, satisfaction, quality, cycle time, and an organization-owned business KPI. A higher modeled value alongside poor resolution or satisfaction is a prompt to inspect assumptions and agent design, not a success conclusion.

7. Publish & Share Reports

Once your report is built (Sections 1–4), you have two delivery surfaces for getting it in front of leaders. They are not interchangeable — they differ in where recipients view the report, what access it grants them, and how well they respect agency boundaries. Pick the surface that matches your distribution model, then follow the matching sub-section below.

 7.1 Publish via Viva Insights7.2 Share via Power BI
Where recipients viewViva Insights web app → Reports tabPower BI service (app.powerbi.com) or a Power BI app
Best forFast, native distribution to leaders who already have Copilot Dashboard accessControlled, named-audience distribution — especially multi-agency
Viewer licenceViva Insights service plan (included with Copilot)Power BI Pro / PPU, or Premium capacity
Respects agency isolation⚠ Risk — see the 5.1 warning✅ Yes — per-workspace, optionally RLS
Implementing the Isolation model? Use 7.2, not 7.1

If you are delivering reports under the Multi-Agency Isolation Architecture, distribute to agency heads via Power BI (Section 7.2). Do not use Viva Insights Publish (7.1) to reach agency leaders: enabling a manager so they can receive a published report grants them Copilot Dashboard access, including the “Your company” scope that shows aggregated data across all agencies — which breaks isolation. Reserve 7.1 for Central IT analysts who already have tenant-wide dashboard access. See the Isolation playbook, Phase 4b.

7.1 Publish via Viva Insights

You can share query results with business leaders directly from Viva Insights using the Publish feature. This is the fastest route for leaders who already have Copilot Dashboard access.

Step 1 — Select Publish

In the Viva Insights analyst experience, navigate to your completed query and select Publish.

Step 2 — Set Report Metadata

Enter:

  • Title — the report title that recipients will see
  • Publisher name — your name or team name
  • Description — brief summary of what the report covers

Step 3 — Choose Scope

Select one of:

  • Unfiltered original — publishes the full dataset. All recipients see the same data.
  • Filter by manager hierarchy — each recipient sees only data for their reporting line.

Step 4 — Set Expiry Date

Set when the published report expires. Maximum is 365 days from publication.

Step 5 — Publish

Confirm and publish. Recipients receive an email notification with a link to view the report.

No Auto-Notification Before Expiry

Published reports expire silently — there is no automatic notification sent before a report expires. Set a calendar reminder to re-publish before the expiry date to avoid gaps in leader access.

Multi-Agency Tenants: Manager Hierarchy Risk

Enabling a manager so they can receive a published report also grants them Copilot Dashboard access — including the “Your company” scope, which shows aggregated data across all agencies, not just their own reporting line. In a multi-agency tenant this breaks agency isolation. For agency-head delivery, use Power BI sharing (Section 7.2) instead, and reserve Viva Insights Publish for Central IT who already have tenant-wide dashboard access. See the Multi-Agency Setup Guide and the Isolation playbook (Phase 4b).

7.2 Share via Power BI

Unlike Publish via Viva Insights (Section 7.1), which distributes results natively inside Viva Insights, this method uses Power BI’s own sharing. It is the preferred method for multi-agency tenants because it delivers a report to named people without granting them any Copilot Dashboard access — whereas 7.1’s “Filter by manager hierarchy” option can (see its warning).

New to Power BI? Start Here

This section assumes an Insights Analyst has already built the report in Sections 1–4 (run a query, connect the template, save the file). If you are an admin handling only distribution, you can start at Step 1 below with a report file the analyst hands you.

Three Power BI terms used in this section:

  • .pbix — the finished report file. When the analyst loads data into the .pbit template (Section 4) and saves it, it becomes a .pbix. That is the file you publish.
  • Workspace — a shared container in the Power BI service that holds reports and controls who can see them.
  • Dataset — the data behind a report; publishing a .pbix uploads the report and its dataset together.

Step 1 — Publish the .pbix to the Power BI Service

In Power BI Desktop, select Publish and choose a destination workspace (not “My workspace”, which can’t be shared with others). The report and its dataset upload to app.powerbi.com.

Step 2 — Use a Dedicated Workspace per Audience

Create one workspace per agency or audience (e.g., Agency A — Copilot Analytics). Keeping each agency’s reports in a separate workspace is the cleanest way to control who sees what. Store all .pbix files here rather than on individual machines so reporting survives staff changes.

Step 3 — Assign Stakeholders via Workspace Roles

In app.powerbi.com, open the workspace → select Manage access → Add people or groups, then assign a workspace role from the table below. Prefer adding an Entra security group over individuals, so access is managed centrally and consistently with your other governance controls.

RoleCan doAssign to
ViewerView and interact with reports onlyMost stakeholders (agency leaders)
ContributorCreate/edit content; can’t manage accessCo-analysts who build reports
MemberEdit content and add others (below Admin)Trusted leads who help run the workspace
AdminFull control, including deleting the workspaceThe owning analyst / Central IT
Cleaner Distribution for Wide Audiences

For a large, read-only audience, publish the workspace as a Power BI app — a packaged, read-only view of selected reports (not a desktop or mobile application). Give the audience access to the app rather than the workspace, so recipients get a curated view without seeing the underlying workspace contents.

Step 4 — Confirm Viewer Licensing

Power BI sharing is licensed separately from Viva Insights and Copilot:

  • Each viewer needs a Power BI Pro or Premium Per User (PPU) licence; or
  • if the workspace sits on a Power BI Premium capacity (P or F SKU), viewers with a free Power BI licence can view the reports — removing per-viewer costs. Confirm with your licensing owner.
Multi-Agency: Keep Boundaries Intact

Grant each agency workspace’s Viewer role only to that agency’s leaders — via an agency-scoped Entra group. If multiple agencies must share one workspace, add Row-Level Security (RLS) as a secondary control. The full agency-isolation delivery procedure (including the Publish Reports pitfall) is in the Isolation Architecture playbook, Phase 4b.

8. Ongoing Maintenance

Keep your Copilot analytics current with these recurring tasks:

  • Refresh org data monthly — work with your Insights Admin to ensure organisational data (new hires, transfers, departures) stays up to date. Stale org data leads to inaccurate segmentation.
  • Re-run queries periodically — either enable auto-refresh during query setup or manually re-run queries on a monthly or quarterly cadence to capture current trends.
  • Update PBI reports — when Microsoft releases updated template versions, download the new .pbit and reconnect to benefit from new metrics and visualisations.
  • Share via Publish Reports or Power BI workspaces — for ongoing distribution, either re-publish through Viva Insights or save .pbix files to a shared Power BI workspace (see Section 7.2 for the step-by-step).

9. Analyst Departure — Transfer Guide

When an Insights Analyst leaves the organisation or changes roles, follow this checklist to ensure continuity:

ItemWhat HappensAction Required
Completed query results Persist in Viva Insights — a new analyst with partition access can view them No action required (results are not tied to the analyst’s account)
Scheduled / recurring queries May stop running when the account is disabled or deleted Recreate scheduled queries under the new analyst’s account
Published reports Remain accessible until their expiry date, but cannot be renewed by a different analyst Re-publish reports under the new analyst’s account before the originals expire
Power BI files (.pbix) Stored locally on the analyst’s machine — lost if not transferred Transfer all .pbix files to a shared Power BI workspace before account deletion
Partition access Role assignment is per-user — does not transfer automatically Insights Admin must assign partition access to the replacement analyst
Redundancy Is Essential

Always have at least two analysts with partition access. Store all .pbix files in a shared Power BI workspace rather than on individual machines. This prevents data loss and reporting gaps during transitions.
