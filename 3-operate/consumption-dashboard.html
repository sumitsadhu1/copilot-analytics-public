1. Purpose and Current Coverage

The Consumption Dashboard is one of the six current Copilot Analytics areas documented by Microsoft. It is intended for operational monitoring: identify active users, review credit usage over time, see which services and employee groups drive usage, understand whether consumption is concentrated among a small number of users, and find spending policies whose users are near their limits.

QuestionUse the dashboard forDo not infer
Who is using covered services?Active users and sessions for the selected month-to-date scopeAll Copilot or agent activity across every Microsoft product
Where are credits being consumed?Trends and breakdowns by covered service, group, organization, or job functionInvoice-level allocation or final chargeback
Is usage concentrated?Usage-intensity segments and per-user/per-session averagesWhether high usage is productive, wasteful, or anomalous without business context
Which policies need attention?Users and policies near or at displayed limitsThat the dashboard configures, changes, or guarantees enforcement of a spending policy
Current Microsoft 365 services coverage is narrow

As documented on 1 July 2026, consumption data on the Microsoft 365 services page is currently available only for Copilot Cowork and the Work IQ API. Its active-user and session cards use activity from those services. Absence of other Copilot, Copilot Studio, or agent usage is not evidence that no such activity occurred.

The dashboard introduction also describes visibility for Copilot Credits and GitHub AI Credits. Keep the data boundary explicit: Insights Analyst queries with consumption metrics exclude GitHub data, and official GitHub AI credit charges remain in GitHub.com.

No delegation

Consumption Dashboard access cannot currently be delegated. Delegation for the Microsoft Copilot Dashboard or Agent Dashboard does not grant access here. Assign the least-privileged supported role and scope instead.

An Insights Administrator manages analyst access by adding or removing the analyst from the global partition. Removing that access also removes the analyst from every other experience in that global partition. Managers are controlled through Manager settings; consumption insights are enabled for all managers by default unless the setting is changed.

4. Operating the Dashboard

Step 1 — Open Insights and confirm scope

Open the Viva Insights web app and select the Consumption Dashboard. Confirm the scope before interpreting any value:

  • Company is available only to global-partition Insights Analysts and Microsoft 365 Global Administrators.
  • Group limits the view to the manager's group.
  • Organization and Job function use the same employee-attribute sources as the Copilot Dashboard.
  • Time period is currently month to date.

Organization can be derived from Microsoft Entra ID or uploaded organizational data. Job function depends on the corresponding uploaded organizational attribute; a missing field can produce an empty or incomplete filter.

Step 2 — Read the summary cards

MetricOperational meaningValidation question
Active usersUnique users who took at least one action in Copilot Cowork or the Work IQ APIDid covered activity occur in this scope during the current month?
Total Copilot credit usageCredits consumed by covered activity in the selected scopeDoes the direction align with the official billing view, allowing for scope and timing?
Total session countCopilot Cowork or Work IQ API sessions initiated by usersDid a rollout, campaign, integration, or automation change session volume?

Step 3 — Analyze trends and service drivers

Use the trend controls to change the plotted metric, break it down by group, organization, or job function, and choose the available date granularity. Then use the service breakdown to identify which covered service and employee segment account for the largest share. Treat this as a prioritization signal, then validate against service ownership and business context.

Step 4 — Review usage intensity

Usage intensity shows whether consumption is broadly distributed or concentrated. Review these segments:

  • All users
  • Top 1 percent of users
  • Top 5 percent of users
  • Top 6–25 percent of users
  • Top 26–50 percent of users

For each segment, compare total credits, average credits per user, average credits per session, average sessions per user, and active users. A concentrated distribution is a prompt to investigate workload design, automation, value, and policy fit; it is not by itself evidence of misuse.

Step 5 — Review spending-policy limits

Use the spending-policy table to identify the top policies in use, average credits used per user, the displayed spending limit, and the number of users near or at that limit. Contact the policy owner before users are depleted. The dashboard reports the condition; policy configuration and capacity enforcement occur in the relevant administration and billing surfaces.

Capacity has contractual and service consequences

Copilot Studio documentation states that purchased capacity is enforced monthly, unused Copilot Credits do not carry over, and exceeding purchased capacity can lead to technical enforcement and service denial. Confirm the applicable licensing model, included usage, zero-rated scenarios, and current terms in the official licensing and Copilot Credit guides before taking purchasing or enforcement action.

5. Export and Reconciliation

The Consumption Dashboard export is a public-preview feature for de-identified, user-level AI cost metrics. It is currently available to Insights Analysts with global partition access and Microsoft 365 Global Administrators. Manager export is not yet generally available in the 1 July 2026 source. The export does not include GitHub metrics.

ExportWindow and freshnessUse
Export by weekComplete data for the previous five months plus month-to-date data for the current month; activity is usually 2–8 days behindWeekly and monthly trend review or longer-term planning
Export by dayStart of the previous month through the export date; activity is usually up to two days behindDay-to-day cost review and emerging usage trends

Only complete days or weeks are included. Each download is a ZIP containing two CSV files joined by PeopleHistoricalId:

  • PersonServiceCreditsMetrics.csv — one row per person, service, and date, with hashed person ID, service, policy, sessions, limits, credits, licence status, and join key.
  • PeopleMetaData.csv — one row per person, with the join key plus Organization and Job function where available.
  1. At the top right of the dashboard, select the download button.
  2. Select Export by week or Export by day and allow the ZIP download to complete.
  3. Record the export type, timestamp, complete-period boundary, service coverage, role, and source URL.
  4. Keep both CSVs together and join only on PeopleHistoricalId. Treat the hashed identifier and HR attributes as protected data.
  5. Check that the services and population match the documented export fields; do not silently blend them with broader Copilot or GitHub activity.
  6. Reconcile Copilot Credit totals against the Microsoft 365 admin center. Reconcile GitHub AI credits separately in GitHub.com; they are not in this export. Document scope, timing, and coverage differences rather than forcing the figures to match.
Do not substitute the Copilot Dashboard export

The separate Copilot Dashboard export is a public-preview, de-identified user-level usage export with week-level and day-level options. It answers adoption questions; it is not the Consumption Dashboard export and is not an official billing record.

6. Success Criteria

Before operationalizing a recurring review, verify all of the following:

  • The expected role can open the dashboard and sees only the documented company or group scope.
  • A manager cannot switch to Company scope.
  • A non-global partition analyst cannot open the dashboard or run consumption-metric queries.
  • Covered Cowork or Work IQ API activity produces month-to-date users, sessions, and credits after processing.
  • Organization and job-function filters return the expected population and change the totals coherently.
  • Usage-intensity totals reconcile internally and do not exceed the selected population.
  • Displayed spending-policy limits are confirmed with the policy owner.
  • The export completes, carries the expected scope, and is stored under appropriate access controls.
  • The official billing surface remains the source of record, with any discrepancy documented.

7. Troubleshooting and Escalation

SymptomLikely causeAction
Dashboard is absentUnsupported role, analyst lacks global partition, manager disabled, or feature not available in the tenant/cloudVerify the access table, global partition membership, Manager settings, tenant UI, and Message Center.
Company scope is absentUser is a managerExpected behavior. Managers receive group scope only; do not attempt delegation.
Values are zero or unexpectedly lowNo current-month Cowork/Work IQ API activity, processing delay, or filters exclude usersClear filters, confirm covered service activity and date, then compare with the source service. Do not use unrelated Copilot activity as the test.
Organization or job function is blankMissing, stale, or low-quality organizational attributesValidate Entra and uploaded org data, then allow the documented processing cycle before retesting.
Service-by-group matrix is absentUser is not a manager of managers or the group is below the required sizeVerify hierarchy and group size; use supported summary views instead.
Consumption query is unavailableUser is not a global-partition Insights AnalystUse the dashboard, or have the Insights Administrator assess least-privileged global partition access. GitHub data remains excluded.
Dashboard and invoice differReference-only dashboard, different scope/coverage/timing, or billing adjustmentsUse the official billing surface as authoritative and document the discrepancy.
Export is missing or failsFeature availability, role, browser download, or transient service issueReconfirm eligibility, retry once, test another supported browser/network, and capture the error before escalation.

Escalation evidence

Before escalating, capture:

  • Tenant and cloud environment, user role, and whether the analyst has global partition access
  • Manager setting and expected group size, where relevant
  • Dashboard URL, selected scope and filters, month-to-date boundary, and covered service
  • Timestamp, screenshot, exact error text, and export behavior
  • Expected Cowork or Work IQ API activity and the official billing comparison
  • Any relevant Message Center or Service Health advisory

Route access, partition, manager-setting, and org-data issues to the Insights Administrator; role and tenant-policy issues to the Microsoft 365 Global Administrator; billing differences to the billing or GitHub administrator; and unresolved feature, cloud, or data-processing issues to Microsoft Support.
