1. Copilot Dashboard Pages

The Copilot Dashboard is accessible at insights.cloud.microsoft and is divided into three main pages: Readiness, Adoption, and Impact (which includes a Sentiment section). Each page answers a different question about your organisation’s Copilot deployment.

1.1 Readiness Page

The Readiness page answers: “How many licences have we assigned, and is the count growing?”

  • Licence assignment status — current count of assigned vs available M365 Copilot licences.
  • Trend chart — historical view of licence assignments over time, so you can spot stalled rollouts.
Feature Availability by Licence Count

Tenants with 1+ assigned Copilot licences get core Readiness, Adoption, and Impact capabilities. For capabilities using the 50-plan threshold, Microsoft’s Environment requirements for Viva Insights (Last updated 12 May 2026) explicitly counts the Viva Insights service plan included with Copilot and gives 30 Copilot + 20 Viva Insights = 50 qualifying service plans. The newer, Dashboard-specific Connect to the Microsoft Copilot Dashboard for Microsoft 365 customers (Last updated 1 July 2026) instead says 50 Copilot or 50 Viva Insights licences. Do not assume a mixed deployment fails: verify assigned service plans, feature presence, and processing in the tenant; escalate to Microsoft Support if the documented qualifying mix does not unlock the capability.

For Agent Dashboard (preview), conservatively require 50 assigned Microsoft 365 Copilot licences plus qualifying agent activity. Microsoft’s feature-specific Agent article conflicts with the broader Dashboard article on this prerequisite.

1.2 Adoption Page

The Adoption page answers: “Who is using Copilot, how often, and in which apps?”

Key Metrics

MetricDefinition
Active Copilot usersNumber of licensed users who had at least one Copilot interaction in the 28-day measurement window.
Copilot Actions TakenTotal count of Copilot interactions across all users. Excludes agent interactions — those are tracked on the Agent Dashboard.
Usage IntensityAverage number of actions per active user (Actions Taken ÷ Active Users). Measures depth of engagement beyond simple headcount.
Consistent UsageUsers who were active in both the current and the previous 28-day period. Shows sustained adoption rather than one-time trial.
Internal BenchmarkYour own historical trend line — compare current period to your past performance.
External BenchmarkAnonymised comparison against similar-size organisations. Helps answer “Are we ahead or behind?”

Per-App Breakdown

The Adoption page breaks down usage across individual M365 applications:

AppWhat It Tracks
TeamsMeeting summaries, chat Copilot interactions, Teams Copilot actions
OutlookEmail drafting, summarisation, coaching suggestions
WordDocument drafting, rewriting, summarisation
PowerPointSlide generation, design suggestions, narrative creation
ExcelFormula generation, data analysis, chart creation
Copilot ChatCurrent merged Copilot Chat usage metric across the supported Chat experience
Copilot Chat (web) — legacyHistorical transition series only; expect it to trend toward zero as activity moves into the merged Copilot Chat metric
License Filter: Microsoft 365 Copilot vs Copilot Chat

At the top of the Adoption page, a License filter lets you compare Microsoft 365 Copilot, Copilot Chat, and, where the 50-plan experience is available, All. The Copilot Chat view covers active users without the paid Microsoft 365 Copilot add-on when reporting is enabled.

Merged metric and legacy-web transition

Use Copilot Chat as the current reporting metric. Do not interpret a decline in the legacy Copilot Chat (web) series as declining adoption by itself; Microsoft is merging current activity into Copilot Chat, so the legacy series is expected to approach zero.

Source: Connect to the Microsoft Copilot Dashboard for Microsoft 365 customers (Last updated 1 July 2026), Adoption section, and Export Copilot metrics from the Microsoft Copilot Dashboard (preview) (Last updated 15 July 2026), Metrics section.

Current admin reporting for unlicensed Copilot Chat

The Microsoft 365 admin center has a dedicated Microsoft 365 Copilot Chat usage report for users without a Microsoft 365 Copilot licence, with user activity and CSV export. This is separate from the paid-Copilot usage report, the Viva Dashboard view, Microsoft Graph’s licensed-user-only Copilot usage endpoint, and Purview audit evidence.

Source: Microsoft 365 Copilot Chat usage report (Last updated 10 July 2026) and Microsoft 365 Copilot reporting options for admins (Last updated 16 September 2025).

“View by: Group” Shows Only One Person?

If a leader selects “View by: Group” and sees only themselves (or far fewer people than expected), the most common cause is missing intermediate managers in the org data. The manager hierarchy chain must include a row for every manager between the leader and their reports. Verify that your CSV (or Entra Manager field) has rows for every intermediate manager. Gaps break the chain and orphan the downstream employees from the leader’s view.

Sources: Roles in Viva Insights (Last updated 7 July 2026), Delegate access to the Copilot Dashboard, Agent Dashboard, and Copilot insights (Last updated 9 February 2026), and Configure manager settings (Last updated 25 June 2026).

Suppressed Values (“--”)

When you see “--” instead of a number, it means the group size is below the minimum group size threshold (default: 10). This is a privacy protection, not an error. Data is present but suppressed to prevent identification of individuals. If you need to see data for smaller teams, an Insights Admin can adjust the minimum group size in privacy settings — but this should be done carefully with HR and legal approval.

Agent-Related Insight Cards on the Adoption Page (Preview)

In addition to the core Adoption insights (overview, usage intensity, returning users, per-app breakdown), the Adoption page now surfaces two agent-focused insight cards when your tenant has agent activity. These are independent of the Agent Dashboard itself and are visible to anyone who can already view the Adoption page.

CardWhat It ShowsRequires
Agent adoption by Copilot users (preview) The number of Copilot users who have used at least one agent, the number of agents with at least one active Copilot user, and the top three agents sorted by active users (with change-over-time and agent-response counts). Lets you jump directly into the agent deep-dive page from the Adoption page. At least one active agent user in the tenant. No PAYG required.
Credit usage overview Copilot Credits consumed by Copilot Chat users, the number of agents those credits were used across, and the number of Copilot Chat users driving credit consumption. Also breaks out the top groups driving credit usage (by group / organisation / job function) with 28-day change. Your tenant must have Pay-As-You-Go enabled for Copilot Studio / Copilot Chat. Without PAYG the card does not appear.
Why these cards matter even before you open the Agent Dashboard

Both cards live on the regular Adoption page, so leaders who never opened the Agent Dashboard can still see whether agents are being used and whether they cost money. The Agent adoption by Copilot users card is the easiest way for a CIO to answer “are people actually trying our agents yet?”. The Credit usage overview card is the first place to spot a runaway PAYG bill before it shows up in the billing report. Click into either card to jump to the corresponding deep-dive view in the Agent Dashboard. Both cards are tagged as preview features — UI may shift before GA.

1.3 Impact Page

The Impact page answers: “What value is Copilot delivering?”

Check diagnostic coverage before interpreting impact

Optional diagnostic data affects Copilot assisted hours, emails sent using Copilot, Teams meetings summarized or recapped, meeting hours summarized or recapped, and Teams chats summarized. Required diagnostic data still provides a subset, so the metrics may populate while remaining materially undercounted. Record the tenant’s optional diagnostic-data policy and keep it stable across comparison periods.

Source: Connect to the Microsoft Copilot Dashboard for Microsoft 365 customers (Last updated 1 July 2026), Impact section.

Top-Level Metrics

MetricDefinitionHow It’s Calculated
Copilot Actions TakenTotal Copilot interactions in the periodDirect count of all Copilot actions (same as Adoption page)
Copilot Assisted HoursEstimated hours saved by CopilotMeeting summaries + search/summaries valued at 6 min each + content creation valued at 6 min each
Copilot Assisted ValueDollar value of time savedAssisted Hours × hourly rate (default $72/hr). Hourly rate is session-only — changes are not saved between visits.
Satisfaction RateUser satisfaction with Copilot responsesThumbs up ÷ (thumbs up + thumbs down) reactions, 28-day rolling window. No survey required — based on in-app feedback.
Estimate, not causal proof

Assisted Hours and Assisted Value are modeled estimates using Microsoft’s assumptions and the hourly rate entered for the current session. They are not realized cash savings and do not establish that Copilot caused an outcome. For decision-grade analysis, document the assumptions, use a stable baseline and diagnostic-data policy, compare suitable cohorts or periods, and pair usage measures with independently governed business outcome data.

Current documented metric issue and incident validation

The current controlling Dashboard article documents the email issue below. A previously documented meeting issue is not present in that article as of 1 July 2026, so do not carry that older qualifier into a current report without a dated source.

Metric CategoryAffected PeriodDetails

Do not automatically attribute a dip to this issue. Check Microsoft 365 admin center Service Health for current incidents and Message Center for rollout or metric-change notices, then compare the incident dates, cloud, apps, and affected metrics with the observed pattern.

Export Copilot Metrics Safely (Preview)

The Copilot metrics export is a preview feature with a fixed, non-customizable schema. It exports anonymized user-level metric rows, not the aggregated chart currently visible on screen.

Prerequisites

  • The tenant must meet the documented 50-licence threshold. Because Microsoft pages conflict on mixed service-plan counting, verify assigned service plans and export availability in the tenant.
  • The exporter must have global / Your company access. A manager or “Your group” delegate cannot export.
  • The Copilot Metrics Export feature must be enabled for the user, and the capability must be available in the tenant’s cloud. It is not available in GCC.

How to Export

  1. Open the Copilot Dashboard in the Viva Insights web app.
  2. Select the Copilot metrics Export control.
  3. Choose Week for weekly rows covering the past six months or Day for daily rows covering the past 28 days.
  4. Download the CSV and record the export date, granularity, source article date, and tenant/cloud used.

What’s in the Export

PropertyCorrect interpretation
Grain and periodOne anonymized user-level record per time interval: up to six months for Week, or 28 days for Day.
IdentityDe-identified user identifiers are included. The export is therefore anonymized user-level data, not aggregate-only data.
CustomizationThe schema and metric selection are not customizable, and Dashboard filters/groupings do not redefine the export.
MetricsThe documented export metrics are provided in the fixed schema. Copilot Assisted Hours is not included.

Privacy Handling

  • Treat the CSV as sensitive analytics data even though direct identities are removed; repeated user-level rows and joins with organizational data can increase re-identification risk.
  • Store it in an access-controlled location, apply retention and sensitivity requirements, restrict downstream joins, and document every recipient and purpose.
  • Do not present row-level data or attempt re-identification. Aggregate under approved minimum-group and privacy rules before wider distribution.
  • Validate row counts, dates, granularity, metric definitions, missing values, and diagnostic-data coverage before publishing trends.
Export vs Advanced Analytics

The preview export provides a fixed anonymized user-level dataset. Use Advanced Reporting when an Insights Analyst needs governed metric selection, partition-scoped queries, or custom segmentation. Neither route removes the need for privacy review.

Source: Export Copilot metrics from the Microsoft Copilot Dashboard (preview) (Last updated 15 July 2026).

1.4 Sentiment Section

The Sentiment section lives within the Impact page and shows qualitative user feedback about Copilot.

ComponentWhat It ShowsNotes

Sentiment by groupHeat map showing sentiment across organisational groupsViva Insights customers only — requires Viva Insights licence
Manager team viewSentiment breakdown for a manager’s direct reportsSubject to minimum group size suppression
No Survey Data?

If no Copilot survey has been deployed, the Sentiment section will show “No survey responses available.” This is expected. The Satisfaction Rate metric (thumbs up/down) works independently of surveys and will still display on the Impact page regardless of whether surveys have been configured.

2. How to Delegate Access

Dashboard access can be shared with others through delegation. This is how leaders give their managers or analysts visibility into Copilot usage for their scope.

Two Methods to Add Delegates

MethodSteps
Method 1: SettingsDashboard → Settings (gear icon) → Delegation → Add → search for user → select scope → Save
Method 2: ShareDashboard → Share button (top right) → Add → search for user → select scope → Save

Delegation Rules

RuleDetail
Maximum delegates75 per delegator
No cascadingDelegation does not flow downward. If Alice delegates to Bob, Bob cannot re-delegate to Carol.
“Your group” scopeDelegate sees the delegator’s reporting chain only
“Your company” scopeDelegate sees the entire tenant — use sparingly
Agent Dashboard accessRequires “Your company” scope. “Your group” delegates cannot see the Agent Dashboard.
Delegator leaves orgAll delegates immediately lose access. Recovery: another leader must re-delegate to those users.
Plan for Delegator Departures

If a key delegator (e.g., a VP or CIO) leaves the organisation, all 75 delegates could lose dashboard access simultaneously. Document who has delegated to whom and ensure at least two senior leaders have overlapping delegation lists for continuity.

3. Copilot Analytics Reports

Copilot Analytics Reports are pre-built reports available at insights.cloud.microsoft → Reports. They provide deeper analysis than the dashboard pages without requiring Power BI setup.

Report Types

TypeExamplesDescription
Out-of-the-boxCopilot Studio Agents, Copilot for Sales AdoptionPre-built by Microsoft. Available automatically once prerequisites are met.
SharedCustom reports published by your analystsAnalysts with partition access can create queries and publish results as shared reports visible to others in the Reports section.

Access

  • Anyone with Dashboard access (direct or delegated) can view reports.
  • Analysts with a global partition can create and share reports.
  • Global Admins have full access.

No additional setup is required beyond what is already configured for the Copilot Dashboard.

5. Agent Dashboard (Preview)

The Agent Dashboard preview provides visibility into supported Copilot Studio and declarative-agent activity for eligible users with “Your company” scope. Managers and “Your group” delegates do not receive Agent Dashboard access.

Preview prerequisite conflict and cloud limits

Connect to the Agent Dashboard (preview) (Last updated 16 June 2026) and Copilot Analytics introduction (Last updated 9 February 2026) require at least 50 Microsoft 365 Copilot licences plus agent activity. The broader Dashboard article (Last updated 1 July 2026) also lists a 50-Viva-Insights route. Use 50 Copilot licences plus activity as the conservative prerequisite; validate a Viva-Insights-only tenant through its UI, Message Center, and Microsoft Support.

The Agent Dashboard is unavailable in GCC and other national or regional clouds. Preview behavior, metrics, filters, and data handling can change. Confirm current support in Supported Microsoft 365 environments for Viva Insights features (Last updated 30 June 2026) before using this procedure.

Three Practical Scenarios

Scenario 1: Post-Launch Adoption Review

Your organisation deployed three custom agents last month. Use the Agent Dashboard to answer:

  • How many users actually tried each agent? (Active users per agent)
  • Are people coming back? (Returning users)
  • Which agent is getting the most traction? (Most popular agent)

Action: Double down on training for the popular agent; investigate why low-adoption agents are being ignored (discoverability? relevance?).

Scenario 2: Cost Governance for Pay-As-You-Go

Your tenant uses PAYG billing for agent interactions. Use the Agent Dashboard to answer:

  • How many Copilot Credits are being consumed? (Credits metric)
  • Which agents are the biggest consumers? (Sort by responses)
  • Are partner-built agents driving unexpected costs? (Filter by creator type: Partner)

Action: Set credit alerts, review high-consumption agents with their owners, consider blocking non-essential agents.

Scenario 3: Shadow AI Detection

Users are creating their own agents without IT approval. Use the Agent Dashboard to answer:

  • How many user-created agents exist? (Filter by creator type: User)
  • Are any user-created agents being widely shared? (Most shared agent)
  • Do any of these agents have external data connections that violate policy?

Action: Follow the Detect → Act workflow below.

Detect → Act Workflow

StageToolAction
DetectAgent DashboardIdentify agents that are unapproved, high-cost, or widely shared without governance review
DecideGovernance processEvaluate the agent against your organisation’s AI governance policy (data access, compliance, purpose)
Act QuickM365 Admin CentreImmediate actions: Block (disable agent), Approve (whitelist), or Review (flag for governance team)
Act PolicyPower Platform Admin CentreLong-term controls: apply DLP policies to restrict connector access, enforce environment boundaries, set maker policies

6. Suggested Review Cadence

Operational validation before diagnosis

Before treating a missing page, changed metric, or sudden data dip as tenant misconfiguration, check Microsoft 365 admin center Service Health for incidents and Message Center for rollout, preview, or reporting changes. Record the notice ID and date alongside the report, then retest the affected surface after the published recovery or rollout window.
