1. Copilot Dashboard Pages

The Copilot Dashboard is accessible at insights.cloud.microsoft and is divided into three main pages: Readiness, Adoption, and Impact (which includes a Sentiment section). Each page answers a different question about your organisation’s Copilot deployment.

1.1 Readiness Page

The Readiness page answers: “How many licences have we assigned, and is the count growing?”

  • Licence assignment status — current count of assigned vs available M365 Copilot licences.
  • Trend chart — historical view of licence assignments over time, so you can spot stalled rollouts.
Available Below 50 Licences

The Readiness page is the only dashboard page that works even when your tenant has fewer than 50 Copilot licences assigned. The Adoption and Impact pages require ≥50 assigned licences to display data.

1.2 Adoption Page

The Adoption page answers: “Who is using Copilot, how often, and in which apps?”

Key Metrics

MetricDefinition
Active Copilot usersNumber of licensed users who had at least one Copilot interaction in the 28-day measurement window.
Copilot Actions TakenTotal count of Copilot interactions across all users. Excludes agent interactions — those are tracked on the Agent Dashboard.
Usage IntensityAverage number of actions per active user (Actions Taken ÷ Active Users). Measures depth of engagement beyond simple headcount.
Consistent UsageUsers who were active in both the current and the previous 28-day period. Shows sustained adoption rather than one-time trial.
Internal BenchmarkYour own historical trend line — compare current period to your past performance.
External BenchmarkAnonymised comparison against similar-size organisations. Helps answer “Are we ahead or behind?”

Per-App Breakdown

The Adoption page breaks down usage across individual M365 applications:

AppWhat It Tracks
TeamsMeeting summaries, chat Copilot interactions, Teams Copilot actions
OutlookEmail drafting, summarisation, coaching suggestions
WordDocument drafting, rewriting, summarisation
PowerPointSlide generation, design suggestions, narrative creation
ExcelFormula generation, data analysis, chart creation
Copilot Chat (work)Chat interactions grounded in tenant data (SharePoint, emails, calendar)
Copilot Chat (web)Chat interactions grounded in internet data only
License Filter: Microsoft 365 Copilot vs Copilot Chat

At the top of the Adoption page, a License filter lets you toggle between Microsoft 365 Copilot and Copilot Chat views. The Microsoft 365 Copilot filter shows insights for employees with a paid Copilot licence. The Copilot Chat filter shows insights for active users of the free, web-based Copilot Chat app — employees with an M365 licence but without the paid Copilot add-on. The Copilot Chat filter appears only if your tenant has Copilot Chat reporting enabled.

Copilot Chat (work) vs Copilot Chat (web)

Copilot Chat (work) uses your organisation’s tenant data — files in SharePoint, emails, calendar events, and Teams messages — to generate responses. Copilot Chat (web) searches the public internet for answers. The distinction matters because “work” mode reflects enterprise data utilisation, while “web” mode shows general AI assistance patterns.

Intent Patterns (Preview)

The Adoption page includes intent-based metrics that categorise individual Copilot Chat (work) prompts into common intent categories. These help you understand how people use Copilot, not just how much.

Intent CategoryWhat It CapturesExample Prompts
Ask and findQuestions and information retrieval“Find the recording of meeting X from May”, “Who is the project lead for initiative Y?”
Catch upSummaries, recaps, and action items“Recap this meeting with key takeaways”, “Summarise what happened while I was out”
Draft and brainstormContent generation and creative ideation“Draft a status update email about project X”, “Brainstorm creative presentation ideas”
OtherPrompts that don’t fit the above categoriesGeneral questions, casual interactions

Intent metrics are also available in Viva Insights advanced reporting for deeper analysis.

Expanded App Coverage

Beyond the core M365 apps listed above, the Dashboard also tracks Copilot usage in additional surfaces. Scenario-level breakdowns (e.g., suggested reply, translate, coach, clean data) are available for some apps:

App/SurfaceScenarios Tracked

Intent Metrics Are in Public Preview

Intent-based metrics are rolling out in public preview as of April 2026. Availability may vary by tenant. These metrics will also be available in Viva Insights custom person queries once generally available, enabling analysts to correlate intent patterns with organisational attributes.

“View by” Options

The Adoption page can be sliced by different organisational dimensions. Each requires specific data to be present:

“View by: Group” Shows Only One Person?

If a leader selects “View by: Group” and sees only themselves (or far fewer people than expected), the most common cause is missing intermediate managers in the org data. The manager hierarchy chain must include a row for every manager between the leader and their reports. Verify that your CSV (or Entra Manager field) has rows for every intermediate manager. Gaps break the chain and orphan the downstream employees from the leader’s view.

Scope Filter

ScopeWhat It ShowsWho Gets It
Your companyEntire tenant (all licensed Copilot users)Global Admin, Insights Admin, users with “Your company” delegation
<Leader Name>That leader’s reporting chain (direct + indirect reports)Leaders with enabled Copilot Dashboard access whose hierarchy is in org data
Your groupYour own direct and indirect reportsAny leader or manager with dashboard access and a manager hierarchy in org data

Suppressed Values (“--”)

When you see “--” instead of a number, it means the group size is below the minimum group size threshold (default: 10). This is a privacy protection, not an error. Data is present but suppressed to prevent identification of individuals. If you need to see data for smaller teams, an Insights Admin can adjust the minimum group size in privacy settings — but this should be done carefully with HR and legal approval.

1.3 Impact Page

The Impact page answers: “What value is Copilot delivering?”

Top-Level Metrics

MetricDefinitionHow It’s Calculated
Copilot Actions TakenTotal Copilot interactions in the periodDirect count of all Copilot actions (same as Adoption page)
Copilot Assisted HoursEstimated hours saved by CopilotMeeting summaries + search/summaries valued at 6 min each + content creation valued at 6 min each
Copilot Assisted ValueDollar value of time savedAssisted Hours × hourly rate (default $72/hr). Hourly rate is session-only — changes are not saved between visits.
Satisfaction RateUser satisfaction with Copilot responsesThumbs up ÷ (thumbs up + thumbs down) reactions, 28-day rolling window. No survey required — based on in-app feedback.
Known Metric Underreporting Periods

Microsoft disclosed two periods where certain metrics were underreported due to telemetry issues. Data from these windows may appear lower than actual usage.

Metric CategoryAffected PeriodDetails
Email (Outlook Copilot)June 1, 2025 – February 18, 2026Generate email draft and Summarise email thread metrics underreported. Fix applied from February 18, 2026; historical data restoration pending.
Meetings (Teams Copilot)September 6 – November 3, 2025Copilot assisted hours, assisted value, and meeting summary metrics underreported due to data loss. Fix applied from November 4, 2025; historical data cannot be corrected.

If you see dips in these periods, the underreporting issue — not reduced usage — is the most likely explanation.

Export Dashboard Data

You can export de-identified Copilot Dashboard data as a CSV file covering weekly metrics for the past six months. This enables offline analysis, board-ready reporting, or integration with other analytics tools.

How to Export

  1. Open the Copilot Dashboard in the Viva Insights web app.
  2. Navigate to the page containing the data you want to export (Adoption or Impact).
  3. Select the Export button (typically in the top-right area of the dashboard).
  4. The download starts automatically as a .csv file.

What’s in the Export

FieldDescription
Time periodWeekly data points covering the past 6 months
MetricsAdoption and impact metrics matching the dashboard view (active users, actions, assisted hours, etc.)
GroupingsMetrics broken down by the filters/groups currently applied in the dashboard
PrivacyAll data is de-identified — no user-level details are included

Practical Use Cases

  • Board and executive presentations: Import the CSV into Excel or Power BI to create polished charts with your organisation’s branding.
  • Trend analysis beyond 28 days: The export covers 6 months of weekly data, overcoming the dashboard’s 28-day rolling limitation for visual trend tracking.
  • Cross-tool integration: Feed Copilot metrics into existing BI pipelines, data warehouses, or reporting platforms.
  • Snapshot archiving: Schedule regular exports to build a historical archive of adoption and impact data.
  • Multi-agency reporting: Export tenant-level data and combine with org data in Power BI for agency-specific views.
Export vs Advanced Analytics

The dashboard export provides aggregated, de-identified data matching what you see on screen. For user-level or custom-segmented analysis, use Advanced Analytics (person queries in the Viva Insights analyst workbench). The export is ideal for quick reporting; Advanced Analytics is needed for deeper, customisable analysis.

1.4 Sentiment Section

The Sentiment section lives within the Impact page and shows qualitative user feedback about Copilot.

ComponentWhat It ShowsNotes
Sentiment results by questionAggregated responses to Copilot experience questionsIncludes benchmark comparison against similar orgs
Sentiment by groupHeat map showing sentiment across organisational groupsViva Insights customers only — requires Viva Insights licence
Manager team viewSentiment breakdown for a manager’s direct reportsSubject to minimum group size suppression
No Survey Data?

If no Copilot survey has been deployed, the Sentiment section will show “No survey responses available.” This is expected. The Satisfaction Rate metric (thumbs up/down) works independently of surveys and will still display on the Impact page regardless of whether surveys have been configured.

2. How to Delegate Access

Dashboard access can be shared with others through delegation. This is how leaders give their managers or analysts visibility into Copilot usage for their scope.

Two Methods to Add Delegates

MethodSteps
Method 1: SettingsDashboard → Settings (gear icon) → Delegation → Add → search for user → select scope → Save
Method 2: ShareDashboard → Share button (top right) → Add → search for user → select scope → Save

Delegation Rules

RuleDetail
Maximum delegates75 per delegator
No cascadingDelegation does not flow downward. If Alice delegates to Bob, Bob cannot re-delegate to Carol.
“Your group” scopeDelegate sees the delegator’s reporting chain only
“Your company” scopeDelegate sees the entire tenant — use sparingly
Agent Dashboard accessRequires “Your company” scope. “Your group” delegates cannot see the Agent Dashboard.
Delegator leaves orgAll delegates immediately lose access. Recovery: another leader must re-delegate to those users.
Plan for Delegator Departures

If a key delegator (e.g., a VP or CIO) leaves the organisation, all 75 delegates could lose dashboard access simultaneously. Document who has delegated to whom and ensure at least two senior leaders have overlapping delegation lists for continuity.

3. Copilot Analytics Reports

Copilot Analytics Reports are pre-built reports available at insights.cloud.microsoft → Reports. They provide deeper analysis than the dashboard pages without requiring Power BI setup.

Report Types

TypeExamplesDescription
Out-of-the-boxCopilot Studio Agents, Copilot for Sales AdoptionPre-built by Microsoft. Available automatically once prerequisites are met.
SharedCustom reports published by your analystsAnalysts with partition access can create queries and publish results as shared reports visible to others in the Reports section.

Access

  • Anyone with Dashboard access (direct or delegated) can view reports.
  • Analysts with a global partition can create and share reports.
  • Global Admins have full access.

No additional setup is required beyond what is already configured for the Copilot Dashboard.

5. Agent Dashboard

The Agent Dashboard provides visibility into Copilot Studio agents and declarative agents running in your tenant. It is accessible from the Copilot Dashboard for users with “Your company” scope.

Key Metrics

MetricDefinition
Active agentsNumber of agents that received at least one user interaction in the period
Active usersUnique users who interacted with any agent
Agent responsesTotal number of responses generated by agents
Returning usersUsers who interacted with agents in both the current and previous period
% of Copilot users using agentsAgent active users ÷ total Copilot active users — shows agent penetration
Copilot CreditsCredits consumed by agent interactions (relevant for pay-as-you-go billing)

Unique Insights

InsightWhat It Tells You
Most popular agentAgent with the highest number of active users
Most versatile agentAgent used across the widest variety of use cases or groups
Most shared agentAgent shared with the most users or groups in the tenant

Filters

FilterOptions
Agent creator typeUser, Your org, Microsoft, Partner
Agent sourceFilter by where the agent was built or published

Three Practical Scenarios

Scenario 1: Post-Launch Adoption Review

Your organisation deployed three custom agents last month. Use the Agent Dashboard to answer:

  • How many users actually tried each agent? (Active users per agent)
  • Are people coming back? (Returning users)
  • Which agent is getting the most traction? (Most popular agent)

Action: Double down on training for the popular agent; investigate why low-adoption agents are being ignored (discoverability? relevance?).

Scenario 2: Cost Governance for Pay-As-You-Go

Your tenant uses PAYG billing for agent interactions. Use the Agent Dashboard to answer:

  • How many Copilot Credits are being consumed? (Credits metric)
  • Which agents are the biggest consumers? (Sort by responses)
  • Are partner-built agents driving unexpected costs? (Filter by creator type: Partner)

Action: Set credit alerts, review high-consumption agents with their owners, consider blocking non-essential agents.

Scenario 3: Shadow AI Detection

Users are creating their own agents without IT approval. Use the Agent Dashboard to answer:

  • How many user-created agents exist? (Filter by creator type: User)
  • Are any user-created agents being widely shared? (Most shared agent)
  • Do any of these agents have external data connections that violate policy?

Action: Follow the Detect → Act workflow below.

Detect → Act Workflow

StageToolAction
DetectAgent DashboardIdentify agents that are unapproved, high-cost, or widely shared without governance review
DecideGovernance processEvaluate the agent against your organisation’s AI governance policy (data access, compliance, purpose)
Act QuickM365 Admin CentreImmediate actions: Block (disable agent), Approve (whitelist), or Review (flag for governance team)
Act PolicyPower Platform Admin CentreLong-term controls: apply DLP policies to restrict connector access, enforce environment boundaries, set maker policies
