Current Support

  • Semantic headings, main and navigation landmarks, labels for icon controls, keyboard focus indicators, and skip-to-content links.
  • Keyboard-operated navigation, role filters, topic filters, search, flyout panels, and Escape-to-close behavior.
  • Responsive layouts with horizontally scrollable data tables on narrow screens and print-specific styling.
  • Text alternatives for content images and tagged PDF structure generated from HTML.
  • Visible focus restoration after dismissing modal/flyout or mobile-navigation surfaces.

PDF Accessibility

HTML is the canonical and preferred accessible format. Generated PDFs receive document title, author, language, tagged structure, and link annotations, and they pass a structural preflight. That automation does not prove PDF/UA conformance or replace manual testing of reading order, headings, tables, alternative text, links, reflow, and screen-reader behavior.

Known Limits

  • Wide technical tables may require horizontal scrolling on mobile or high zoom.
  • Some command examples and architecture diagrams are easier to interpret in the HTML version than in print/PDF.
  • The site is maintained in English. Official product/UI labels retain Microsoft's documented spelling even when surrounding prose uses Australian English.
  • Third-party destinations, including Microsoft Learn and GitHub, have their own accessibility behavior and policies.

Report an Accessibility Problem

Open an accessibility issue and include the page URL, browser, operating system, assistive technology, zoom level, steps to reproduce, expected behavior, and observed behavior. Do not include confidential or personal data.

If an HTML page and its PDF differ, report both URLs and use the HTML page until the PDF is corrected.

Release Test Checklist

  • Keyboard-only navigation and visible focus at desktop and 375px mobile width.
  • Skip link reaches the main content.
  • Escape closes navigation/flyouts and returns focus to the invoking control.
  • Search/filter result changes are announced to assistive technology.
  • No overlap or loss of content at 320px width and 200%, 320%, and 400% zoom.
  • Heading hierarchy, landmarks, link purpose, image alternatives, and table headers are checked.
  • Generated PDFs pass structural preflight and manual screen-reader/reading-order spot checks.

This statement describes the project's target and current validation controls; it is not a certification of conformance. See the change history and content governance for ownership and review cadence.
