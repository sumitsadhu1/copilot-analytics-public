1. Attribute Mapping Table

This table maps legacy Viva Insights attribute names to the new M365 Admin Center naming convention (MODIS). Use this when migrating from the legacy upload path or preparing a new CSV.

Legacy Name (Viva Insights)M365 Admin Center NameRequired?Description / Notes
PersonId Microsoft_PersonEmail Required Employee’s primary email address (UPN). Must match Entra ID. Every row must have a value.
ManagerId Microsoft_ManagerEmail Required Manager’s email address. Must exist in Entra ID or upload causes orphan warnings. Every ManagerEmail value should match a PersonEmail in the same file.
Organization Microsoft_Organization Required Department or organisational unit. Drives the Organisation filter on dashboards.
EffectiveDate N/A (date picker in UI) Required (legacy) / Not needed (new) Legacy path: include in CSV, must be earlier than upload date. New M365 Admin Center path: the date is selected via the UI instead.
FunctionType Microsoft_JobDiscipline Optional Drives the Job Function filter on the Copilot Dashboard.
LevelDesignation Microsoft_LevelDesignation Optional Enables level-based views and analysis (e.g., IC vs Manager vs Director).
Location Microsoft_CompanyOfficeLocation Optional Drives the Location filter on the Copilot Dashboard.
HireDate Microsoft_HireDate Optional Used for tenure-based analysis and before/after cohort segmentation (format: YYYY-MM-DD).
HourlyRate Microsoft_HourlyRate Optional Confidential. Required for per-employee Assisted Value calculation. Set access classification to Confidential. If not provided, the dashboard uses the global default ($72/hour).
Layer Microsoft_Layer Optional Hierarchy depth from the top. Auto-derived from manager hierarchy if not explicitly uploaded.
SupervisorIndicator Microsoft_SupervisorIndicator Optional Indicates whether the employee is a manager (has direct reports). Auto-derived if not uploaded.
Custom (e.g., BusinessUnit) Custom (same name) Optional Up to 105 total attributes (reserved + custom). Use for Agency, Cost Centre, Region, Project, etc.

2. Data Classification

When uploading organisational data, each attribute is assigned an access classification that controls which applications and roles can read the data.

ClassificationVisibilityExample Attributes
Public Visible to all users and all apps that consume Viva Insights data. Organization, Location, LevelDesignation
Confidential Visible only to apps and roles you explicitly choose. Not available for dashboard filters. HourlyRate, CostCentre
Custom You choose which apps and roles can access the attribute (per-attribute granularity). Any custom attribute where selective access is needed.
Best Practice

Set HourlyRate and any compensation-related attributes to Confidential. Set demographic attributes like Organization, Location, and FunctionType to Public so they are available as dashboard filters.

3. Naming Conventions

RuleDetails
Use underscores, not spaces Column names must not contain spaces. Use Business_Unit or BusinessUnit, not Business Unit.
No Microsoft_ prefix for custom attributes The Microsoft_ prefix is reserved for system attributes. Custom columns using this prefix will cause validation errors.
Case sensitivity Attribute names are case-insensitive during upload but are stored as provided. Use consistent casing for clarity.
Maximum length Attribute names should not exceed 128 characters.
Total attribute limit Maximum of 105 total attributes per upload (reserved system attributes + your custom attributes combined).
Avoid special characters Use only letters, numbers, and underscores. Avoid hyphens, periods, and other special characters in attribute names.

4. Reserved Attribute Names

The following attribute names are reserved by the system and cannot be used for custom attributes. If you need a similar concept, use a different name (e.g., Agency_Location instead of Location).

Reserved Name (Legacy)Reserved Name (M365 Admin Center)
PersonIdMicrosoft_PersonEmail
ManagerIdMicrosoft_ManagerEmail
OrganizationMicrosoft_Organization
EffectiveDateN/A (UI-based)
FunctionTypeMicrosoft_JobDiscipline
LevelDesignationMicrosoft_LevelDesignation
LocationMicrosoft_CompanyOfficeLocation
HireDateMicrosoft_HireDate
HourlyRateMicrosoft_HourlyRate
LayerMicrosoft_Layer
SupervisorIndicatorMicrosoft_SupervisorIndicator
Important

Any custom attribute name beginning with Microsoft_ will be rejected during validation. If you need to add custom attributes with similar concepts, use your own naming convention (e.g., Agency, CostCentre, Region).
