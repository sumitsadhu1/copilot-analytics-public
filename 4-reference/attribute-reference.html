1. Attribute Mapping Table

This table maps Viva Insights organizational-data names to the Organizational Data in Microsoft 365 reserved names used in the Microsoft 365 admin center. Use the names required by the active tenant path and target app; do not infer the path from a date alone.

Legacy Name (Viva Insights)M365 Admin Center NameRequired?Description / Notes
PersonId Microsoft_PersonEmail Platform required Employee’s work email used to associate data with a person. This is the single platform-wide required Organizational Data in Microsoft 365 attribute.
ManagerId Microsoft_ManagerEmail Required for Viva Insights / Copilot Dashboard Manager’s email address. Required when sharing data with Viva Insights and the Copilot Dashboard; validate each reporting link and intended root.
Organization Microsoft_Organization Required for Viva Insights / Copilot Dashboard Department or organizational unit. Required when sharing data with Viva Insights and the Copilot Dashboard and used for organization grouping.
EffectiveDate N/A (date picker in UI) Tenant-path dependent Viva upload path: include in the file as documented by the live workflow. Microsoft 365 admin center: remove the column and set File valid as of in the UI; use Historical import for retroactive Viva corrections.
FunctionType Microsoft_JobDiscipline Optional Drives the Job Function filter on the Copilot Dashboard.
LevelDesignation Microsoft_LevelDesignation Optional Enables level-based views and analysis (e.g., IC vs Manager vs Director).
Location Microsoft_CompanyOfficeLocation Optional Drives the Location filter on the Copilot Dashboard.
HireDate Microsoft_HireDate Optional Used for tenure-based analysis and before/after cohort segmentation (format: YYYY-MM-DD).
HourlyRate Microsoft_HourlyRate Optional Confidential. Required for per-employee Assisted Value calculation. Set access classification to Confidential. If not provided, the dashboard uses the global default ($72/hour).
Layer Microsoft_Layer Optional Hierarchy depth from the top. Auto-derived from manager hierarchy if not explicitly uploaded.
SupervisorIndicator Microsoft_SupervisorIndicator Optional Indicates whether the employee is a manager (has direct reports). Auto-derived if not uploaded.
Custom (e.g., BusinessUnit) Custom (same name) Optional Up to 105 total attributes (reserved + custom). Use for Agency, Cost Centre, Region, Project, etc.

3. Data Classification

When uploading organisational data, each attribute is assigned an access classification that controls which applications and roles can read the data.

ClassificationVisibilityExample Attributes
Public Well-known organizational attributes made available to Microsoft 365 Profile and accessible to tenant users and Microsoft 365 apps, subject to the platform’s app-sharing and precedence behavior. Organization, Location, LevelDesignation
Confidential Sensitive well-known attributes for which the uploader chooses the apps that can receive the data. HourlyRate, CostCentre
Custom Tenant-defined attributes with controls for choosing which apps can access the data. Any custom attribute where selective access is needed.
Best Practice

Set HourlyRate and any compensation-related attributes to Confidential. Set demographic attributes like Organization, Location, and FunctionType to Public so they are available as dashboard filters.

Source: Organizational Data in Microsoft 365 (Microsoft Learn, updated 2 December 2025) defines public, confidential, and custom organizational data and app access controls.

4. Naming Conventions

RuleDetails
Use underscores, not spaces Column names must not contain spaces. Use Business_Unit or BusinessUnit, not Business Unit.
No Microsoft_ prefix for custom attributes The Microsoft_ prefix is reserved for system attributes. Custom columns using this prefix will cause validation errors.
Case sensitivity Attribute names are case-insensitive during upload but are stored as provided. Use consistent casing for clarity.
Maximum length Attribute names should not exceed 128 characters.
Total attribute limit Maximum of 105 total attributes per upload (reserved system attributes + your custom attributes combined).
Avoid special characters Use only letters, numbers, and underscores. Avoid hyphens, periods, and other special characters in attribute names.

5. Reserved Attribute Names

The following attribute names are reserved by the system and cannot be used for custom attributes. If you need a similar concept, use a different name (e.g., Agency_Location instead of Location).

Important

Any custom attribute name beginning with Microsoft_ will be rejected during validation. If you need to add custom attributes with similar concepts, use your own naming convention (e.g., Agency, CostCentre, Region).

6. Update, Delete, and Historical Correction

Organizational Data in Microsoft 365 uses incremental merge. Upload only affected users and include attributes you intend to update. An omitted or blank value preserves the existing value; use the documented marker when a value must be deleted.

Data type / operationSupported value or path
Delete string valueTwo single quotes ('') in CSV; three single quotes when editing through Excel
Delete integer value-1
Delete email valueA nonexistent email in the tenant, for example noname@contoso.com
Delete date value01-01-0001
Correct historical dataCSV Historical import with the appropriate File valid as of date; Microsoft documents this for Viva Insights data only
Restore Entra for Viva hierarchy attributesEnable ManagerId and/or Organization under Configure Entra connection, then remove uploaded optional attributes no longer needed
No Dummy-Row Workaround

Do not use a one-row “Replace All” file as a general rollback. It can create an artificial hierarchy and disrupt filters, partitions, and reports. Use the supported operations above or Microsoft Support for a critical migration rollback.
