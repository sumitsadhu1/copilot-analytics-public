Licensing & Access
Can managers see individual employee Copilot usage?
No. All data is aggregated at group level and protected by minimum group size thresholds (default 10, minimum 5), differential privacy, and distribution masking. No individual user behaviour is ever visible in the dashboards.
What happens if we drop below 50 licences?
The dashboard reverts to limited features: only tenant-level metrics and the Readiness page are available. Group-level filtering, the full Adoption page, and the Impact page require at least 50 assigned Copilot or Viva Insights licences.
Can we trial the Dashboard before licensing everyone?
Yes. The basic Copilot Dashboard is available to any M365/O365 tenant with Exchange Online — no paid Viva licence required. However, full group filtering and the Impact page require the 50-licence threshold.
Does the Dashboard track Copilot Chat (web) usage?
For paid Copilot users: Yes. If Copilot Chat reporting is enabled in your tenant and you have 50+ Copilot or Viva Insights licences, there is a separate licence filter on the dashboard to switch between Microsoft 365 Copilot and Copilot Chat views. For free Copilot Chat users (E3/E5 without Copilot add-on): Yes, with limits. As of April 2026, the Copilot Dashboard includes a Copilot Chat licence filter that shows insights for active users of the free Copilot Chat app. This requires 50+ Copilot or Viva Insights licences and Copilot Chat reporting enabled. Free users are not tracked by Viva Insights custom queries or Admin Center Copilot usage reports.
How often does the data refresh?
The Copilot Dashboard refreshes daily, showing a 28-day rolling window with up to a 6-day delay from the current date. Admin Center reports refresh approximately every 72 hours. Newly licensed employees may take an additional 7 days before appearing.
Is a separate Viva Insights licence needed if we already have Microsoft 365 Copilot?
No. Every M365 Copilot licence automatically includes the Viva Insights service plan — covering personal insights, manager/leader insights, and advanced analysis. A separate Viva Insights purchase is only needed if you want to analyse employees who do not have Copilot licences.
Who needs licences — only leaders viewing dashboards, or also the employees whose data we want to analyse?
Both. Employees whose collaboration data you want to analyse must have a Viva Insights licence assigned (included with every Copilot licence). Leaders and managers viewing dashboards also need a Viva Insights licence to access the full Copilot Dashboard and organisation insights.
Organizational Data
Can we see data by business unit, department, or country?
Yes, but you must upload organizational data (a CSV file with attributes like Business Unit, Department, Country) to enable these filters. Without an org data upload, only the Entra ID “Department” field is available as a filter.
Do I need to manually prepare and upload a CSV if we use SAP SuccessFactors or Workday?
No. Microsoft provides native HRIS connectors for both SAP SuccessFactors and Workday (Workday limited to ≤100K users). These connectors pull employee data directly from your HRIS on a weekly or monthly schedule.
Can I automate monthly CSV uploads?
Yes — multiple options exist: (1) HRIS connectors (SAP SuccessFactors, Workday) auto-sync on a schedule. (2) Azure Blob Storage connector — drop a CSV into an Azure blob and it’s picked up automatically. (3) API-based import (preview) — push data via REST API. Microsoft provides sample code at github.com/microsoft/orgdata_apibasedimport.
We have Department in Entra — why do we still need to upload org data?
Entra ID only syncs 3 attributes to Viva Insights: PersonId (UPN), ManagerId, and Organization (from the Department field). Extension attributes (extensionAttribute1-15), Company, Division, JobTitle, City, and Country are not synced — even if populated in Entra via AD Connect. To use these attributes in Viva Insights, upload them via CSV or configure an HRIS connector.
We uploaded org data and now filters show different values — can we revert?
Yes. You can use parallel data ingestion to restore Entra ID as the source for ManagerId and Organization while keeping custom CSV attributes. To remove all uploaded data entirely, use the “Replace All data” option with a single dummy row, then re-enable Entra sync. See Microsoft Learn. Always validate data in a test upload before applying to production.
“View by: Group” shows only one person’s name instead of sub-groups. How do I fix this?
The Group view uses the uploaded manager hierarchy. If most employees report directly to the top-level leader (flat hierarchy), the dashboard cannot split into sub-groups. Fix: Ensure intermediate managers (VPs, Directors) exist as rows in your data file with correct ManagerEmail chains. Every ManagerEmail value must match a PersonEmail in the same file.
Privacy
Can employees opt out of being analysed?
Users can opt out of Viva Insights personal insights via their settings. For advanced analysis and the Copilot Dashboard, the Insights Administrator can configure exclusion lists to remove specific users from aggregated insights.
Does Power BI bypass the minimum group size?
No. Power BI template reports from Viva Insights enforce the same minimum group size threshold as the Dashboard. The privacy control is applied at the data aggregation layer, not the UI layer.
How is the $72/hour default for Copilot Assisted Value calculated?
The $72/hour is based on data compiled by the U.S. Bureau of Labor Statistics as a broad estimate of loaded labour cost. Customers should adjust this to their own loaded rate for accurate ROI calculations. The rate can be changed in the Impact settings or via the Hours and Value calculator on the dashboard. Note: this rate is session-only and not persistent per current guidance.
General
Can we use the Agent Dashboard without the Copilot Dashboard?
Both are part of the Viva Insights web app. You cannot enable one without the other being available, but access to each can be controlled separately via Viva Feature Access Management (VFAM) policies.
We want broader productivity metrics (meeting time, focus time, after-hours). Is that covered here?
No — that is a separate Viva Insights workstream. Copilot Analytics covers Copilot-specific adoption, impact, and ROI. Broader productivity metrics use the same Viva Insights licence and analyst role but are configured through different queries and templates. See the Collaboration Analysis Guide for details.
What are intent patterns, and how do they appear in the Dashboard?
Intent patterns (public preview, April 2026) categorise Copilot Chat (work) prompts into four groups: Ask and find (questions and info retrieval), Catch up (summaries and recaps), Draft and brainstorm (content creation), and Other. These appear on the Adoption page and help you understand how people use Copilot, not just how much. Intent metrics are also available in Viva Insights custom person queries for deeper analysis.
Can I export data from the Copilot Dashboard?
Yes. The Dashboard now offers a CSV export of de-identified, weekly metrics covering the past 6 months. Select the Export button on the dashboard. The exported data matches what you see on screen — aggregated and de-identified. For user-level or custom-segmented exports, use Advanced Analytics (person queries in the Viva Insights analyst workbench).
Can we use Organizational Messages to boost Copilot adoption?
Yes. Organizational Messages (GA, April 2026) allow IT admins to send targeted Copilot adoption nudges via email, Teams, or Windows surfaces. The key feature is usage-based targeting — messages are sent based on actual Copilot usage behaviour (e.g., “never used” or “tried once”), not static group membership. Configure in M365 Admin Centre → Organizational messages. See the Admin Setup Guide for configuration steps.
Does Microsoft Purview DLP affect Copilot usage metrics?
Potentially. DLP policies can block prompts containing sensitive information types. Blocked prompts are not counted as Copilot actions in the Dashboard or Viva Insights. If a significant number of prompts are blocked by DLP, you may see lower-than-expected usage metrics. Coordinate with your Security team to understand what DLP policies are in place.
