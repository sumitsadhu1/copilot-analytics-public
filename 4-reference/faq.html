Licensing & Access
Can managers see individual employee Copilot usage?
No. All data is aggregated at group level and protected by minimum group size thresholds (default 10, minimum 5), differential privacy, and distribution masking. No individual user behaviour is ever visible in the dashboards.
What happens if we drop below 50 licences?
Core Dashboard capabilities remain available with 1+ Copilot licences. For 50-plan capabilities, Environment requirements for Viva Insights (Last updated 12 May 2026) counts the Viva Insights service plan included with Copilot and explicitly says 30 Copilot + 20 Viva Insights qualifies as 50 service plans. The Dashboard article (Last updated 1 July 2026) separately says 50 Copilot or 50 Viva Insights. Do not assume mixed licensing fails: verify assigned service plans and tenant feature presence. For Agent Dashboard preview, conservatively require 50 Copilot licences plus agent activity.
Can we trial the Dashboard before licensing everyone?
Yes. With 1 assigned Copilot licence, eligible tenants can begin using core Readiness, Adoption, and Impact capabilities. For 50-plan capabilities, count assigned Viva Insights service plans and validate the mixed-licence source conflict described above. Agent Dashboard is preview and should conservatively be treated as requiring 50 Copilot licences plus agent activity.
How is Copilot Chat usage reported?
The Dashboard uses the current merged Copilot Chat metric; retain Copilot Chat (web) only as a legacy transition series that is expected to trend toward zero. For users without a Microsoft 365 Copilot licence, the Microsoft 365 admin center provides a dedicated, exportable Microsoft 365 Copilot Chat usage report (Last updated 10 July 2026). The Dashboard licence view, admin report, licensed-user-only Graph endpoint, and Purview audit are separate reporting surfaces.
How often does the data refresh?
The Copilot Dashboard refreshes daily, showing a 28-day rolling window with up to a 6-day delay from the current date. Admin Center reports refresh approximately every 72 hours. Newly licensed employees may take an additional 7 days before appearing.
Is a separate Viva Insights licence needed if we already have Microsoft 365 Copilot?
No. Every M365 Copilot licence automatically includes the Viva Insights service plan — covering personal insights, manager/leader insights, and advanced analysis. A separate Viva Insights purchase is only needed if you want to analyse employees who do not have Copilot licences.
Who needs licences — only leaders viewing dashboards, or also the employees whose data we want to analyse?
Both. Employees whose collaboration data you want to analyse must have a Viva Insights licence assigned (included with every Copilot licence). Leaders and managers viewing dashboards also need a Viva Insights licence to access the full Copilot Dashboard and organisation insights.
Organizational Data
Can we see data by business unit, department, or country?
Yes, but you must upload organizational data (a CSV file with attributes like Business Unit, Department, Country) to enable these filters. Without an org data upload, only the Entra ID “Department” field is available as a filter.
Do I need to manually prepare and upload a CSV if we use SAP SuccessFactors or Workday?
No. Microsoft provides native HRIS connectors for both SAP SuccessFactors and Workday (Workday limited to ≤100K users). These connectors pull employee data directly from your HRIS on a weekly or monthly schedule.
Can I automate monthly CSV uploads?
Yes — multiple options exist: (1) HRIS connectors (SAP SuccessFactors, Workday) auto-sync on a schedule. (2) Azure Blob Storage connector — drop a CSV into an Azure blob and it’s picked up automatically. (3) API-based import (preview) — push data via REST API. Microsoft provides sample code at github.com/microsoft/orgdata_apibasedimport.
We have Department in Entra — why do we still need to upload org data?
Entra ID only syncs 3 attributes to Viva Insights: PersonId (UPN), ManagerId, and Organization (from the Department field). Extension attributes (extensionAttribute1-15), Company, Division, JobTitle, City, and Country are not synced — even if populated in Entra via AD Connect. To use these attributes in Viva Insights, upload them via CSV or configure an HRIS connector.
We uploaded org data and now filters show different values — can we revert?
Yes, by attribute and surface. Use parallel data ingestion to restore Entra as the source for ManagerId and Organization while retaining uploaded custom attributes. To remove uploaded values, first confirm Entra is configured for those two fields, then use the documented Organizational Data update/delete operations for optional fields. Do not seed dummy rows. Microsoft 365 Profile precedence is a separate setting. See Use Microsoft Entra plus .csv files for parallel data uploads (updated 30 June 2025) and Import and manage your organizational data with a .csv file — Update or delete data (updated 2 April 2026). Test the change, record success/failure criteria, and escalate migration rollback failures through Microsoft Support.
“View by: Group” shows only one person’s name instead of sub-groups. How do I fix this?
The Group view uses the uploaded manager hierarchy. If most employees report directly to the top-level leader (flat hierarchy), the dashboard cannot split into sub-groups. Fix: Ensure intermediate managers (VPs, Directors) exist as rows in your data file with correct ManagerEmail chains. Every ManagerEmail value must match a PersonEmail in the same file.
Privacy
Can employees opt out of being analysed?
Users can opt out of Viva Insights personal insights via their settings. For advanced analysis and the Copilot Dashboard, the Insights Administrator can configure exclusion lists to remove specific users from aggregated insights.
Does Power BI bypass the minimum group size?
No. Power BI template reports from Viva Insights enforce the same minimum group size threshold as the Dashboard. The privacy control is applied at the data aggregation layer, not the UI layer.
How is the $72/hour default for Copilot Assisted Value calculated?
The $72/hour is based on data compiled by the U.S. Bureau of Labor Statistics as a broad estimate of loaded labour cost. Customers should adjust this to their own loaded rate for accurate ROI calculations. The rate can be changed in the Impact settings or via the Hours and Value calculator on the dashboard. Note: this rate is session-only and not persistent per current guidance.
General
Can we use the Agent Dashboard without the Copilot Dashboard?
Treat Agent Dashboard as a separate preview entitlement and access check within the Viva Insights web app. The feature-specific article requires 50 Microsoft 365 Copilot licences plus agent activity, while the broader Dashboard article also lists a 50-Viva-Insights path. Use the Copilot threshold conservatively and validate a Viva-only tenant. It requires company-wide access, is unavailable to managers and group-only delegates, and is unavailable in GCC and other national or regional clouds. Source: Connect to the Agent Dashboard (preview) (Last updated 16 June 2026) and Supported Microsoft 365 environments for Viva Insights features (Last updated 30 June 2026).
We want broader productivity metrics (meeting time, focus time, after-hours). Is that covered here?
No — that is a separate Viva Insights workstream. Copilot Analytics covers Copilot-specific adoption, impact, and ROI. Broader productivity metrics use the same Viva Insights licence and analyst role but are configured through different queries and templates. See the Collaboration Analysis Guide for details.
What are intent patterns, and how do they appear in the Dashboard?
Intent patterns categorise supported Copilot Chat prompts into four groups: Ask and find, Catch up, Draft and brainstorm, and Other. These appear on the Adoption page and help explain how people use Copilot, not just how much. Validate preview status and tenant availability before depending on them.
Can I export data from the Copilot Dashboard?
Yes, in preview, when eligible. The exporter needs the documented 50-licence threshold and global / “Your company” access; group-only access is insufficient, and the feature is unavailable in GCC. The fixed, non-customizable CSV contains anonymized user-level rows: weekly data for up to six months or daily data for 28 days. Dashboard filters do not customize it, and Copilot Assisted Hours is not included. Store it as sensitive analytics data because repeated rows or organizational-data joins can increase re-identification risk. Source: Export Copilot metrics from the Microsoft Copilot Dashboard (preview) (Last updated 15 July 2026).
Can we use Organizational Messages to boost Copilot adoption?
Yes. Organizational Messages (GA, April 2026) allow IT admins to send targeted Copilot adoption nudges via email, Teams, or Windows surfaces. The key feature is usage-based targeting — messages are sent based on actual Copilot usage behaviour (e.g., “never used” or “tried once”), not static group membership. Configure in M365 Admin Centre → Organizational messages. See the Admin Setup Guide for configuration steps.
Does Microsoft Purview DLP affect Copilot usage metrics?
Potentially, but the counting behavior is not verifiable from current Microsoft Learn. DLP can block prompt processing or web grounding depending on the configured control, but the reviewed DLP and Copilot Analytics articles do not state whether a blocked attempt is counted as a Copilot action. Do not use DLP as the explanation for an adoption gap without a tenant test or Microsoft Support confirmation.
Which Microsoft 365 environments support these capabilities?
Availability is feature-specific. Worldwide has the broadest coverage. GCC supports selected Copilot Analytics and advanced-analysis capabilities but not Agent Dashboard, Copilot metrics export, Copilot Power BI reports, or API/Azure organizational-data imports. GCC High, DoD, and other national or regional clouds must be checked feature by feature; plan availability does not imply parity. Use Supported Microsoft 365 environments for Viva Insights features (Last updated 30 June 2026), then verify Message Center and the tenant UI.
Why might Assisted Hours or Value be lower than expected?
Optional diagnostic data affects Assisted Hours, emails sent using Copilot, Teams meetings and meeting hours summarized or recapped, and Teams chats summarized. Required diagnostic data provides only a subset. Confirm diagnostic coverage before comparing groups or periods. Assisted Hours and Value are modeled estimates, not realized savings or proof that Copilot caused a business outcome; use documented assumptions, a stable baseline, and independently governed outcome data. Source: Connect to the Microsoft Copilot Dashboard for Microsoft 365 customers (Last updated 1 July 2026).
