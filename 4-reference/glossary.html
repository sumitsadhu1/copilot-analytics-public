1. Glossary of Key Terms

TermDefinition
AI Administrator Entra ID role that manages applicable AI and Copilot Dashboard settings, access assignment, exclusions, and feature controls. It does not grant automatic Copilot Dashboard viewing or Insights Analyst query rights.
Assisted Hours The estimated number of hours saved by employees using Copilot features. Calculated from per-action time savings applied to Copilot interaction counts. Displayed on the Dashboard Impact page with a 28-day rolling window.
Assisted Value The monetary estimate of Copilot time savings, calculated as Assisted Hours × Loaded Labour Rate. Default rate is $72/hour (session-only, not persistent). Customers should adjust to their own rate.
Auto-enablement The process by which eligible users are automatically assigned Copilot licences based on tenant policy, without manual per-user assignment.
Copilot Analytics Reports Pre-built Power BI reports accessible from the Viva Insights web app, including the M365 Copilot Adoption report and the M365 Copilot Impact report. These provide deeper drill-down than the Dashboard.
Copilot Chat The current merged usage metric for the supported Copilot Chat experience. “Copilot Chat (web)” is a legacy transition series that is expected to trend toward zero; do not interpret that legacy decline as an adoption decline by itself. Unlicensed Chat users have a dedicated Microsoft 365 admin center usage report.
Consumption Dashboard One of Microsoft’s six Copilot Analytics areas. It shows supported Copilot Credit consumption by service and eligible organizational scope. It is an operational analytics view, not the invoice or official billing ledger.
Delegated Access A mechanism where an eligible Dashboard user grants another user access to Copilot insights within the delegator’s permitted scope. A group-scoped delegate sees the delegator’s group; a company-scoped delegate can see company data and may access Agent Dashboard where all other preview prerequisites are met. Delegation does not add an admin or analyst role.
Exclusion List A list of users excluded from Viva Insights data collection and aggregation. Configured by the Insights Administrator in Privacy Settings. Changes take up to 5 days to take effect.
Insights Administrator Entra ID role responsible for configuring Viva Insights settings including privacy, partitions, organizational data, manager settings, and analyst assignments. Roles are independent and non-cumulative: Insights Administrator alone has no Insights Analyst query rights and no automatic Copilot Dashboard view.
Insights Analyst Entra ID role that can run person queries, access Power BI templates, and create custom reports within assigned partitions. Assignment to the Global partition also grants company-wide Copilot Dashboard access; assignment only to a non-Global partition does not.
Loaded Labour Rate The fully-burdened hourly cost of an employee, including salary, benefits, and overhead. Used to calculate Assisted Value. Default is $72/hour; organisations should substitute their own rate.
Minimum Group Size The smallest number of employees for which aggregated data is displayed on dashboards and in reports. Default is 10; lowest allowed is 5. Groups below this threshold show “N/A.”
Minimum Team Size The minimum number of direct reports a manager must have for team-level insights to be available. Configured separately from minimum group size.
Organizational Data Employee attribute data (department, location, job function, etc.) uploaded to Microsoft 365 to enable group filtering and segmentation on dashboards and in Power BI reports.
Partition A scoped subset of the measured population that restricts which employees an Insights Analyst can query. Used to enforce data boundaries (e.g., per-agency isolation).
Rolling Window The calculation period for a metric, not a universal retention limit. Many Dashboard measures use a 28-day rolling window with data latency; selected trendlines can span six months, the preview export provides six months weekly or 28 days daily, and Advanced Reporting retains up to 13 months of Copilot metric history.
Agent Dashboard (preview) A preview view of supported agent adoption and usage. Conservatively require 50 Microsoft 365 Copilot licences plus agent activity because Microsoft sources conflict on the Viva-Insights-only route. Requires company-wide access and is unavailable in GCC and other national or regional clouds.
This guide’s implementation framework The guide’s optional five-stage rollout sequence. It is not a Microsoft product classification. Microsoft’s current official framing has six areas: admin center reports, Copilot Dashboard, Agent Dashboard, Consumption Dashboard, Copilot Analytics reports, and Advanced Reporting.
VFAM (Viva Feature Access Management) The admin control surface in the M365 Admin Center that enables or disables specific Viva features (e.g., Copilot Dashboard, Agent Dashboard) for the tenant or specific groups.

Role and access sources: Roles in Viva Insights (Last updated 7 July 2026), Manage settings for the Microsoft Copilot Dashboard, Agent Dashboard, and Viva Insights web app (Last updated 20 March 2026), Delegate access to the Copilot Dashboard, Agent Dashboard, and Copilot insights (Last updated 9 February 2026), and Configure manager settings (Last updated 25 June 2026).
