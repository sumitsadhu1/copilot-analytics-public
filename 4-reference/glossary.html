1. Glossary of Key Terms

TermDefinition
AI Administrator Entra ID role that manages AI-related settings across the tenant, including Copilot Dashboard access, delegation, and Viva Feature Access Management policies.
Assisted Hours The estimated number of hours saved by employees using Copilot features. Calculated from per-action time savings applied to Copilot interaction counts. Displayed on the Dashboard Impact page with a 28-day rolling window.
Assisted Value The monetary estimate of Copilot time savings, calculated as Assisted Hours × Loaded Labour Rate. Default rate is $72/hour (session-only, not persistent). Customers should adjust to their own rate.
Auto-enablement The process by which eligible users are automatically assigned Copilot licences based on tenant policy, without manual per-user assignment.
Copilot Analytics Reports Pre-built Power BI reports accessible from the Viva Insights web app, including the M365 Copilot Adoption report and the M365 Copilot Impact report. These provide deeper drill-down than the Dashboard.
Delegated Access A mechanism where an AI Administrator or Insights Administrator grants another user access to the Copilot Dashboard at either “Your group” (team) or “Your company” (tenant) scope.
Exclusion List A list of users excluded from Viva Insights data collection and aggregation. Configured by the Insights Administrator in Privacy Settings. Changes take up to 5 days to take effect.
Insights Administrator Entra ID role responsible for configuring Viva Insights settings including privacy, partitions, org data uploads (legacy path), and analyst role assignments.
Insights Analyst Entra ID role that can run person queries, access Power BI templates, and create custom reports within the Viva Insights advanced analysis workbench. Scoped to assigned partitions.
Loaded Labour Rate The fully-burdened hourly cost of an employee, including salary, benefits, and overhead. Used to calculate Assisted Value. Default is $72/hour; organisations should substitute their own rate.
Minimum Group Size The smallest number of employees for which aggregated data is displayed on dashboards and in reports. Default is 10; lowest allowed is 5. Groups below this threshold show “N/A.”
Minimum Team Size The minimum number of direct reports a manager must have for team-level insights to be available. Configured separately from minimum group size.
Organizational Data Employee attribute data (department, location, job function, etc.) uploaded to Microsoft 365 to enable group filtering and segmentation on dashboards and in Power BI reports.
Partition A scoped subset of the measured population that restricts which employees an Insights Analyst can query. Used to enforce data boundaries (e.g., per-agency isolation).
Rolling Window The time period for which the Dashboard displays data. The Copilot Dashboard uses a 28-day rolling window with up to a 6-day delay. Historical snapshots are not retained natively.
VFAM (Viva Feature Access Management) The admin control surface in the M365 Admin Center that enables or disables specific Viva features (e.g., Copilot Dashboard, Agent Dashboard) for the tenant or specific groups.
