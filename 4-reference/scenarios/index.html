When to use this catalogue

Each scenario starts with the symptom a user actually reports — not the underlying feature area — and walks through the most likely causes in priority order with a verification step for each.

  • Use these scenarios when the symptom spans multiple admin surfaces, or when role / surface confusion is the suspected root cause.
  • Use the Troubleshooting Guide symptom-tables for quick single-surface lookups (Admin Centre Reports, Copilot Dashboard, Agent Dashboard, Advanced Analytics, Data Upload).

Suggesting a new scenario

A scenario is worth writing when: (a) the symptom shows up repeatedly in real customer escalations, (b) the root cause is non-obvious or spans multiple surfaces / roles, and (c) the existing symptom-table in troubleshooting.html alone would mislead an investigator. Each scenario should include: a one-sentence quick answer, a 30-second diagnosis flow, causes ranked by likelihood with fix + verification, a “most common misdiagnosis” warning, and a “what done looks like” check.
