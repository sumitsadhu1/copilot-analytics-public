Microsoft 365 Copilot Analytics

Scenario 01
“I can’t see the option to upload org data at aka.ms/copilotdashboard”
Typically reported byInsights Administrator who expected a CSV-upload button on the Copilot Dashboard.
Surfaces involvedM365 Admin Center, analysis.insights.cloud.microsoft, Entra ID
Roles touchedInsights Admin, Organizational Data Source Admin, AI Admin, Global Admin
Version1.0 • Published May 2026

Quick answer

aka.ms/copilotdashboard is a short link that redirects to the Copilot Dashboard view inside Viva Insights (analysis.insights.cloud.microsoft). The dashboard view itself never hosts an “Upload org data” control — for any role. The upload UI lives in a different place depending on whether the tenant has been migrated to MODIS (Microsoft Organizational Data in Microsoft 365):

  • Post-MODIS → upload has moved to the Microsoft 365 Admin Center (admin.microsoft.com, a different surface entirely) and now requires the Organizational Data Source Administrator role — Insights Administrator alone is no longer sufficient.
  • Pre-MODIS → upload stays on analysis.insights.cloud.microsoft but on a different page from the Copilot Dashboard view: Admin menu → Data hub. Insights Administrator (or Global Admin) is sufficient.

Surface map — where each URL takes you

Use this to confirm you’re not asking the dashboard view to host the admin UI:

URL / short linkWhat it isHosts the upload?
aka.ms/copilotdashboard Short link → redirects to the Copilot Dashboard view inside Viva Insights. No — ever.
analysis.insights.cloud.microsoft (Copilot Dashboard page) Same destination as the short link above. Adoption / Readiness / Impact pages. No — ever.
analysis.insights.cloud.microsoft → Admin menu → Data hub A different page on the same Viva Insights surface — the admin area for analytics settings and data sources. Yes, on pre-MODIS tenants only. Requires Insights Administrator or Global Admin.
admin.microsoft.com → Setup → Migration and imports → Organizational Data in Microsoft 365 The Microsoft 365 Admin Center — a completely separate surface from Viva Insights. Yes, on post-MODIS tenants only. Requires Organizational Data Source Administrator (or Global Admin).

30-second diagnosis

  1. Open admin.microsoft.com → Setup → Migration and imports.
  2. Do you see “Organizational Data in Microsoft 365”?
    • Yes → Tenant is post-MODIS. Upload UI lives here, on a different surface from the Copilot Dashboard. The user needs the Organizational Data Source Administrator role (Insights Administrator alone will not show the upload).
    • No → Tenant is pre-MODIS. Upload UI lives on the same Viva Insights surface as the dashboard (analysis.insights.cloud.microsoft), but on a different page: Admin menu → Data hub → .csv upload. Insights Administrator (or Global Admin) is sufficient.
  3. Confirm role assignment in Entra ID and have the user sign out / sign back in to refresh the token.

Causes ranked by likelihood

#CauseWhy it happensFix & verification
1 Looking at the wrong page within the right family of admin surfaces. The Copilot Dashboard view (which is what aka.ms/copilotdashboard redirects to) never shows an “Upload org data” control — for any role. The Copilot Dashboard view is a consumption surface for leaders, managers, and admins to read metrics. The org-data ingestion UI is intentionally separated — either to a different page within the same Viva Insights surface (pre-MODIS: analysis.insights.cloud.microsoft → Admin → Data hub) or to a different surface entirely (post-MODIS: admin.microsoft.com).
  • Post-MODIS — open the Microsoft 365 Admin Center (a different surface): admin.microsoft.com → Setup → Migration and imports → Organizational Data in Microsoft 365 → Data Connections → Get started (first time) or New import.
  • Pre-MODIS — stay on analysis.insights.cloud.microsoft but switch from the dashboard view to the admin area: open the Admin menu (top-left nav) → Data hub → .csv upload → Start.
  • Verify: the upload wizard loads with a file picker and an attribute-mapping step.
2 Tenant is post-MODIS and the user only holds Insights Administrator. The MODIS surface checks a different role. The MODIS data platform serves multiple M365 apps (Viva Insights, Copilot Dashboard, Microsoft 365 Profile, …), so Microsoft introduced a dedicated role for managing org-data ingestion. Insights Administrator continues to govern analytics settings but does not grant upload rights on the MODIS surface. Per Microsoft Learn, only two roles are accepted by Organizational Data in Microsoft 365: Microsoft 365 Global Administrator (auto-enrolled) and Organizational Data Source Administrator (must be assigned by a Global Administrator).
  • Assign Organizational Data Source Administrator in Entra ID: admin.microsoft.com → Users → Active Users → select user → Manage roles → search for “Organizational Data Source Administrator.”
  • Microsoft 365 Global Administrator is the only other role accepted by MODIS — use it as a break-glass option for the first upload, then move to Organizational Data Source Administrator for ongoing operations (least privilege).
  • Verify: after sign-out/sign-in, the “Organizational Data in Microsoft 365” node appears under Setup → Migration and imports, and the Get started / New import button is enabled.
  • Source: Microsoft Learn — Organizational Data in Microsoft 365: Roles and permissions (and the Organizational Data Source Administrator details section).
3 Role was just assigned — token has not refreshed. Entra role activations propagate to access tokens on next sign-in or token refresh, which can take up to 60 minutes if the user keeps an existing session open.
  • Sign out of all Microsoft 365 surfaces, close the browser, and sign back in (incognito window is fastest).
  • Verify: open myaccount.microsoft.com → My roles — the newly assigned role should be listed before reopening the admin surface.
4 Role is granted via Privileged Identity Management (PIM) but not activated. If the org uses PIM, the user may be eligible for the role but not currently holding it. Eligibility alone does not surface admin UI.
  • portal.azure.com → Privileged Identity Management → My roles → activate Organizational Data Source Administrator (or Insights Administrator on pre-MODIS) for the required duration.
  • Verify: activation appears under Active assignments with a remaining timer.
5 Role is scoped to a custom Administrative Unit (AU) rather than the tenant. An AU-scoped role cannot perform tenant-wide ingestion. The MODIS Data Connections node is tenant-scoped, so the UI is hidden for AU-scoped admins.
  • In Entra ID → Roles and administrators, locate the assignment and confirm the Scope column shows “Directory” (tenant), not an AU name.
  • If scoped, re-assign at directory scope (requires a Privileged Role Administrator or Global Administrator).
6 Wrong account in the browser session. Multi-account browsers (work + personal, or two work tenants) often re-use a cached account silently — the dashboard loads under the unprivileged identity.
  • Open aka.ms/copilotdashboard in an incognito / InPrivate window and sign in explicitly with the admin account.
  • Verify: top-right account chip shows the correct UPN.
7 Ingestion is already handled by a connector or Azure Blob source — manual CSV is intentionally hidden / disabled. If the tenant has an active SAP SuccessFactors, Workday, Azure Blob, or API-based source for the same target apps, the CSV-upload tile may be unavailable to prevent two competing sources of truth.
  • admin.microsoft.com → Setup → Migration and imports → Organizational Data in Microsoft 365 → Data Connections → review the existing connection list.
  • To switch sources, either pause/remove the existing connector or update that source (CSV upload is one method of several — not the only path).
  • See the Upload Methods Overview in the Data Onboarding Guide.
8 Viva Feature Access Management (VFAM) policy disables the Copilot Dashboard surface for this user. VFAM gates the viewing surface itself. The user lands on a permission-denied page or a stripped-down view that does not even expose admin entry points if they were ever there.
  • admin.microsoft.com → Viva → Feature Access Management → Viva Insights → verify Copilot Dashboard is ON for the user’s group.
  • Remember: this only affects the dashboard surface. The upload still happens via the M365 Admin Center or analysis.insights.cloud.microsoft → Admin, which are governed by Entra roles, not VFAM.
9 Tenant under 2,500 licensed users without the required admin role. Small-tenant guardrails: certain admin entry points are gated until the user holds AI Administrator, Insights Administrator, or Global Administrator.
  • Confirm the user holds at least one of: AI Administrator, Insights Administrator, Organizational Data Source Administrator (post-MODIS), or Global Administrator.
  • Verify: myaccount.microsoft.com → My roles.
Most common misdiagnosis

Teams escalate this as a “permissions bug” on the Copilot Dashboard. In >90% of cases the cause is one of (1) wrong surface or (2) post-MODIS role mismatch. Always verify the migration state in admin.microsoft.com → Setup → Migration and imports before opening a support case.

What “done” looks like

  • User signs into the correct admin surface (M365 Admin Center post-MODIS, or analysis.insights.cloud.microsoft → Admin pre-MODIS).
  • Holds the matching role (Organizational Data Source Administrator post-MODIS, or Insights Administrator pre-MODIS).
  • Sees and clicks Get started / New import / .csv upload → Start, completes the wizard, and the job appears with status Validating → Processing → Complete.
