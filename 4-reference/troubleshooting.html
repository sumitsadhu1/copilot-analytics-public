1. Admin Centre Reports

SymptomCauseResolution
No data displayed Reports take up to 72 hours to populate after initial licence assignment. Wait 72 hours after assigning Copilot licences. Verify licence assignment in admin.microsoft.com → Users → Active Users.
No Copilot licences visible Licences not purchased or not assigned to users. Check admin.microsoft.com → Billing → Licences. Purchase and assign Microsoft 365 Copilot licences to target users.
Can’t export CSV from reports User does not have the Reports Reader role. Assign the Reports Reader role in Entra ID to the user who needs to export data.
User details hidden / only aggregates visible Concealed user details is enabled (privacy setting). Navigate to admin.microsoft.com → Settings → Org Settings → Reports → uncheck “Display concealed user, group, and site names in all reports.” Allow 72 hours for changes to take effect.
0 active users despite assigned licences Users have not yet interacted with Copilot, or reporting has not caught up. Verify users have actually used a Copilot feature (e.g., prompted Copilot in Word or Teams). Wait 72 hours for data to appear.

2. Copilot Dashboard

SymptomCauseResolution
Access denied / cannot open dashboard Multiple possible causes: VFAM (Viva Feature Access Management) is off, AI Administrator not added, delegation policy not configured, or tenant has fewer than 2,500 users without appropriate roles.
  1. Verify VFAM is enabled: admin.microsoft.com → Viva → Feature Access Management → Viva Insights → “Copilot Dashboard” must be ON.
  2. Ensure the user has Admin access or has been added as a delegate by an AI Administrator.
  3. For tenants <2,500 users: user needs AI Administrator, Insights Administrator, or Global Admin role.
Impact page missing collaboration data Behavioral Experiences (formerly “Collaboration data”) is turned OFF in Viva Insights privacy settings. Enable Behavioral Experiences: insights.cloud.microsoft → Settings → Privacy → toggle “Behavioral Experiences” ON. Allow 7 days for data to populate.
Only Readiness page visible (no Adoption or Impact) No Copilot licences assigned, or data processing not yet complete. Assign at least 1 Copilot licence and wait up to 7 days for data processing. For advanced features (benchmarks, Agent Dashboard, sentiment, delegation), assign 50+ combined Copilot/Viva Insights licences.
No group filtering available Organisational data has not been uploaded. Upload organisational data via CSV, HRIS connector, or another supported method. See the Data Onboarding Guide.
Impact page shows no data Requires at least 7 days of usage data after licence assignment. Wait at least 7 days after licences are assigned and users begin using Copilot. If data still doesn’t appear, verify Behavioral Experiences is ON.
Data suppressed / shows “N/A” Group is below the minimum group size threshold (default 10). Ensure the group being viewed meets the minimum group size requirement. The minimum can be adjusted (lowest is 5) by the AI Administrator or Insights Administrator in Privacy Settings.
Managers can’t see team data Manager Settings are not enabled in Viva Insights. Enable Manager Settings: insights.cloud.microsoft → Settings → Manager Settings → toggle ON. The manager’s team must also meet the minimum team size threshold.
Sentiment section is empty No Copilot survey has been configured or responses haven’t been collected. Configure the Copilot satisfaction survey in Viva Insights Settings. Sentiment data appears once enough responses are collected (subject to minimum group size).
Exclusion list not taking effect Exclusion list changes take up to 5 days to propagate. Wait 5 days after adding users to the exclusion list. Verify the list was saved correctly in Privacy Settings.
Org data not showing in filters Wrong attribute names in uploaded CSV or missing EffectiveDate (legacy path).
  • Verify attribute names match expected naming convention (legacy: PersonId, ManagerId; new: Microsoft_PersonEmail, Microsoft_ManagerEmail)
  • For legacy path: ensure EffectiveDate column exists and date is earlier than upload date
  • Check the Data Quality page in Viva Insights for validation errors

3. Agent Dashboard

SymptomCauseResolution
Agent Dashboard not visible Delegates need “Your company” scope (not “Your group”) to access the Agent Dashboard. Update delegation scope: insights.cloud.microsoft → Settings → Delegation → change the delegate’s scope to “Your company.”
Agent Dashboard disabled VFAM policy for AgentDashboard is set to OFF. Check admin.microsoft.com → Viva → Feature Access Management → Viva Insights → verify “AgentDashboard” is enabled.
No agent data displayed No agents have been deployed in the tenant, or agents have not been used. Verify agents are deployed and users are interacting with them. Agent data requires active usage to populate.

4. Advanced Analytics (Power BI)

SymptomCauseResolution
Can’t access analyst experience User does not have the Insights Analyst role assigned. Assign the Insights Analyst role in Entra ID. Alternatively, use the Insights Administrator role (which includes analyst capabilities).
Query returns no data Analyst’s partition does not include the target users. Verify partition membership. The Insights Administrator can check and update partition assignments in insights.cloud.microsoft → Settings → Partitions.
Query fails with time period error Selected time period exceeds available historical data. Reduce the query time period to fall within available data. Viva Insights retains up to 27 months of collaboration data and 12 months of Copilot data.
Power BI won’t connect to Viva Insights Authentication issue — incorrect account or expired token. Sign in with the same account that has the Insights Analyst role. Clear cached credentials in Power BI Desktop: File → Options → Data Source Settings → Clear Permissions. Retry connection.
No segmentation available in PBI reports Organisational data has not been uploaded or custom attributes are missing. Upload organisational data with the attributes you need for segmentation. See the Attribute Reference.
Before/after analysis unavailable No pre-adoption baseline data exists — Copilot was deployed before Viva Insights was configured. Before/after comparisons require data from both periods. If no pre-adoption data was captured, use the existing Impact page metrics as a proxy or begin collecting a baseline now for future comparison.

5. Data Upload Issues

Validation Failures

These errors occur before data is processed. The upload is rejected and no data is ingested.

ErrorCauseResolution
Blank PersonEmail / PersonId One or more rows have an empty email field. Ensure every row has a valid email address in the PersonEmail (or PersonId / Microsoft_PersonEmail) column.
Missing required attributes CSV is missing one or more required columns: PersonEmail, ManagerEmail, Organization (or their Microsoft_* equivalents). Add the missing required columns to the CSV header row. See the Attribute Reference for the full mapping.
Empty rows in CSV Blank rows (including trailing empty rows) cause validation to fail. Remove all empty rows from the CSV. Pay special attention to rows at the end of the file.
Unlicensed apps referenced CSV references applications or services not licensed in the tenant. Remove or correct app references. Ensure all referenced services are active in the tenant.
Unsupported custom attributes Custom attribute names contain spaces, special characters, or exceed 105 total attribute limit. Use underscores instead of spaces. Remove special characters. Reduce total attribute count to ≤105.
Microsoft_ prefix on custom attributes Custom attributes must not use the Microsoft_ prefix — this is reserved for system attributes. Rename custom columns to remove the Microsoft_ prefix. Use a plain name like BusinessUnit or CostCentre.

Processing Failures

StatusMeaningResolution
Processing failed Backend error during data ingestion. Retry the upload. If the issue persists, contact Microsoft Support with the error details and correlation ID.
Suspended Credentials expired or backend system unavailable (common with HRIS connectors). Re-authenticate the connector credentials. Retry the next day. If suspended repeatedly, check service health at status.office.com.
Partial success Some rows failed validation (API, Workday, or SAP connectors only). Download the error report from the upload status page. Fix the failing rows and re-upload. Common causes: duplicate emails, malformed dates, invalid manager references.
Validation Error Reference

For the complete list of upload validation errors and error codes, see Validation errors for organizational data uploads on Microsoft Learn.

6. Auditing Delegation

Delegation changes are logged in Microsoft Purview audit logs. Use these operation names to track delegation activity:

Purview OperationDescription
AddDelegates A delegate was added to the Copilot Dashboard or Viva Insights.
RemoveDelegates A delegate was removed from the Copilot Dashboard or Viva Insights.

To view these logs: compliance.microsoft.com → Audit → Search → filter by Activities: AddDelegates or RemoveDelegates → filter by Workload: Viva Insights.

7. Escalation Path

If the troubleshooting steps above do not resolve the issue, contact Microsoft Support:

  1. Navigate to admin.microsoft.com → Support → New Service Request
  2. Select Viva Insights as the product
  3. Provide the following information:
    • Tenant ID (from admin.microsoft.com → Settings → Org Settings → Organization Profile)
    • Licence counts (total Copilot and Viva Insights licences assigned)
    • Error messages (screenshot or exact text, including correlation IDs if available)
    • Steps already taken from this troubleshooting guide
