## Ten Most Important Findings

1. **P0 licensing error:** Ten pages state that 30 Microsoft 365 Copilot plus 20 Viva Insights licenses do not qualify. Current Learn explicitly gives that combination as a qualifying 50 Viva Insights service-plan total.
2. **P0 security-boundary overclaim:** Multi-agency pages say partitions create “hard data isolation” and Agency A cannot see Agency B “under any circumstance.” Learn defines partitions as analyst row/column query workspaces, not an enforceable cross-surface or cryptographic tenant boundary.
3. **P1 role/access errors:** A troubleshooting scenario says Global, Insights, and AI Administrator roles all grant tenant-wide dashboard access; another page says Insights Administrator includes analyst capability. Current Learn says roles are independent/non-cumulative and neither statement is true.
4. **P1 Copilot Chat reporting is outdated:** Seven HTML pages and several PDFs say unlicensed Copilot Chat usage is not in admin-center reports. A dedicated Microsoft 365 Copilot Chat usage report now provides 7/30/90/180-day, per-user, app, prompt, and export data.
5. **P1 export procedure misstates data and access:** The guide calls dashboard exports screen-matching aggregate data including Assisted Hours. Learn says this is a preview, requires 50 licenses and global access, exports de-identified **user-level** data, cannot be customized, and does not currently include Assisted Hours.
6. **P1 DLP guidance is unsafe:** Prompt SIT blocking is labeled GA but remains preview; web-search blocking and prompt blocking are conflated; the assertion that blocked prompts are excluded from analytics is not documented by Learn.
7. **P1 org-data guidance blends conflicting transition states:** Current official pages disagree about whether Viva web-app uploads remain available. The guide presents a passed “mid-2026” deadline and an unsupported dummy-row reversal instead of tenant Message Center validation and documented delete/update paths.
8. **P1 residency/compliance overclaim:** “Same geographic region as your Microsoft 365 tenant” omits provisioning-date limits and grandfathered hosting. Learn documents local residency only for specified tenant default mailbox regions and provisioning dates.
9. **P1 Agent Dashboard status is unresolved:** The feature remains public preview and unavailable in GCC/national clouds. Learn also conflicts on whether 50 Viva Insights-only licenses qualify versus requiring 50 Microsoft 365 Copilot licenses plus agent activity.
10. **P1 taxonomy and capability gap:** “Five analytics tiers” is an author framework, not Microsoft terminology, and now omits the officially documented Consumption Dashboard and current programmatic/reporting options.

## Production Reliance Decision

**No. The site is not currently safe to rely on as the sole implementation authority for production or regulated deployments.** It remains useful as an orientation and planning aid if every licensing, access, data-source, preview, environment, and multi-agency decision is revalidated against the cited current Learn pages and the target tenant. P0 and P1 items should be corrected before republishing or recommending the hub for implementation.
