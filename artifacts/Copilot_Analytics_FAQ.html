Microsoft 365 Copilot Analytics

Frequently Asked Questions — For Customers & IT Administrators
Licensing & Access
Is a separate Viva Insights licence needed if we already have Microsoft 365 Copilot?
No. Every M365 Copilot licence automatically includes the Viva Insights service plan — covering personal insights, manager/leader insights, and advanced analysis. A separate Viva Insights purchase is only needed if you want to analyse employees who do not have Copilot licences.
Who needs licences — only leaders viewing dashboards, or also the employees whose data we want to analyse?
Both. Employees whose collaboration data you want to analyse must have a Viva Insights licence assigned (included with every Copilot licence). Leaders and managers viewing dashboards also need a Viva Insights licence to access the full Copilot Dashboard and organisation insights. Unlicensed employees who collaborate with licensed users appear as "other internal collaborators" in aggregate but are not directly measured.
What is the minimum number of licences needed for full dashboard features?
At least 50 assigned Copilot or Viva Insights licences (or a combination) are required for group-level filtering, the full Adoption page, and the Impact page. Below 50, only the Readiness page and tenant-level metrics are available.
What happens if we drop below 50 licences?
The dashboard reverts to limited features (tenant-level only, Readiness page). Previously available group-level data is no longer shown.
Can we trial the Dashboard before licensing everyone?
Yes. The basic Copilot Dashboard is available to any M365/O365 tenant with Exchange Online — no paid Viva licence required. But full group filtering and the Impact page require the 50-licence threshold.
Privacy & Data Protection
Can managers see individual employee Copilot usage?
No. All data is aggregated at group level and protected by minimum group size thresholds (default 10, minimum 5), differential privacy, and distribution masking. No individual user behaviour is ever visible in the dashboards.
Can employees opt out of being analysed?
Users can opt out of Viva Insights personal insights via their settings. For advanced analysis and the Copilot Dashboard, the Insights Administrator can configure privacy settings and create exclusion lists to remove specific users from aggregated insights.
What is the minimum group size, and can we change it?
The minimum group size is the smallest number of people for which data will be displayed. Default is 10; the lowest allowed setting is 5. It can be configured by the AI Administrator or Insights Administrator. Groups smaller than this threshold will show "N/A" instead of data.
Data & Reporting
How often does the data refresh?
The Copilot Dashboard refreshes daily, showing a 28-day rolling window with up to a 6-day delay from the current date. Admin Center reports refresh approximately every 72 hours. Newly licensed employees may take an additional 7 days before appearing.
Can we see data by business unit, department, or country?
Yes, but you must upload organizational data (a CSV file with attributes like Business Unit, Department, Country) to enable these filters. Without an org data upload, only the Entra ID "Department" field is available as a filter. Custom attributes can also be added.
Does the Dashboard track Copilot Chat (web) usage?
Yes, if Copilot Chat reporting is enabled in your tenant and you have 50+ Copilot or Viva Insights licences. There is a separate licence filter on the dashboard to switch between Microsoft 365 Copilot and Copilot Chat views.
How is the $72/hour default for Copilot Assisted Value calculated?
The $72/hour is based on data compiled by the U.S. Bureau of Labor Statistics as a broad estimate of loaded labour cost. Customers should adjust this to their own loaded rate for accurate ROI calculations. The rate can be changed in the Impact settings or via the Hours and Value calculator on the dashboard.
Agent Dashboard
Can we use the Agent Dashboard without the Copilot Dashboard?
Both are part of the Viva Insights web app. You cannot enable one without the other being available, but access to each can be controlled separately via Viva Feature Access Management (VFAM) policies.
Why can my delegates see the Copilot Dashboard but not the Agent Dashboard?
Managers cannot access a team-level view in the Agent Dashboard. Delegates must be given "Your company" scope (not "Your group") to access the Agent Dashboard. Update their delegation scope in Viva Insights → Settings → Delegation.
Scope & Broader Productivity
We want broader productivity metrics (meeting time, focus time, after-hours) — not just Copilot reporting. Is that available?
Yes, but through a separate workstream. Copilot Analytics covers Copilot-specific adoption, impact, and ROI. Broader Viva Insights productivity metrics (meeting time, focus time, after-hours, collaboration networks) use the same Viva Insights licence and analyst role but are configured through the Advanced Analysis workbench with different queries and templates. The organizational data and privacy settings you configure for Copilot Analytics carry over to the broader Viva Insights analysis.

Common Misconceptions

Copilot data is only available in the global partition, not in individual partitions — true?
Incorrect. The interactive Copilot Dashboard is tenant-level only and not partition-scoped. However, all 100+ Copilot metrics (adoption, impact, assisted hours, per-app actions) are available in individual partitions via custom person queries and Power BI template queries. Partitioned analysts get scoped Copilot data — they just access it through the analyst workbench, not the Dashboard.
Can I use Power BI to bypass the minimum group size and see data for small teams?
No. Power BI template reports from Viva Insights enforce the same minimum group size threshold as the Dashboard. The privacy control is applied at the data aggregation layer, not the UI layer. Moving to Power BI does not circumvent it. For finer granularity, you can export raw query results as CSV and build custom PBI visuals — but you are then responsible for enforcing privacy thresholds per your organisation's data handling policy.
Security groups can't be used for dashboard access — I can only add individuals?
Incorrect. Microsoft Learn confirms that dashboard access supports Entra security groups, Microsoft 365 groups, and mail-enabled security groups. If group-based access doesn't work, check: (a) all group members have a Viva Insights service plan, (b) the group is one of the supported types (not a distribution-only group), (c) 24 hours have elapsed since assignment. See Learn reference.
Published reports can only be shared with individual users, not groups?
Partially incorrect. For built-in Viva Insights PBI reports, “Unfiltered” targets dashboard-access holders and “Filtered” targets enabled managers — these are predefined audiences. However, for custom report publishing, you can target specific users or groups (security groups and mail-enabled security groups are supported). See Learn reference.
On-premises Active Directory extension attributes automatically flow to Viva Insights through Entra sync?
No. Only three attributes from Entra ID sync to Viva Insights: PersonId (UPN), ManagerId, and Organization (from the Department field). Extension attributes (extensionAttribute1-15), Company, Division, JobTitle, City, and Country are not synced — even if they are populated in Entra via AD Connect. To use these attributes in Viva Insights, upload them via CSV.
