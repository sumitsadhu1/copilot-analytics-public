Both formats produce the same validation. The modular flow splits the analysis into three sequential steps to fit within M365 Copilot input limits. Always attach or paste the customer’s org data file (Excel or CSV) with each prompt.

Using a Non-Microsoft AI Tool?

The org data file contains real personnel data. Pasting or attaching it to tools outside your Microsoft 365 boundary — such as ChatGPT, Claude, or GitHub Copilot — moves that data beyond your Microsoft 365 data-protection and compliance commitments. Check your organisation’s data-handling policy first.

File Schema Does Not Prove the Tenant Upload Path

Microsoft_* versus PersonId/ManagerId headers identify a file schema, not whether the tenant currently supports add/edit uploads in a particular portal. Microsoft Learn sources conflict about Viva web-app upload. The tenant’s latest Message Center notice and actual controls in both admin surfaces are the operational authority. The prompts below must report the path as unconfirmed when that evidence is not supplied.

Sources: Manage settings for the Microsoft Copilot Dashboard, Agent Dashboard, and Viva Insights web app (Microsoft Learn, updated 20 March 2026); Use Microsoft Entra plus .csv files for parallel data uploads (visible date 30 June 2025; content refreshed 7 July 2026); and Upload and maintain data through the Microsoft 365 admin center — Migration and timeline (5 December 2025).

1. Modular Flow — 3 Prompts

How the Flow Works

Run these three prompts in order, attaching the same org data file to each. Each prompt produces a self-contained report you can save. The prompts guide you to the next step automatically.

Prompt 1: Data Readiness  ➔  Prompt 2: Dashboard Impact  ➔  Prompt 3: Remediation Plan

First time? Follow these steps:

  1. Open your AI tool — the Microsoft 365 Copilot Analyst Agent or Copilot Chat (recommended).
  2. Have the org data file (Excel or CSV) saved and ready on your computer.
  3. Find Prompt 1 below and click its Copy button, then paste it into the AI tool.
  4. In the same message, attach the org data file, then send it.
  5. Read the report it produces and save it (for example, copy it into a Word or text document).
  6. Repeat steps 3–5 for Prompt 2, then Prompt 3 — attaching the same file each time.
  7. Fix the file using the reports’ guidance, then run Prompt 1 again to confirm it passes before uploading.

Prompt 1 of 3 — Data Readiness Assessment

Purpose: Is the file structurally ready for upload? Checks column names, data types, hierarchy integrity, and data quality.

3. Usage Scenarios

Scenario 1: Pre-Upload Validation
A customer sends you their org data CSV before uploading to Viva Insights. Paste the prompt + attach the file. The AI produces a validation report with fixes to apply before upload.
Scenario 2: Dashboard Filter Planning
A customer has multiple hierarchy columns (Groups, Branch, Section, Org Level) and wants help deciding what to map to Organization vs FunctionType vs custom attributes. The prompt’s filter mapping section handles this.
Scenario 3: Post-Upload Troubleshooting
The Copilot Dashboard shows “--” for most groups, or the Organization filter is missing. Paste the prompt + attach the file they uploaded. The AI identifies what went wrong (e.g., empty Organization column, broken manager hierarchy).
Scenario 4: Multi-Agency Data Review
A government or multi-agency customer has complex hierarchy with agencies, divisions, branches, and sections. The prompt helps decide the right mapping for isolation vs default-priority architecture.
