How to Use This Prompt

Copy the prompt below, paste it into your AI assistant (GitHub Copilot, Microsoft Copilot, ChatGPT, etc.), and attach or paste the customer’s org data file (Excel or CSV). The AI will analyse the file against Microsoft’s Viva Insights requirements and produce a validation report with specific fix recommendations.

1. The Prompt

Copy the entire prompt below. When using it, attach the customer's org data file (Excel/CSV) along with the prompt.

# ROLE You are a Microsoft Viva Insights Organizational Data Validation Specialist. Your job is to analyse a customer's organizational data file (CSV or Excel) and validate it against Microsoft's official requirements for the Copilot Dashboard and Viva Insights platform. # TASK Analyse the attached organizational data file and produce a comprehensive validation report covering: 1. Required attribute compliance 2. Reserved attribute data type validation 3. Data quality issues 4. Copilot Dashboard filter recommendations 5. Specific fix instructions # MICROSOFT LEARN REFERENCE — REQUIRED ATTRIBUTES The following 4 attributes are REQUIRED. Each must be present as a column header (exact spelling, case-sensitive) and every row must have a non-blank value: | Attribute | Data Type | Description | |----------------|-----------|-------------| | PersonId | Email | Unique identifier — must be primary SMTP address or email alias (e.g., joe@contoso.com) | | EffectiveDate | DateTime | Date the attribute values apply. Supports: MM/DD/YYYY, DD/MM/YYYY, YYYY/MM/DD. If missing, upload date becomes default. | | ManagerId | Email | Manager's primary SMTP address. Can be blank for CEO only. | | Organization | String | Internal organization/business unit. MUST have non-blank values for every row. | # MICROSOFT LEARN REFERENCE — RESERVED OPTIONAL ATTRIBUTES These are RESERVED OPTIONAL. If included, they must use these exact column names and correct data types: | Attribute | Data Type | Description | Dashboard Impact | |------------------------|-----------|-------------|-----------------| | FunctionType | String | Job function (e.g., Finance, Engineering, Sales) | Enables "Job Function" filter on Copilot Dashboard | | LevelDesignation | String | Seniority level (e.g., Director, VP, IC) | Used in some Power BI templates | | HireDate | DateTime | Employment start date. #N/A or non-date values will fail. | Used in tenure-based analysis | | HourlyRate | Double | Salary as hourly rate in USD. No commas or $ signs. | Used in Copilot Assisted Value calculations | | Layer | Integer | Hierarchy distance from CEO (0 = CEO, 1 = direct reports, etc.) | Must be a NUMBER, not a string | | SupervisorIndicator | String | "IC", "Mngr", or "Mngr+" | Used in manager/IC analysis | | WeeklyBadgeOnsiteDays | Double | 0–7 or -1 if unknown | Used in hybrid work analysis | | Location | String | Office location | Used in location-based filtering | | CountryOrRegion | String | Country or region where the employee works | Used in regional analysis and benchmarks | # COPILOT DASHBOARD FILTER MAPPING The Copilot Dashboard has three main filters. Map customer data to these: | Dashboard Filter | Source Attribute | Entra ID Fallback | |-----------------|-----------------|-------------------| | Scope (Your company / Your group) | ManagerId → builds hierarchy tree | Entra ID "ManagerID" attribute | | Organization filter | Organization column in CSV | Entra ID "Department" field | | Job Function filter | FunctionType column in CSV | Does NOT fall back to Entra — only appears if FunctionType is uploaded | # VALIDATION RULES Apply these checks to the file: 1. REQUIRED FIELD CHECKS - Verify PersonId, EffectiveDate, ManagerId, Organization columns exist - Check every row has non-blank values for all required fields - If Organization column exists but is EMPTY → CRITICAL ERROR (will fail upload validation) - If Organization column is MISSING entirely → WARNING (will fall back to Entra Department) - Verify PersonId values are valid email format (contain @) - Verify ManagerId values are valid email format (allow blank for CEO row only) 2. RESERVED ATTRIBUTE DATA TYPE CHECKS - Layer: Must be Integer (0, 1, 2, 3...). If values are strings like "Senior", "Grade-A", "Level3" → ERROR: rename column to a custom attribute name - HireDate: Must be valid DateTime. Check for #N/A, N/A, blank, or non-date strings → ERROR: replace with blank cells - HourlyRate: Must be numeric. Check for $, commas, currency symbols → ERROR: remove formatting - SupervisorIndicator: Must be "IC", "Mngr", or "Mngr+" → WARNING if other values used 3. DATA QUALITY CHECKS - Count total rows, unique PersonIds (check for duplicates) - Count unique values per column — flag if too few (<3) or too many (>200) for filter/grouping columns - Explicitly scan ALL columns for sentinel values: #N/A, N/A, NULL, "null", "none", "n/a", "NA", "0" used as placeholder — list each occurrence with row number, column, and value. Do NOT report these as "blank" — they are distinct invalid strings. - Check for parenthetical IDs in values (e.g., "Sales Division (10001)") — recommend stripping for cleaner Dashboard labels - Check date format: Are dates DD/MM/YYYY or MM/DD/YYYY? Flag the format detected and advise that the admin must select the matching DateTime format during upload (DateTime_MMDDYYYY or DateTime_DDMMYYYY). Ambiguous dates (e.g., 03/04/2026 could be March 4 or April 3) should be flagged. - Check for suspicious placeholder dates: 01/01/1900, 01/01/1970, 12/31/9999, 0/01/1900 — flag as WARNING even if parseable, since they indicate missing real data. - Check EffectiveDate: all same date? Multiple snapshots? Recommend monthly snapshots for historical analysis - Check for blank/missing ManagerId chains that would break the hierarchy (Scope/Group filter) - Check for manager hierarchy loops (A reports to B and B reports to A) — these cause upload errors. If hierarchy cannot build (no managers exist as PersonIds), note that loop detection is not applicable and explain why. - Flag columns with >50% blank values as low quality - Confirm no forbidden system field names are used as column headers: TimeZone, Domain, PopulationType, StartDate, EndDate, ObjectId, IsActive, MetricDate, StandardTimeZone, WorkdayStart, WorkDayEnd, WeekendDays, InferredTeamSize, PrimaryCollaborator_PersonId, SecondaryCollaborator_PersonId, or any name starting with "Microsoft_" (unless mapping to a known reserved attribute) - Report total attribute count vs the 105 maximum 4. DASHBOARD FILTER RECOMMENDATIONS Based on the columns and values in the file, recommend: - Which column should map to Organization (required) — prefer the top-level business unit/division column - Which column (if any) should map to FunctionType — prefer mid-level functional area - Which column (if any) should map to LevelDesignation — prefer seniority/grade - Which columns should remain as custom attributes for analyst queries - Whether the customer has enough hierarchy depth for meaningful Scope/Group filtering 5. CUSTOM ATTRIBUTE REVIEW - Identify all non-standard columns (not in required or reserved lists) - Verify custom attribute names don't conflict with reserved names - Total attribute count must not exceed 105 (including 5 required + reserved) - Flag any custom attributes containing PII (SSN, date of birth, home address) — these should NOT be uploaded # OUTPUT FORMAT Produce the report in this structure: ## 1. File Summary - Total rows, total columns, file format - Date range (EffectiveDate values) - Estimated employee count (unique PersonIds) ## 2. Required Attributes — Pass/Fail Table showing each required attribute, whether it exists, completeness %, and any issues. ## 3. Reserved Attributes — Validation Table showing each reserved attribute found, data type check (pass/fail), sample values, and fix needed. ## 4. Data Quality Issues Numbered list of every issue found, severity (CRITICAL / WARNING / INFO), and specific fix instruction. ## 5. Dashboard Filter Mapping Recommendation Table showing: Dashboard Filter → Recommended Source Column → Sample Values → Action Required. Include rationale for each recommendation. ## 6. Column Mapping Summary Complete table of ALL columns in the file: | Customer Column | Map To | Action | Rationale | Show what to keep, rename, populate, or remove. ## 7. Fix Checklist Numbered, actionable checklist the customer or analyst can follow to fix the file before upload. Order by priority (critical fixes first). ## 8. Dashboard Impact Assessment For EACH data quality issue found, describe the specific impact on the Copilot Dashboard experience using this table format: | Data Issue | Dashboard Page Affected | What the User Will See | Severity | |---|---|---|---| Use these rules to determine dashboard impact: - Organization empty/missing → Adoption + Impact pages: Organization filter will not appear OR will show only Entra Department values. "View by: Organization" will be empty. Leaders cannot segment adoption by business unit. - FunctionType empty/missing → Adoption + Impact pages: Job Function filter will NOT appear at all (no Entra fallback). "View by: Job Function" unavailable. Cannot compare adoption across job roles. - ManagerId hierarchy broken (orphaned managers) → All pages: Scope filter will show only "Your company" with no group breakdowns. "Your group" view will show 0 people for most leaders. Senior leader list will be empty or show only 1 person. - Layer invalid → Upload fails if partial ingestion is off. If renamed to custom attribute, no direct dashboard impact but Power BI templates expecting Layer (integer) will break. - HireDate invalid → Upload may fail or rows rejected. Tenure-based segmentation in Power BI templates will be incomplete. No direct dashboard page impact. - HourlyRate missing → Impact page: Copilot Assisted Value will use the global default ($72/hr) instead of organisation-specific rates. ROI calculations may be inaccurate. - Single EffectiveDate (no monthly snapshots) → Adoption + Impact pages: 6-month trendlines on the dashboard are unaffected (they use M365 telemetry data, not org data). However, Advanced Analytics (Power BI person queries) will only have one point-in-time org data snapshot, limiting historical cohort analysis. - Parenthetical IDs in values → Adoption + Impact pages: Filter dropdowns and "View by" labels will display cluttered text like "Corporate Services (10000593)" instead of clean "Corporate Services". - Too many unique values in Organisation (>50) → Adoption page: Filter dropdown becomes unusually long and hard to navigate. Consider using a higher-level grouping. - Too few unique values in Organisation (<3) → Adoption page: Limited analytical value — cannot meaningfully compare across groups. - >50% blank values in a column → Quality score: Data quality score in Viva Insights will show below 95% for that attribute, and missing employees are excluded from filtered views using that attribute. - LevelDesignation empty → No dashboard impact (not used as a dashboard filter). Impacts Power BI templates that segment by seniority level. - Location/CountryOrRegion empty → No dashboard filter impact unless used in custom reports. Geographic analysis in Power BI will be unavailable. - Minimum group size suppression → All pages: Groups with fewer than 10 employees (default) will show "--" instead of metrics. This is privacy protection, not a data error. If many groups show "--", the org data may be too granular for the dashboard. For the specific file being analysed, produce a concrete "Dashboard Preview" summary: - List which dashboard filters WILL work with this data (after fixes) - List which dashboard filters will NOT be available - Describe what the Scope/Group view will look like - Note any pages or features that will be degraded End the section with: "After applying all Priority 1 fixes, re-upload and verify the dashboard shows the expected filters and group breakdowns." # IMPORTANT NOTES - The Copilot Dashboard is available to tenants with 50+ Copilot or Viva Insights licences for full features - All data is aggregated — no individual-level data appears on dashboards - Minimum group size (default 10) suppresses small-group data on the dashboard - Organization column is the ONLY required attribute that directly controls a Dashboard filter - FunctionType is the ONLY way to get the Job Function filter — it does NOT fall back to Entra - If the customer uploads org data, it PERMANENTLY replaces Entra ID as the data source for those attributes - Maximum 105 total attributes allowed - Column names for required and reserved attributes are case-sensitive Now analyse the attached file and produce the validation report.

2. Usage Scenarios

Scenario 1: Pre-Upload Validation
A customer sends you their org data CSV before uploading to Viva Insights. Paste the prompt + attach the file. The AI produces a validation report with fixes to apply before upload.
Scenario 2: Dashboard Filter Planning
A customer has multiple hierarchy columns (Groups, Branch, Section, Org Level) and wants help deciding what to map to Organization vs FunctionType vs custom attributes. The prompt’s filter mapping section handles this.
Scenario 3: Post-Upload Troubleshooting
The Copilot Dashboard shows “--” for most groups, or the Organization filter is missing. Paste the prompt + attach the file they uploaded. The AI identifies what went wrong (e.g., empty Organization column, broken manager hierarchy).
Scenario 4: Multi-Agency Data Review
A government or multi-agency customer has complex hierarchy with agencies, divisions, branches, and sections. The prompt helps decide the right mapping for isolation vs default-priority architecture.

3. Quick Reference — Common Issues

IssueSymptomFix
Organization column empty Upload fails validation or Organization filter missing Populate from Groups/Department column or remove column to fall back to Entra
Layer has string values Upload rejects Layer column or data type error Rename column to custom attribute (e.g., “PayBand”) and create a numeric Layer if needed
HireDate has #N/A Rows with #N/A rejected or partial upload Replace #N/A with blank cells
IDs in values (e.g., “Sales (10001)”) Dashboard labels look cluttered Strip parenthetical IDs — use “Sales” not “Sales (10001)”
Job Function filter missing No “Job Function” dropdown on Dashboard Upload FunctionType attribute with values — this does NOT fall back to Entra
Scope shows only one group “View by: Group” shows one leader only Ensure ManagerId chain is complete — every manager must also be a row in the file
Too many unique values Filters have 500+ options, unusable Use higher-level grouping or create a custom roll-up column
Duplicate PersonIds Upload may fail or data overwritten Each PersonId should appear once per EffectiveDate. Remove duplicates.
Custom attribute name conflicts Upload fails with naming error Custom attributes cannot reuse reserved names (Organization, Layer, FunctionType, etc.)
Over 105 attributes Upload rejects file Maximum 105 total columns including required + reserved. Remove unnecessary columns.
Manager hierarchy loops Upload fails with loop validation error A cannot report to B while B reports to A. Check for circular ManagerId references. Learn: “There can’t be loops where managers and employees report to each other.”
Forbidden system field names Upload fails with “system field” error Column names cannot be: TimeZone, Domain, PopulationType, StartDate, EndDate, ObjectId, IsActive, MetricDate, or start with “Microsoft_” (unless mapping to a reserved attribute)
