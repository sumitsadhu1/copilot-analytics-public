Multi-Agency Reporting Pattern Decision
Evaluate two author-created patterns for a shared tenant
THIS GUIDE’S AUTHOR-CREATED ARCHITECTURE PATTERNS

Default Priority and Restricted Reporting (formerly labelled “Isolation” and retained in existing URLs) are names created by this guide. They are not Microsoft product, security, compliance, or licensing terms. Microsoft documents the component controls; it does not document either combination as an end-to-end security boundary.

This guide offers two reporting patterns for a shared Microsoft 365 tenant: Default Priority makes an agency group the leader’s expected starting scope while accepting broader access; Restricted Reporting combines partition-scoped analyst queries with separate controls for Dashboard, Agent, export, delegation, managers, privileged actors, and Power BI. Neither pattern creates cryptographic, tenant, or independently certified security isolation.

Supported Environment and Licensing Gate

Check the exact cloud, assigned service plans, tenant UI, Message Center, and every required surface before choosing a path. Microsoft’s current matrix says GCC does not support Agent Dashboard, Copilot metrics export, or Copilot Power BI reports; the Agent Dashboard is preview and unavailable in national/regional clouds. For advanced access, Microsoft Learn describes the 50-plan threshold differently: the environment article explicitly gives 30 Copilot + 20 Viva Insights as 50 qualifying Viva Insights service plans, while the Dashboard article describes 50 Copilot or 50 Viva Insights. Do not claim mixed licences fail. For Agent Dashboard, use the conservative feature-specific prerequisite of 50 Microsoft 365 Copilot licences plus agent activity and validate in the tenant. Escalate a documented mismatch to Microsoft Support.
