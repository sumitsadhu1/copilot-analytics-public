Copilot Lifecycle & Billing Playbook

Operational Guide for Licence Management, Billing Models & Analytics Data Impact

TL;DR — Key Facts
  • Every M365 Copilot licence includes Viva Insights — no separate purchase needed (Section 2)
  • 50+ licences required for full dashboard features; dropping below reverts to limited view (Section 3)
  • Copilot Credits do not roll over — monitor weekly and set alerts (Section 4)
  • When credits are exhausted, agents are blocked immediately — see emergency procedure (Section 4)
  • Licence changes take up to 7 days to appear in dashboards (Section 3)

1. Purpose

This playbook is an operational guide for managing Microsoft Copilot licence lifecycle events and billing models. It covers what happens to analytics data when licences are assigned, removed, or reassigned — and provides step-by-step billing configuration procedures.

Use this document alongside the main Copilot Analytics guide to ensure licence changes do not create gaps in your analytics reporting or unexpected billing charges.

2. Licensing Models Overview

Licensing Model Billing Type Key Features
Microsoft 365 Copilot (add-on licence) Per-user, monthly/annual Includes Viva Insights service plan; Copilot in all M365 apps (Word, Excel, PowerPoint, Outlook, Teams, etc.); zero-rated agent usage in M365 Copilot, Teams, and SharePoint
Microsoft 365 Copilot Chat Included in M365 subscription Web-based only (copilot.microsoft.com); metered charges for shared tenant data (SharePoint, Graph) access via Copilot Credits
Copilot Studio Standalone Prepaid Copilot Credits subscription Build and publish agents via Copilot Studio; prepaid credit pool; agent responses consume credits
Copilot Studio Pay-As-You-Go Azure subscription billing Monthly actual usage billed to an Azure subscription; no upfront commitment; suitable for variable workloads
Copilot Credits Pre-Purchase Plan One-year prepaid Bulk credit purchase at a discount; usable across eligible products (Copilot Studio, M365 Copilot Chat tenant data, etc.)

3. Lifecycle Events — Impact on Analytics Data

The following table describes common licence lifecycle events and their impact on Copilot analytics dashboards and Viva Insights data.

Event Impact on Copilot Dashboard Impact on Advanced Analytics (Viva Insights)
New licence assigned User data appears in the dashboard within 7 days. Employee joins the measured population automatically. User is added to the measured population. Activity data begins accumulating for query inclusion.
Licence removed Employees with only a Copilot licence are removed from analysis after licence removal. Viva Insights service plan removal removes the user from the measured population. Historical data in existing query results is retained.
Licence reassigned to a different user Previous user's historical data remains in the measured period. New user data appears within 7 days. Previous user retains data in completed queries. New user begins accumulating data from assignment date.
Employee leaves organisation If Copilot licence was removed before departure, they are excluded going forward. Exclusion lists can also be used to remove them. Historical data in existing queries persists. User is removed from the active measured population upon account deletion.
Agency/BU merge Dashboard reflects updated org data after the CSV is re-uploaded with new organisational structure. Update org data CSV to reflect the new org structure. Partition boundaries may need restructuring (see Multi-Agency Isolation Architecture).
Scale-up (adding 50+ more licences) Dashboard automatically includes new users within 7 days. No configuration change is needed. New users added to measured population automatically. Queries will include them on the next run.
Scale-down below 50 licences Dashboard reverts to limited features: only Readiness page and tenant-level adoption visible. Group-level filtering, Impact page, and Sentiment page become unavailable. Data is retained but inaccessible — restoring to 50+ licences re-enables full features without data loss. Advanced Analytics workbench remains accessible if existing Viva Insights licences total ≥50. If total drops below 50, the analyst experience shows “Viva Insights is below the minimum assigned licence requirement”. Existing query results and Power BI reports remain intact but new queries cannot be created.
Key Point: The 7-day data appearance window is a processing delay, not a configuration requirement. No manual action is needed — the system detects new licence assignments automatically. For dashboard enablement steps, see the Implementation Guide Section 5.

4. Copilot Credits — Understanding Consumption

Copilot Credits are the consumption currency for Copilot Studio agent interactions and certain M365 Copilot Chat scenarios. Understanding what costs credits — and what doesn't — is essential for budget management.

What Costs Credits

  • Agent responses using Copilot Studio capabilities beyond zero-rated usage
  • Tenant data grounding (SharePoint, Graph) in M365 Copilot Chat (web-based)
  • Custom agent actions, plugins, and connector invocations
  • Generative answers using non-M365 knowledge sources

Zero-Rated (No Credits Required)

  • Classic (deterministic) answers in Copilot Studio agents
  • Generative answers using Microsoft Graph tenant grounding when used by M365 Copilot licensed users in M365 apps (Word, Excel, Teams, etc.)
  • Standard Copilot-in-app interactions for licensed M365 Copilot users

Credit Mechanics

  • Credits reset monthly — there is no rollover of unused credits to the next month.
  • Overage behaviour: when credits are exhausted, model runs are blocked until more credits are purchased or allocated.
  • Credits are pooled at the tenant level for prepaid plans, or billed per-environment for pay-as-you-go.
Warning: Credits do not roll over. Monitor consumption weekly to avoid unexpected service interruptions at month-end.

Emergency: Credits Exhausted — Agents Blocked

If agents stop responding and users see errors, follow this resolution path:

  1. Confirm the cause: Navigate to Power Platform Admin Center → Billing → Licenses. Check if the credit balance shows 0 or “Exhausted”.
  2. Immediate fix — Add credits:
    • Pay-as-you-go: Verify the Azure subscription linked to the billing policy is active and has available budget. If the subscription spending limit was hit, increase it in Azure Portal → Cost Management.
    • Prepaid: Purchase additional Copilot Credits via your Microsoft account team or the M365 Admin Center → Billing → Purchase Services. Credits are applied within 1–2 hours.
  3. Prevent recurrence — Set up alerts:
    • In PPAC: Billing → select the billing policy → configure usage notifications at 50%, 75%, and 90% thresholds.
    • In Azure (for pay-as-you-go): Cost Management → Budgets → create a budget with action group email alerts.
  4. Review consumption: Use the Copilot Credits Activity Report in Admin Center to identify which agents consumed the most credits and whether any can be optimised.

Reference: Copilot Studio Message and Credit Management

5. PPAC Billing Configuration

Follow these steps to configure and monitor billing in the Power Platform Admin Center:

  1. Navigate to admin.powerplatform.microsoft.com and sign in as a Power Platform Administrator.
  2. Go to Licensing → Copilot Studio in the left navigation pane.
  3. Review the Summary view:
    • Prepaid capacity — total credits purchased, credits consumed, credits remaining
    • Pay-as-you-go — active billing plans linked to Azure subscriptions
    • Consumption trend — daily/weekly/monthly usage graphs
  4. View Capacity consumption by product to see breakdown across Copilot Studio, M365 Copilot Chat, and other eligible products.
  5. To allocate prepaid credits to specific environments:
    • Select Allocate capacity
    • Choose the target environment
    • Enter the number of credits to allocate
    • Confirm the allocation
  6. To create a pay-as-you-go billing policy:
    • Select New billing policy
    • Link to an Azure subscription and resource group
    • Assign the billing policy to one or more environments
    • Set spending limits if desired
  7. Use the Agent Usage Estimator tool to forecast credit consumption for planned agents: microsoft.github.io/copilot-studio-estimator

Reference: Manage Copilot Studio Messages Capacity

6. Admin Center Billing Reports

The Microsoft 365 Admin Center provides a Copilot Credits report that gives detailed visibility into credit consumption across your organisation.

Accessing the Report

  1. Sign in to admin.microsoft.com.
  2. Navigate to Reports → Usage → Copilot Credits.
  3. Review the report components:
    • Total credits used: Aggregate consumption for the billing period
    • Time series: Cumulative and daily credit consumption charts
    • Per-user breakdown: Credits consumed by individual users
    • Per-agent breakdown: Credits consumed by each published agent
    • Per-billing-policy: Consumption grouped by billing policy
Tip: Export the per-user breakdown to CSV and join it with your org data file to produce agency-level or cost-centre-level chargeback reports.

Reference: Copilot Credits Activity Report

7. Multi-Agency Chargeback Model

For organisations with multiple agencies, departments, or cost centres, a chargeback model distributes Copilot costs to the consuming business units.

Option A: Org Data–Based Attribution

  1. Upload org data with an Agency (or CostCentre) attribute via Viva Insights or PPAC.
  2. Export the Copilot Credits report filtered by user from the M365 Admin Center.
  3. Join user-level consumption data with org data to map each user's credit usage to their agency.
  4. Aggregate by agency to produce the chargeback summary.

Option B: Separate Billing Policies

  1. Create separate billing policies in PPAC — one per agency.
  2. Link each billing policy to a different Azure subscription or cost centre.
  3. Assign environments to the appropriate billing policy based on the owning agency.
  4. Each agency's Azure subscription is billed directly for their consumption.

Quarterly Reconciliation Process

  • At the end of each quarter, export consumption data from both PPAC and M365 Admin Center.
  • Reconcile prepaid credit consumption against allocations per agency.
  • Reconcile pay-as-you-go charges against Azure invoices.
  • Review and adjust allocations based on actual usage trends.
  • Produce a chargeback summary report for finance teams.
Note: Option B (separate billing policies) provides cleaner cost separation but requires more administrative overhead. Option A is simpler to maintain for organisations with fewer than 10 agencies.
Agent Billing Governance: For detailed credit mechanics, safe scaling patterns, irreversible billing decisions, and the billing boundary design framework, see the Copilot & Agent Governance project — specifically the Agent Billing & Credit Governance guide.

8. Operational Checklist

Monthly Tasks

  • Review licence utilisation in the M365 Admin Center — identify unused licences
  • Review credit consumption in PPAC — check against budget thresholds
  • Reallocate unused licences to requesting users or business units
  • Check for blocked agent runs due to credit exhaustion
  • Review new licence assignments and confirm dashboard data appearance

Quarterly Tasks

  • Audit exclusion lists in Viva Insights — remove departed employees, add new exclusions
  • Review billing policies — ensure environments are assigned to correct policies
  • Reconcile chargeback — produce agency-level cost reports (see Multi-Agency Isolation Architecture for partition-based chargeback setup)
  • Assess scale needs — forecast licence and credit requirements for next quarter
  • Review lifecycle events — confirm org data CSV reflects current organisational structure
  • Document any licence model changes (e.g., shift from prepaid to pay-as-you-go)
Best Practice: Create calendar reminders for these tasks and assign a named owner to each checklist item. Governance lapses most often occur when tasks lack clear ownership.
