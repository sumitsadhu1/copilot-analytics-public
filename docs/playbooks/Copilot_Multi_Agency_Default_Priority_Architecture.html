TL;DR — Agency-Default Views in Under a Day

Leverage native Copilot Dashboard scoping and delegation so each agency leader sees their own data by default — no lockdowns, no partitions required.

  1. Verify manager hierarchy — ensure each agency CEO is the top-level manager for their agency in Entra ID (Section 4.1)
  2. Enable AutoCxO & Delegation — agency CEOs auto-identified, then delegate down to VPs (Phase 1)
  3. Enrich org data (optional) — CSV or Entra attributes for department/location filtering within each agency view (Section 4.2)
  4. Configure Publish Reports — enable manager settings so filtered reports reach agency leaders automatically (Phase 3)
  5. Create partitions (optional) — for analysts who need focused query access within a single agency (Phase 4)

Time estimate: 2–4 hours for core setup (Phases 0–2). Add half a day if using partitions or CSV enrichment.

1. Executive Summary

This document provides a solution architecture for deploying Microsoft Copilot Dashboard and Copilot Analytics across a single Microsoft 365 tenant that hosts multiple agencies, where each agency wants to see its own Copilot adoption and impact data by default, but does not require hard data isolation between agencies.

Key Difference from the Isolation Architecture:
The Isolation Architecture disables native dashboard features (AutoCxO, Delegation, Agent Dashboard) to prevent cross-agency visibility. This guide embraces those features — they naturally scope data to the viewer’s management hierarchy, which is exactly the “agency-default” experience.
Recommended Approach: Native Scoping + Optional Partitions
  1. Copilot Dashboard with “Your group” default — agency leaders auto-identified; see their own hierarchy’s data by default.
  2. Dashboard Delegation — CEOs delegate down to VPs and directors for cascading agency-scoped views.
  3. Publish Reports — filtered by manager hierarchy for automatic agency-scoped delivery.
  4. Partitions (optional) — for analyst-level deep dives scoped to a single agency; analysts may access multiple partitions if collaboration requires it.

How “Your Group” Scoping Works

The Copilot Dashboard identifies senior leaders using the Entra ID manager hierarchy (Auto CxO Identification). Leaders automatically see two scopes:

ScopeWhat It ShowsDefault?
“Your group”Copilot data for all employees in the leader’s management chainYes — lands on this scope by default when opening the dashboard
“Your company”Tenant-wide Copilot data across all agenciesNo — available via toggle, but not the default view

When each agency CEO is the top of their hierarchy, “Your group” = their entire agency. The dashboard opens to this view by default, immediately satisfying the “see my agency data first” requirement without any lockdown.

What about “Your company”?
Agency leaders can toggle to “Your company” and see aggregate data across all agencies. In this architecture, that is explicitly acceptable (FR-06). If it is not acceptable, use the Isolation Architecture instead.

4.1 Manager Hierarchy Setup

The entire default-priority architecture depends on a clean manager hierarchy in Entra ID. Each agency must form a self-contained tree.

Common Pitfall: In some shared tenants, a “super admin” or “Secretary General” is set as the manager of all agency CEOs. This creates a single tree where that person’s “Your group” covers the entire tenant. Solution: Leave the Manager field blank for all agency CEOs, or have them all report to a shared-services administrative account that is not a real user (and therefore will not access the dashboard).

5. Implementation Runbook

0 Phase 0: Pre-Flight Verification

Audit the current state before configuring anything.

  • Verify manager hierarchy: Confirm each agency CEO has no cross-agency manager relationship. Use Entra portal → Users → select each CEO → check Manager field. Run the PowerShell audit in Appendix A for bulk verification.
  • Confirm licence counts: Verify 50+ Copilot licences are assigned. M365 Admin Center → Billing → Licences → Microsoft 365 Copilot.
  • Identify agency leaders: List each agency’s CEO and their direct reports (VPs). These will be the primary dashboard consumers and delegation targets.
  • Review current VFAM state: Admin Center → Settings → Viva → Viva Insights → Turn Advanced Insights on or off. Document which features are currently enabled or disabled.
  • Check existing dashboard access: Admin Center → Settings → Viva → Viva Insights → Manage user access for Copilot Dashboard. Note who currently has access.

1 Phase 1: Enable Native Features

Ensure all dashboard access features are enabled. This is the opposite of the Isolation Architecture’s “lockdown” phase.

1.1 — Enable Auto CxO Identification

Admin Center → Settings → Viva → Viva Insights → Turn Advanced Insights on or off → Auto CxO Identification → ensure set to On for Everyone (or verify no disable policy exists).

This automatically identifies agency CEOs (and other senior leaders) based on the Entra ID hierarchy and grants them Copilot Dashboard access with the “Your group” scope.

1.2 — Enable Dashboard Delegation

Same page → Copilot Dashboard Delegation → ensure set to On for Everyone.

This allows agency CEOs to delegate their dashboard view to their direct reports (VPs, directors). Delegates see the same scope as their delegator — the agency’s data.

1.3 — Enable Agent Dashboard

Same page → Agent Dashboard → ensure set to On for Everyone.

1.4 — Metrics Export (optional)

If you want agency leaders to export Copilot metrics, leave CopilotMetricsReport enabled. If you want a slightly more controlled experience, disable it and direct users to the dashboard and published reports instead.

After Phase 1 — Verification:
  1. Wait 1–2 hours for AutoCxO to process.
  2. Check Manage user access for Copilot Dashboard — agency CEOs should appear in the list as auto-identified.
  3. Have an agency CEO navigate to insights.cloud.microsoft and confirm they see the Copilot Dashboard with “Your group” as the default scope.
  4. Confirm the “Your group” employee count matches the agency’s expected headcount.

2 Phase 2: Configure Delegation & Manager Settings

Set up the delegation chain so agency leaders can share their dashboard view with their teams.

2.1 — Agency CEO Delegates to VPs

Each agency CEO should:

  1. Navigate to insights.cloud.microsoft → Copilot Dashboard.
  2. Select the Share or Delegate option (top-right corner of the dashboard).
  3. Add their direct reports (VPs, directors) as delegates.
  4. Delegates will receive access and see the same agency-scoped view (“Your group” inheriting the CEO’s scope).

2.2 — Enable Manager Settings for Publish Reports

Navigate to insights.cloud.microsoft → Admin → Manager settings.

  • Enable managers to receive published reports.
  • Set the minimum team size threshold (this is the same tenant-wide privacy setting).
  • Agency CEOs with team sizes above the threshold will be eligible to receive published reports filtered to their hierarchy.

2.3 — Enrich Organizational Data (Optional)

  • Navigate to Viva Insights → Organizational data → Configure Entra connection → enable ManagerId and Organization checkboxes → Save.
  • If you need custom attributes: prepare and upload a CSV per the format in Appendix B. This enriches dashboard filters but does not change scoping.
After Phase 2 — Verification:
  1. Have a VP who was delegated access navigate to the dashboard. Confirm they see their agency’s data.
  2. If org data was uploaded, verify filters (Department, Location, etc.) appear on the dashboard.

3 Phase 3: Publish Reports

Use Viva Insights Publish Reports to deliver curated analytics to agency leaders automatically.

3.1 — Run Copilot PBI Template Queries

  • Insights analysts (Central IT or agency-level) navigate to insights.cloud.microsoft → Create analysis.
  • Select a Copilot Power BI template (Copilot Adoption, Impact Explorer, Business Impact, etc.).
  • Run the query. If partitions are enabled, run within the appropriate partition. If not, run against the global population.

3.2 — Publish with “Filter by Manager Hierarchy”

  • After the query completes, select Publish report.
  • Choose “Filter report by manager hierarchy”.
  • Each enabled manager (CEO, VP) will receive a version of the report filtered to their team.
  • Set an expiry date (up to 365 days) and republish monthly to keep data current.
Why this works in Default-Priority (but not in Isolation):
In the Isolation Architecture, enabling a manager grants them automatic Copilot Dashboard access, which shows “Your company” data — a problem when cross-agency visibility must be blocked. In this architecture, “Your company” visibility is acceptable (FR-06), so Publish Reports with manager filtering is a simple, effective delivery mechanism.

4 Phase 4: Partitions for Analyst Deep Dives (Optional — Mandatory for CIO Reporting)

If your organisation has designated analysts per agency who need to run custom queries, partitions provide focused workspaces. This phase is optional for agency-level reporting (dashboard views and published reports may suffice), but mandatory if the CIO requires cross-agency comparison reports — the global partition is the only way to query across all agencies. See Section 6.2.

Before Enabling Partitions:
(a) Understand that enabling partitions is irreversible without contacting Microsoft support and will delete all partition configurations if disabled.
(b) Export all existing query results before enabling partitions.
(c) Partitions are only needed when analysts require scoped custom queries. Dashboard and published reports already provide agency-scoped views via the hierarchy.
  • Navigate to insights.cloud.microsoft → Admin → Privacy settings → Partitions → Enable Partitions.
  • Create a partition per agency (e.g., “Agency A”), filtered by Agency attribute (CSV) or Domain (Entra).
  • Assign agency analysts to their primary partition.
  • Key difference from Isolation Architecture: Analysts may be assigned to multiple partitions if cross-agency analysis is needed. There is no rule preventing an analyst from seeing multiple agencies’ data.
  • Assign Central IT analysts to the Global Partition for tenant-wide queries.
After Phase 4 — Verification:
  1. Have an agency analyst sign in and confirm they see their agency’s partition in the partition switcher.
  2. Run a test person query and verify the employee count matches the expected agency headcount.
  3. If a cross-agency analyst exists, confirm they can switch between partitions they are assigned to.

6.1 ROI Analysis per Agency: Capabilities & Gaps

Agency leaders asking “What is Copilot’s ROI for my agency?” will find that some metrics scope naturally to their hierarchy and some do not. Set expectations upfront using the matrix below.

Which ROI Metrics Are Agency-Scoped?

ROI MetricScoped to “Your group” (Agency)?Notes
Copilot Assisted Hours ✅ Yes Impact page shows Assisted Hours for the leader’s management hierarchy. Agency CEOs see their agency’s total.
Copilot Assisted Value ✅ Yes Calculated as Assisted Hours × Hourly Rate. Scoped to “Your group.” However, the hourly rate is a single tenant-wide setting (see limitation below).
Adoption metrics (active users, usage intensity, consistent usage) ✅ Yes Adoption page filters to “Your group” by default.
Sentiment (Glint / Pulse / CSV survey) ❌ No — tenant-level only Sentiment page on the Copilot Dashboard shows tenant-wide data only. Not filterable by “Your group.” Agency leaders see all agencies’ survey responses.
Agent usage (Agent Dashboard) ❌ No — “Your company” only Agent Dashboard has no “Your group” scope. Agency leaders see tenant-wide agent adoption.
Licence cost data ❌ Not in dashboard Licence costs are in M365 Admin Center / billing. Not connected to the Copilot Dashboard. See the Lifecycle & Billing Playbook for cost-side data.
Limitation: Single Tenant-Wide Hourly Rate
Copilot Assisted Value is calculated as Assisted Hours × Hourly Rate. The hourly rate is configured once at insights.cloud.microsoft → Admin → Impact settings and applies to all employees across all agencies. There is no way to set different rates per agency, department, or role.

Impact: If Agency A’s average loaded labour rate is $140/hr and Agency B’s is $65/hr, the Assisted Value figure shown to both CEOs uses the same rate — making one agency’s ROI appear over- or under-stated.

Workaround: Use Power BI templates in partitions (Phase 4) to apply agency-specific hourly rates in the Power BI calculation layer. The raw Copilot Assisted Hours data flows into the PBI template; the analyst replaces the default rate with the agency’s actual loaded labour rate in a DAX measure or parameter. This produces accurate per-agency Assisted Value figures outside the dashboard.

Limitation: No Historical ROI Trending on the Dashboard
The Copilot Dashboard uses a 28-day rolling window and provides no before/after comparison or historical trendline. Agency leaders who need to demonstrate ROI growth over quarters (e.g., “Agency B’s Copilot value grew from $20K/month in Q1 to $35K/month in Q3”) cannot do this from the dashboard alone.

Workaround — Recommended for ROI-focused agencies: Use Power BI templates within partitions (Phase 4). Run the Copilot Impact Explorer or Business Impact template monthly, each time capturing the month’s metrics. Publish each refresh as a new dataset snapshot in the agency’s Power BI workspace. Stack monthly snapshots in a summary report to build a quarter-over-quarter ROI trendline.

Implication: For agencies that need ROI trending, Phase 4 (partitions) is not optional — it becomes a required component of the architecture. Plan the partition and analyst setup accordingly.

Recommendation for ROI-Focused Multi-Agency Deployments
  1. Dashboard for real-time adoption & current-month ROI: Agency CEOs use the Copilot Dashboard (“Your group”) for live adoption metrics and this month’s Assisted Hours/Value.
  2. Partitions + PBI for accurate, historical, agency-calibrated ROI: Treat Phase 4 as mandatory. Analysts run monthly PBI template queries within their agency partition, apply agency-specific hourly rates, and build cumulative ROI reports.
  3. Billing Playbook for cost-side data: Combine the Assisted Value from PBI reports with licence cost data from the Lifecycle & Billing Playbook to produce a net ROI figure per agency.

6.2 Cross-Agency Impact & Adoption Analysis (CIO View)

A CIO or Secretary-General sitting above all agencies needs a consolidated cross-agency view that the Copilot Dashboard does not natively provide. The dashboard shows either “Your group” (one agency) or “Your company” (blended tenant aggregate) — neither provides side-by-side agency comparison.

What the CIO Needs vs. What the Dashboard Provides

CIO NeedDashboard SupportSolution
Cross-agency side-by-side comparison❌ NoneGlobal partition analyst runs Copilot person query grouped by Agency attribute → builds cross-agency PBI report
Feature adoption depth per agency❌ None (per-app breakdown exists but not compared across groups)M365 Copilot Adoption PBI template via global partition, grouped by Agency
Behavioral impact per agency (meeting time, focus time changes)❌ NoneCustom person queries with Copilot + collaboration metrics, grouped by Agency
Consolidated executive roll-up report❌ NoneCustom PBI dashboard: agency scorecards ranked by adoption %, Assisted Hours, feature depth
Cross-agency benchmarking❌ NonePBI with internal percentile calculations per agency
CIO Reporting Architecture
  1. Phase 4 (partitions) is mandatory for CIO reporting — the global partition is the CIO's primary data source. Without partitions, cross-agency comparison is impossible.
  2. Designate a Central IT analyst on the global partition as the CIO’s primary reporting resource. This analyst runs queries across all employees, grouped by the Agency org data attribute.
  3. Recommended PBI executive dashboard structure:
    • Page 1 — Agency Scorecard: Ranked table of agencies by adoption %, Assisted Hours, and Assisted Value
    • Page 2 — Feature Adoption Heatmap: Agency × M365 App matrix showing per-app adoption intensity
    • Page 3 — Behavioral Impact: Before/after comparison of meeting time, focus time, email time per agency
    • Page 4 — Trends: Monthly adoption and Assisted Hours trendlines per agency (requires monthly query snapshots)
  4. Deliver via Power BI workspace with CIO as Viewer. Refresh monthly.
Product Limitation: All CIO cross-agency views require a Central IT analyst to build custom PBI reports. The Copilot Dashboard itself has no “compare groups” or “segment by attribute” capability. This is a platform limitation, not a configuration gap.

Agency later requires hard isolation (regulatory change) Medium Migrate to the Isolation Architecture. The partition structure (if created in Phase 4) carries over. The main change is adding the Phase 1 lockdown (disable AutoCxO, Delegation, Agent Dashboard for agency users).

10. Discovery Questions for Customer

  1. Is cross-agency data visibility acceptable, or is hard isolation required? (If isolation is required, use the Isolation Architecture instead.)
  2. How is the Entra ID manager hierarchy structured? Does each agency CEO sit at the top of an independent tree, or is there a shared parent?
  3. How many agencies, and how many employees per agency? (Each agency needs enough employees to meet the minimum group size threshold for “Your group” to display data.)
  4. Do agency CEOs want to delegate dashboard access to their VPs and directors? (Determines whether to actively configure delegation in Phase 2.)
  5. Are there shared-services teams? Where should they appear — in an agency’s “Your group,” or only in the “Your company” view?
  6. Do you need custom analytical filters (Department, Location, Job Function)? (Determines whether to enrich with CSV org data.)
  7. Will dedicated analysts per agency run custom queries? (Determines whether to enable partitions in Phase 4.)
  8. Is there a risk that the cross-agency data-sharing posture might change (e.g., future regulation)? (Plan for possible migration to the Isolation Architecture.)
  9. What is the minimum privacy threshold (minimum group size) acceptable to all agencies? (Tenant-wide setting — all agencies must agree.)
  10. Are Copilot agents deployed across agencies? With the Agent Dashboard enabled for everyone, all leaders see tenant-wide agent data. Confirm this is acceptable.

Connect-MgGraph -Scopes "User.Read.All"

# Get all users and their managers — export to CSV for review
Get-MgUser -All -Property Id,DisplayName,UserPrincipalName,Department,Manager `
  -ExpandProperty Manager |
  Select-Object DisplayName, UserPrincipalName, Department,
    @{N='ManagerUPN';E={$_.Manager.AdditionalProperties.userPrincipalName}},
    @{N='ManagerName';E={$_.Manager.AdditionalProperties.displayName}} |

  -ExpandProperty Manager |
  Where-Object { $null -eq $_.Manager } |
  Select-Object DisplayName, UserPrincipalName |
