Copilot Dashboard & Analytics for Multi-Agency Single Tenant

Document Type: Solution Architecture Artifact
Version: 2.0 | Date: March 2026
Classification: Internal — Customer Deliverable

TL;DR — 5 Things to Do
  1. Lock down the Copilot Dashboard to Central IT only (VFAM + Entra group) — Phase 1
  2. Upload org data with an Agency attribute for every employee — Section 6
  3. Create one partition per agency in Viva Insights, filtered by Agency — Phase 3
  4. Assign agency-specific analysts to their partitions only — Section 7
  5. Deliver reports via Publish Reports or isolated Power BI workspaces — Phase 4

Time estimate: 2–3 days. Prerequisites: 50+ Copilot licences, Insights Admin role, org data CSV ready.

1. Executive Summary

This document provides a validated, end-to-end solution architecture for deploying Microsoft Copilot Dashboard and Copilot Analytics in a single Microsoft 365 tenant that hosts multiple independent agencies. Each agency operates as a distinct organization with its own CEO and full management hierarchy, but all share one Entra ID tenant.

Core Constraint & Key Finding:
The customer requires that Agency A cannot see Agency B's Copilot usage data. After validation against official Microsoft documentation, the Copilot Dashboard does not natively support hard data isolation between organizational units within a single tenant. Users identified as "senior leaders" by Entra ID hierarchy will always see "Your company" data, which includes all agencies.
Recommended Approach: A Two-Tier Architecture
  1. Viva Insights Advanced Analytics with Partitions — for agency-isolated Copilot reporting (primary reporting surface for agencies).
  2. Copilot Dashboard with controlled access — for tenant-level oversight by central IT only (secondary, restricted use).

5. Implementation Runbook

0 Phase 0: Pre-Flight Audit

Before locking anything down, audit the current state. Disabling policies does not revoke existing access.

  • Run Get-VivaModuleFeaturePolicy -ModuleId VivaInsights to audit all current VFAM policies.
  • In M365 Admin Center → Settings → Viva → Viva Insights → Manage user access for Copilot Dashboard, check who currently has access.
  • Identify any agency CEOs or leaders who already have dashboard access (auto-enabled or manually added).
  • Document the current state of all VFAM policies (AutoCxoIdentification, CopilotDashboardDelegation, AgentDashboard, CopilotMetricsExport).
  • Verify the Entra ID ManagerID hierarchy — confirm each agency CEO reports to a different top-level node (not to each other).

1 Phase 1: Foundation (Lockdown)

Prevent agency CEOs from being auto-identified as senior leaders and getting tenant-wide access.

  • Disable AutoCxoIdentification via PowerShell for the entire tenant.
  • Remove any existing auto-enabled agency leaders in M365 Admin Center.
  • Disable CopilotDashboardDelegation via PowerShell to prevent cross-agency leaks.
  • Create Entra group CentralIT-CopilotDashboard-Access and grant dashboard access ONLY to this group.
  • Disable the Agent Dashboard for everyone except Central IT using VFAM: Add-VivaModuleFeaturePolicy -ModuleId VivaInsights -FeatureId AgentDashboard -Name "RestrictAgentDashboard" -IsFeatureEnabled $false -Everyone. Then create a group-level policy to re-enable for Central IT only.
  • Disable the Copilot Metrics Export feature to prevent user-level data downloads: Add-VivaModuleFeaturePolicy -ModuleId VivaInsights -FeatureId CopilotMetricsReport -Name "DisableMetricsExport" -IsFeatureEnabled $false -Everyone.
After Phase 1 — Verification:
Confirm no agency leaders appear in the dashboard access list. Verify all four VFAM policies (AutoCxo, Delegation, AgentDashboard, MetricsExport) show as disabled. Have an agency CEO attempt to access insights.cloud.microsoft and confirm access is denied.

2 Phase 2: Organizational Data

Prepare and upload the foundational data that enables partitioning.

  • Create a consolidated CSV file containing ALL employees across ALL agencies.
  • Include the critical custom attribute: Agency (e.g., "Agency A", "Agency B").
  • Upload via Viva Insights web app or M365 Admin Center.
After Phase 2 — Verification:
Check Viva Insights Admin → Data Quality to verify the Agency attribute has 100% coverage. Verify employee counts per agency match HR records.
Shared Services Decision: If your organisation has shared-services teams (IT, Legal, Finance) that serve multiple agencies, choose one of these approaches:
  1. Dedicated “Shared Services” partition (recommended) — Create a separate partition filtered by Agency = "Shared Services". Central IT analysts access this partition alongside the global partition.
  2. Global partition only — Exclude shared services from all agency partitions. Their data is only visible in the global partition (Central IT only).
  3. Include in every agency partition — Not recommended. Creates duplicate data and inflates agency metrics.

3 Phase 3: Partitions Setup

Create the hard data boundaries.

Before Enabling Partitions — Confirm:
(a) You have a reliable, repeatable process for monthly org data uploads with the Agency attribute.
(b) You have identified at least one Insights Analyst per agency.
(c) You accept that once enabled, turning off partitions requires contacting Microsoft support and will delete all partition configurations, query results, and custom metrics created within partitions.
(d) You understand that privacy settings (minimum group size) are tenant-wide, not per-partition — all agencies must agree on the number.
  • Enable Partitions in Viva Insights Privacy Settings.
  • Create a partition per agency using the filter: Agency = "Agency A". Verify the Filtered employees counter matches the expected headcount for that agency.
  • Assign designated Insights Analysts to their respective agency partitions only. Do NOT assign agency analysts to the global partition.
  • Assign Central IT analysts to the automatically created Global Partition for tenant-wide queries.
Why agency analysts must NOT have global partition access: Analysts with global partition access automatically get Copilot Dashboard access AND Copilot Analytics Reports access — both of which show tenant-wide data across all agencies. This would defeat the entire isolation architecture.
After Phase 3 — Verification:
Have each agency analyst sign in to insights.cloud.microsoft. Confirm they only see their agency’s partition in the partition switcher. Have them run a test person query and verify the employee count matches their agency only.

4 Phase 4: Copilot Reporting

Generate and distribute the isolated reports.

  • Agency analysts run Copilot Power BI template queries within their isolated partition.

Option A: Power BI Workspaces (requires Power BI Pro/Premium licences for viewers)

  • Create dedicated Power BI workspaces per agency (e.g., Agency-A-Copilot-Analytics).
  • Publish reports and grant Viewer access to agency leadership via Entra group.
  • Configure Row-Level Security (RLS) as a secondary control if multiple agencies share a workspace.

Option B: Viva Insights Publish Reports (no Power BI licence required for viewers)

  • Use the Publish Reports feature in the Viva Insights analyst experience.
  • Select “Filter report by manager hierarchy” to automatically scope data to the recipient’s team.
  • Recipients view published reports in the Viva Insights web app → Reports tab.
  • Set expiry date (up to 365 days) and republish as needed.
After Phase 4 — Verification:
Have an agency CEO view their report (either PBI or Viva Insights). Confirm: (a) only their agency’s employees appear, (b) no cross-agency group names or data are visible, (c) the minimum group size threshold is respected for small departments within the agency.

6. Organizational Data Strategy

Data Type Source Refresh Cadence
Employee hierarchy (ManagerId) Centralized HR or per-agency HR feeds Monthly (minimum)
Custom attributes (Agency, FunctionType) HR system (via CSV) Monthly
Copilot usage data Automatic from M365 signals Daily (6-day delay)

Data Quality Rule: The Agency attribute must have 100% coverage. Missing values will exclude employees from all partitions.

Data Refresh Failure Alerting:
If a monthly org data upload fails or the Agency attribute is missing for new employees, those employees silently disappear from all partitions. Nobody gets alerted.

Mitigations:
  • After each upload, check the Data Quality page in Viva Insights Admin to verify attribute coverage.
  • Set a calendar reminder to verify Agency attribute has 100% coverage within 7 days of each upload.
  • Consider automating uploads via the Viva Insights API-based import or Azure blob import for more reliable cadence.

7. Access Control Matrix

Role Copilot Dashboard Partition Access Power BI Reports
Global / Insights Admin ✅ Full ("Your company") ✅ Manage all partitions (Requires Analyst role on Global Partition to query) ✅ All workspaces
Insights Analyst (Agency A) ❌ Not enabled ✅ Agency A partition only Agency A workspace (Contributor)
Agency A CEO / VP ❌ Not enabled ❌ No direct access Agency A workspace (Viewer)

8. Governance & Ongoing Operations

  • Monthly: Upload updated organizational data CSV (Central IT).
  • Monthly: Refresh Copilot PBI template queries and publish reports (Agency Analysts).
  • Quarterly: Audit dashboard access list and partition memberships.
  • Onboarding: To add a new agency, add them to the CSV, create a new partition, assign an analyst, and spin up a new PBI workspace.

Day 2 Operations Runbook

Scenario Who Action
Employee transfers from Agency A to Agency B Central IT Update Agency attribute in the next CSV upload. Employee moves between partitions automatically on next data refresh (up to 7 days).
New agency created Central IT + Insights Admin Add agency to CSV → create new partition → assign analyst → create PBI workspace or use Publish Reports → grant CEO viewer access.
Agency merged or retired Insights Admin Update Agency attribute for merged employees → delete the retired partition (warning: deletes all queries and custom metrics in that partition) → reassign analysts.
Agency analyst leaves Insights Admin Remove from partition → assign replacement. Note: Query results belong to the original analyst — the replacement cannot access historical query results. Re-run queries under the new analyst’s account.
AutoCxo accidentally re-enabled Central IT Run Get-VivaModuleFeaturePolicy -ModuleId VivaInsights → verify state → re-disable → remove any newly auto-enabled agency leaders from the access list immediately.
New hire not appearing in any partition Central IT Check if the employee has the Agency attribute in the org data CSV. If missing, they are excluded from all partitions. Add the attribute and re-upload.

9. Risk Register

Risk Impact Mitigation
Auto-enablement policy is accidentally re-enabled High Document the policy; set quarterly audit of PowerShell feature policies.
Partition filter misconfigured, includes wrong agency High Validate filtered employee count during partition creation; quarterly audit.
Power BI workspace permissions grant cross-agency access High Use dedicated workspaces per agency with strict Entra group-based access.
Copilot Metrics Export enabled — user-level data downloadable High Disable CopilotMetricsReport VFAM control for entire tenant. Only re-enable for Central IT if needed.
Agent Dashboard exposes cross-agency agent usage Medium Disable AgentDashboard VFAM control for everyone except Central IT. Agent Dashboard only has company-level scope.
Analytics Reports auto-available to global partition analysts High Ensure agency analysts are NOT assigned to the global partition. Only Central IT analysts should have global access.
Org data upload fails — employees silently drop from partitions Medium Verify Agency attribute coverage after each upload. Set calendar reminder. Consider automated API-based imports.
Analyst query results not transferable when analyst leaves Low Document this limitation. Re-run critical queries under the replacement analyst before the departing analyst’s access is revoked.

10. Discovery Questions for Customer

  1. How is the Entra ID ManagerID hierarchy currently structured? (Determines if CEOs are auto-identified as senior leaders).
  2. How many total employees per agency, and how many have Copilot licenses? (Partitions need sufficient population to meet the minimum group size threshold, default 10).
  3. Are there shared-services teams (IT, Legal, Finance) that serve multiple agencies? (Determines if a "Shared Services" partition is needed).
  4. What minimum privacy threshold (minimum group size) is acceptable? (This is set tenant-wide; all agencies must agree on the number).
  5. Do agency leaders need Power BI Pro/Premium licences, or should we use Viva Insights Publish Reports? (Determines report distribution method — Phase 4 options.)
  6. Has anyone already been granted Copilot Dashboard access or does any analyst have global partition access? (Must audit and remediate before proceeding with Phase 1 lockdown.)
  7. Is there an automated HR data feed, or will org data uploads be manual CSV? (Impacts reliability of partition boundaries over time.)
  8. Are Copilot agents deployed across agencies? If so, does agent usage data also need isolation? (Determines whether Agent Dashboard needs VFAM lockdown.)

Appendix A: PowerShell Reference

# Install required module
Install-Module -Name ExchangeOnlineManagement -Force -AllowClobber
Connect-ExchangeOnline -UserPrincipalName admin@tenant.com

# Disable Auto-Enablement for Dashboard
Add-VivaModuleFeaturePolicy `
  -ModuleId VivaInsights `
  -FeatureId AutoCxoIdentification `
  -Name "DisableAutoEnableForAllAgencies" `
  -IsFeatureEnabled $false `
  -Everyone

# Disable Delegation
Add-VivaModuleFeaturePolicy `
  -ModuleId VivaInsights `
  -FeatureId CopilotDashboardDelegation `
  -Name "DisableDelegationForTenant" `
  -IsFeatureEnabled $false `
  -Everyone

# Disable Agent Dashboard (company-level scope exposes cross-agency data)
Add-VivaModuleFeaturePolicy `
  -ModuleId VivaInsights `
  -FeatureId AgentDashboard `
  -Name "DisableAgentDashboardForTenant" `
  -IsFeatureEnabled $false `
  -Everyone

# Disable Copilot Metrics Export (prevents user-level data download)
Add-VivaModuleFeaturePolicy `
  -ModuleId VivaInsights `
  -FeatureId CopilotMetricsReport `
  -Name "DisableMetricsExportForTenant" `
  -IsFeatureEnabled $false `
  -Everyone

# Audit current policies
Get-VivaModuleFeaturePolicy -ModuleId VivaInsights
